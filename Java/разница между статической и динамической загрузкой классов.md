    **Статическая загрузка** происходит во время компиляции и запуска JVM, **динамическая** — во время выполнения программы по явному запросу. Различия критичны для гибкости, производительности и архитектуры приложений.

---

## Сравнение подходов

| Характеристика | Статическая загрузка | Динамическая загрузка |
|----------------|---------------------|----------------------|
| **Когда** | При старте JVM, при первом обращении | По требованию (runtime) |
| **Как** | `new`, импорты, статические ссылки | `Class.forName()`, `ClassLoader.loadClass()`, Reflection |
| **Зависимости** | Должны быть доступны при старте | Могут подгружаться позже |
| **Гибкость** | Жёстко зафиксированы | Конфигурируемые, плагины |
| **Производительность** | Быстрее (JIT компиляция) | Накладные расходы на загрузку |
| **Проверка ошибок** | На этапе компиляции | Только в runtime (ClassNotFoundException) |

## Статическая загрузка

### Примеры
```java
// 1. Оператор new
User user = new User();

// 2. Статические методы/поля
int max = Math.max(10, 20);

// 3. Объявление типа
List<String> list = new ArrayList<>();

// 4. Наследование
public class MyServlet extends HttpServlet { ... }

// 5. Импорты (хотя это синтаксический сахар)
import java.util.Date;
```

### Когда происходит загрузка
```java
public class StaticExample {
    // Класс загружается при первом обращении к:
    
    // 1. Создании экземпляра
    User u = new User();          // Загрузка User
    
    // 2. Обращении к статическому члену
    int i = User.MAX_AGE;         // Загрузка User
    
    // 3. Использовании в объявлении метода (не вызове!)
    public User createUser() {    // Загрузка при загрузке StaticExample
        return null;
    }
    
    // 4. Рефлексии (Class.forName с инициализацией)
    Class<?> clazz = Class.forName("com.User"); // Явная загрузка
}
```

### Особенности
```java
public class Demo {
    static {
        System.out.println("Класс Demo загружен и инициализирован!");
    }
}

// В другом классе:
public class Test {
    public static void main(String[] args) {
        Demo d = null;           // НЕ загружает Demo!
        System.out.println("After declaration");
        
        Demo d2 = new Demo();    // Загружает! Выводит сообщение
    }
}
```

**Вывод:**
```
After declaration
Класс Demo загружен и инициализирован!
```

## Динамическая загрузка

### Способы

#### 1. Class.forName() — полная инициализация
```java
// Загружает, связывает, инициализирует (выполняет static блоки)
try {
    Class<?> clazz = Class.forName("com.mysql.jdbc.Driver");
    
    // Создание экземпляра
    Object obj = clazz.getDeclaredConstructor().newInstance();
    
    // Или с параметрами
    Constructor<?> ctor = clazz.getConstructor(String.class, int.class);
    Object obj2 = ctor.newInstance("localhost", 3306);
    
} catch (ClassNotFoundException e) {
    // Класс не найден в classpath
}
```

#### 2. ClassLoader.loadClass() — без инициализации
```java
// Только загрузка, без static блоков!
ClassLoader loader = Thread.currentThread().getContextClassLoader();
Class<?> clazz = loader.loadClass("com.example.Plugin");

// Инициализация отложена до:
clazz.getDeclaredConstructor().newInstance(); // Здесь выполнятся static блоки
```

#### 3. ServiceLoader (Java 6+) — плагинная система
```java
// META-INF/services/com.example.Service
// содержит: com.impl.ServiceImpl

ServiceLoader<Service> loader = ServiceLoader.load(Service.class);
for (Service service : loader) {
    service.execute(); // Автоматическая динамическая загрузка
}
```

#### 4. ModuleLayer (Java 9+) — динамические модули
```java
ModuleFinder finder = ModuleFinder.of(Paths.get("/plugins"));
ModuleLayer parent = ModuleLayer.boot();
Configuration cfg = parent.configuration().resolve(finder, ModuleFinder.of(), Set.of("plugin.module"));
ModuleLayer layer = parent.defineModulesWithOneLoader(cfg, ClassLoader.getSystemClassLoader());

Class<?> clazz = layer.findLoader("plugin.module").loadClass("plugin.Main");
```

## Практические примеры

### Пример 1: Фабрика драйверов БД
```java
// Статически — жёсткая зависимость
import com.mysql.jdbc.Driver; // Привязка к MySQL!

// Динамически — гибкость
public class DatabaseManager {
    public Connection connect(String dbType, String url) throws Exception {
        // Загружаем драйвер по имени из конфига!
        Class.forName("com." + dbType + ".jdbc.Driver");
        
        return DriverManager.getConnection(url);
    }
}

// Использование:
manager.connect("mysql", "jdbc:mysql://localhost/db");
manager.connect("postgresql", "jdbc:postgresql://localhost/db");
```

### Пример 2: Плагинная система
```java
public class PluginManager {
    private final Map<String, Plugin> plugins = new HashMap<>();
    
    public void loadPlugin(String jarPath, String className) throws Exception {
        URLClassLoader loader = new URLClassLoader(
            new URL[]{new File(jarPath).toURI().toURL()},
            this.getClass().getClassLoader()
        );
        
        // Динамическая загрузка!
        Class<?> clazz = loader.loadClass(className);
        
        // Проверка интерфейса
        if (!Plugin.class.isAssignableFrom(clazz)) {
            throw new IllegalArgumentException("Not a Plugin!");
        }
        
        Plugin plugin = (Plugin) clazz.getDeclaredConstructor().newInstance();
        plugins.put(plugin.getName(), plugin);
    }
}

// Плагин загружается БЕЗ перекомпиляции основного приложения!
```

### Пример 3: Десериализация
```java
// Jackson динамически загружает классы по имени из JSON
public class Deserializer {
    public Object deserialize(String json, String className) throws Exception {
        Class<?> clazz = Class.forName(className);
        return objectMapper.readValue(json, clazz);
    }
}

// JSON: {"@class": "com.example.User", "name": "Alice"}
// Класс User загружается динамически при чтении!
```

### Пример 4: AOP и прокси
```java
// Spring создаёт прокси динамически
public class ProxyFactory {
    public Object createProxy(Class<?> targetClass) {
        // Загрузка CGLIB или JDK прокси класса
        Class<?> proxyClass = createProxyClass(targetClass);
        return proxyClass.getDeclaredConstructor().newInstance();
    }
}
```

## Различия в деталях

### Инициализация static блоков
```java
public class TestClass {
    static {
        System.out.println("Static init executed!");
    }
}

// Статическая загрузка — сразу инициализация
TestClass obj = new TestClass(); // Вывод: Static init executed!

// Динамическая — разница между forName и loadClass
Class<?> c1 = Class.forName("TestClass");           // Вывод сразу!
Class<?> c2 = loader.loadClass("TestClass");        // Без вывода!
Object o = c2.getDeclaredConstructor().newInstance(); // Вывод здесь!
```

### Обработка зависимостей
```java
// Статическая — все классы должны быть при старте
public class OrderService {
    private PaymentService paymentService = new PaymentService(); // Должен быть!
}

// Динамическая — зависимости загружаются по требованию
public class DynamicOrderService {
    private Object paymentService;
    
    public void init() throws Exception {
        Class<?> clazz = Class.forName(
            System.getProperty("payment.service.class")
        );
        this.paymentService = clazz.getDeclaredConstructor().newInstance();
    }
}
```

## Когда что использовать

| Сценарий | Рекомендация |
|----------|-------------|
| Основные классы приложения | Статическая |
| Плагины, расширения | Динамическая |
| Драйверы БД, JDBC | Динамическая (`Class.forName`) |
| Фреймворки (Spring, Hibernate) | Динамическая (рефлексия) |
| Критичные по скорости участки | Статическая (JIT оптимизация) |
| Модули, которые могут отсутствовать | Динамическая с graceful degradation |

## Производительность

```java
// Статическая — JIT может инлайнить, оптимизировать
public void staticCall() {
    Helper.help(); // Быстро, прямой вызов
}

// Динамическая — overhead на рефлексию
public void dynamicCall() throws Exception {
    Method method = Class.forName("Helper")
        .getMethod("help");
    method.invoke(null); // Медленнее! Проверки доступа, boxing
}

// Оптимизация: MethodHandle (Java 7+) или LambdaMetafactory
MethodHandle handle = MethodHandles.lookup()
    .findStatic(Helper.class, "help", MethodType.methodType(void.class));
handle.invoke(); // Быстрее reflection, почти как прямой вызов
```

## Итог

```java
// Статическая — предсказуемость, скорость, проверка на этапе компиляции
User user = new User();

// Динамическая — гибкость, расширяемость, позднее связывание
Class<?> clazz = Class.forName(config.getUserClassName());
Object user = clazz.getDeclaredConstructor().newInstance();
```

Динамическая загрузка — основа фреймворков, плагинов и reflection-based инструментов, но требует аккуратной обработки исключений и понимания ClassLoader'ов.
