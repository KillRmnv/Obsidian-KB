Сделал для тебя структурированный гайд. Поскольку ты используешь **JetBrains** как базовую раскладку (`"base_keymap": "JetBrains"` в твоих настройках), большинство команд будут тебе привычны, но я добавил специфичные для Zed моменты.

# ⌨️ Шпаргалка Zed (JetBrains Keymap)

## 🔍 Поиск и Навигация

|**Действие**|**Комбинация клавиш**|**Контекст**|
|---|---|---|
|**Поиск по файлу**|`Ctrl + F`|Регулярные выражения включаются кнопкой `.*` в строке поиска|
|**Поиск и замена в проекте**|`Ctrl + Shift + F`|Глобальный поиск (Find and Replace in Files)|
|**Переход к файлу**|`Ctrl + Shift + N`|Быстрое открытие любого файла в проекте|
|**Переход к символу (методу/классу)**|`Ctrl + F12`|Навигация по структуре текущего файла|
|**Переход к реализации**|`Ctrl + Клик` или `Ctrl + B`|Go to Definition|
|**Показать Javadoc / Ошибку**|`Ctrl + Q` (или наведение мыши)|Просмотр сигнатуры и документации|

---

## 🛠️ Рефакторинг и Написание кода

|**Действие**|**Комбинация клавиш**|**Примечание**|
|---|---|---|
|**Переименовать везде**|`F2`|Переименование функции/переменной во всем проекте (LSP)|
|**Быстрые исправления**|`Ctrl + .`|Auto-import, создание методов, исправление ошибок|
|**Форматировать файл**|`Ctrl + Alt + L`|Использует JDTLS (Java) или Pyright (Python)|
|**Окружить кодом**|`Ctrl + Alt + T`|(Если поддерживает LSP) Например, в `try-catch`|
|**Code Actions**|`Alt + Enter`|Аналог лампочки в IntelliJ|

---

## 🖱️ Редактирование и Выделение

|**Действие**|**Комбинация клавиш**|**Описание**|
|---|---|---|
|**Мультикурсор**|`Alt + Клик`|Ставит дополнительные курсоры|
|**Дублировать строку**|`Ctrl + D`|Копирует текущую строку вниз|
|**Удалить строку**|`Ctrl + Y`|Быстрое удаление строки|
|**Переместить строку вверх/вниз**|`Alt + Shift + ↑ / ↓`|Двигает строку по коду|
|**Расширить выделение**|`Ctrl + W`|Умное выделение блоков кода (Scope)|

## 🖥️ Управление интерфейсом Zed

|**Действие**|**Комбинация клавиш**|**Описание**|
|---|---|---|
|**Командная палитра**|`Ctrl + Shift + P`|Доступ ко всем командам Zed|
|**Показать логи**|`Ctrl + Shift + P` -> `zed: open logs`|Тот самый поиск ошибок LSP|
|**Скрыть/Показать боковую панель**|`Ctrl + 1`|Управление Project Panel|
|**Встроенный терминал**|`` Ctrl + ` ``|Открыть/закрыть терминал|
|**Разбить экран (Split)**|`Ctrl + \`|Вертикальное разделение окна|

## 🔧 Как добавить свои (если чего-то не хватает)

Если ты хочешь добавить функцию "Выделение в функцию" (Extract Method) на конкретную клавишу, открой `keymap.json` (`Ctrl + Shift + P` -> `zed: open keymap`) и добавь:

JSON

```
[
  {
    "context": "Editor",
    "bindings": {
      "ctrl-alt-m": "editor:code_actions" 
    }
  }
]
```

> **Совет:** В Zed для Java переименование файла при переименовании класса лучше всего делать через `F2` на самом имени класса внутри кода. Тогда JDTLS корректно обновит всё дерево проекта.
