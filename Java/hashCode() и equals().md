

Связь между `equals()` и `hashCode()` — это один из фундаментальных контрактов в Java. Если кратко: **они должны быть согласованы**, чтобы объекты корректно работали в коллекциях типа `HashMap`, `HashSet` и `Hashtable`.

### Суть контракта

Согласно документации Oracle, правила игры следующие:

1. **Если два объекта равны** (вызов `obj1.equals(obj2)` возвращает `true`), то их **хэш-коды должны быть одинаковыми** (`obj1.hashCode() == obj2.hashCode()`).
    
2. **Если хэш-коды одинаковы**, это **не означает**, что объекты равны. Это называется _коллизией_.
    
3. **Если хэш-коды разные**, то объекты **гарантированно не равны**.
