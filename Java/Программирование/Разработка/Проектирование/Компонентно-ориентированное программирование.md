Парадигма, в которой система строится из **самостоятельных, переиспользуемых компонентов**, каждый из которых реализует определённый функционал и имеет **чётко определённый интерфейс**.

- **Компонент** — автономный модуль с закрытой реализацией, предоставляющий набор операций через интерфейс. Внутреннее устройство компонента скрыто, что обеспечивает изоляцию и повторное использование.
    
- **Клиент** — часть системы, использующая функционал компонента через его интерфейс, не зная деталей реализации.
    

Ключевые принципы:

- **Интерфейсы и контракт**: взаимодействие через строго определённые точки входа.
    
- **Повторное использование**: компоненты можно применять в разных системах без изменений.
    
- **Изоляция реализации**: изменения внутри компонента не влияют на клиентов.
    

Компонентно-ориентированное программирование строит систему из **обособленных компонентов**, которые предоставляют функционал клиентам через интерфейсы, обеспечивая модульность, расширяемость и повторное использование.
