**Entity–Relationship**
***
Диаграмма, которая используется для **проектирования и описания структуры базы данных**. Она показывает, какие **сущности** есть в системе, какими **атрибутами** они обладают и **как связаны друг с другом**.

ER-диаграммы применяют на этапе проектирования БД, чтобы понять модель данных до создания таблиц, ключей и связей. Они помогают избежать дублирования данных и логических ошибок.

**Основные концепции:**
- Сущность
- Атрибуты сущности
- Идентичность сущности
- Экземпляр сущности
- Связи


## Типы связи

|    Тип связи     | Обозначение | Смысл                                                                        |
| :--------------: | :---------: | :--------------------------------------------------------------------------- |
|  Один к многим   |     1:M     | Один экземпляр первой сущности связан с многими второй.                      |
|  Один к одному   |     1:1     | Один к одному.                                                               |
| Многие к одному  |     M:1     | Обратная 1:M.                                                                |
| Многие ко многим |     M:N     | Множество экземпляров каждой стороны может быть связано с множеством другой. |
