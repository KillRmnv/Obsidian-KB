**Формальное определение:**  
Функциональное программирование — парадигма программирования, в которой процесс вычисления трактуется как вычисление значений математических функций, с минимизацией или полным исключением изменения состояния и побочных эффектов. Программа рассматривается как совокупность композированных чистых функций, преобразующих входные данные в выходные, без модификации исходных данных.

**Ключевые характеристики:**

- Функции как первоклассные объекты (можно присваивать переменным, передавать параметрами, возвращать из других функций)
    
- Неизменяемость данных (immutability) — данные не изменяются, создаются новые копии
    
- Отсутствие побочных эффектов (side effects-free) — функции не используют и не изменяют глобальное состояние
    
- Чистые функции (pure functions) — результат зависит только от входных параметров
    
- Высший порядок функций (higher-order functions) — функции могут принимать и возвращать функции
    

**Примеры языков:** Haskell, Lisp, Clojure, Scala; поддержка в Java, Python, JavaScript

## Ключевой концепт: Чистые функции

**Определение:** Функция чистая, если:

1. **Результат зависит только от входных параметров** — нет обращения к глобальному состоянию
    
2. **Нет побочных эффектов** — не изменяет состояние переменных вне себя, не выполняет I/O, не записывает в БД
    

**Нечистая функция (антипаттерн):**

text

`let globalCounter = 0; function impureAdd(x, y) {     globalCounter++;  // побочный эффект!    return x + y; } impureAdd(2, 3);  // 5, и globalCounter стал 1 impureAdd(2, 3);  // 5, и globalCounter стал 2 // Один и тот же вызов дал разные побочные эффекты!`

**Чистая функция (правильно):**

text

`function pureAdd(x, y) {     return x + y; } pureAdd(2, 3);  // 5 (всегда одинаково) pureAdd(2, 3);  // 5 (всегда одинаково)`

**Почему это важно для моделирования:**

- Чистые функции **предсказуемы** — их поведение не зависит от того, в каком порядке они вызываются
    
- Чистые функции **легко тестировать** — просто проверяешь входы и выходы, никакого контекста
    
- Чистые функции можно **параллелить** — система может вызывать их в любом порядке без риска гонок данных

## 5. Неизменяемость (Immutability)

**В ООП:** Объект может менять свое состояние

text

`account.balance = 1000 account.deposit(500) account.balance = 1500  // состояние изменилось`

**В функциональном подходе:** Данные не меняются, создаются новые

text

`balance_v1 = 1000 balance_v2 = deposit(balance_v1, 500)  // balance_v2 = 1500 // balance_v1 остался 1000 (не изменилась)`

**Преимущество:**

- История всех состояний сохранена (как в системе контроля версий)
    
- Невозможно случайно испортить данные (они read-only)
    
- Ясно, где и как изменяются значения

## 6. Высший порядок функций (Higher-Order Functions)

**Определение:** Функция, которая принимает функцию как параметр или возвращает функцию.

**Пример 1: Функция как параметр**

text

`function applyOperation(x, y, operation) {     return operation(x, y); } const add = (a, b) => a + b; const multiply = (a, b) => a * b; applyOperation(5, 3, add);      // 8 applyOperation(5, 3, multiply); // 15`

**Пример 2: Возврат функции (Currying)**

text

`function multiplier(factor) {     return function(number) {        return number * factor;    } } const double = multiplier(2); const triple = multiplier(3); double(5);  // 10 triple(5);  // 15`

**Смысл для моделирования:** Позволяет создавать абстракции функционального уровня. Вместо того чтобы создавать класс `Multiplier`, просто создаёшь функцию, которая возвращает функцию-множитель.

## 7. Композиция функций

**Определение:** Объединение нескольких функций в одну функцию, которая их последовательно применяет.

**Антипаттерн (ООП):**

text

`class DataProcessor {     parse() { ... }    validate() { ... }    transform() { ... } }`

**Функциональный подход:**

text

`const parse = (data) => JSON.parse(data); const validate = (obj) => obj.isValid() ? obj : null; const transform = (obj) => ({ ...obj, processed: true }); const pipeline = (data) => transform(validate(parse(data))); // или с helper'ом compose: const pipeline = compose(transform, validate, parse);`

**Преимущество:** Функции полностью независимы. Ты можешь переиспользовать `parse` в другом месте кода, не беспокоясь о контексте класса.

## 8. Принципиальное различие в расширении кода

## ООП: добавляем новое поведение через новые методы

text

`Система: есть 3 типа животных (Dog, Cat, Bird) Операция: издавать звук Решение ООП:   class Dog {    speak() { return "Woof"; }  }  class Cat {    speak() { return "Meow"; }  }  // Каждый класс отвечает за свой звук`

**Хорошо, если:** часто добавляются новые типы животных (новые классы)  
**Плохо, если:** часто добавляются новые операции (нужно менять все классы)

---

## Функциональный подход: добавляем новые функции

text

`Система: 3 вида данных (dog, cat, bird) — просто объекты/записи Операция: издавать звук Решение ФП:   const dogSound = (animal) => animal.type === "dog" ? "Woof" : null;  const catSound = (animal) => animal.type === "cat" ? "Meow" : null;     // Или проще:  const sounds = {    dog: "Woof",    cat: "Meow",    bird: "Tweet"  };     const speak = (animal) => sounds[animal.type];`

**Хорошо, если:** часто добавляются новые операции (просто новая функция)  
**Плохо, если:** часто добавляются новые типы (нужно расширять все функции)
