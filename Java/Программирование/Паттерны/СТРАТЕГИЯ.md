[[Поведенческий паттерн|Поведенческий паттерн проектирования]], который определяет семейство схожих алгоритмов и помещает каждый из них в собственный класс, после чего алгоритмы можно взаимозаменять прямо во время исполнения программы.
***
## Пример

```python
from abc import ABC, abstractmethod

# Абстрактная стратегия
class DiscountStrategy(ABC):
    @abstractmethod
    def apply_discount(self, price: float) -> float:
        pass

# Конкретная стратегия: Без скидки
class NoDiscount(DiscountStrategy):
    def apply_discount(self, price: float) -> float:
        return price

# Конкретная стратегия: 10% скидка
class NewCustomerDiscount(DiscountStrategy):
    def apply_discount(self, price: float) -> float:
        return price * 0.9  # 10% скидка

# Конкретная стратегия: 20% скидка на распродажу
class SaleDiscount(DiscountStrategy):
    def apply_discount(self, price: float) -> float:
        return price * 0.8  # 20% скидка

# Контекст: Заказ
class Order:
    def __init__(self, price: float, discount_strategy: DiscountStrategy):
        self.price = price
        self.discount_strategy = discount_strategy

    def final_price(self) -> float:
        return self.discount_strategy.apply_discount(self.price)

    def change_strategy(self, new_strategy: DiscountStrategy):
        self.discount_strategy = new_strategy
```
