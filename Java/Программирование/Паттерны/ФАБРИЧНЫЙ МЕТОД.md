[[Порождающий паттерн|Порождающий паттерн проектирования]], который определяет общий интерфейс для создания объектов в суперклассе, позволяя подклассам изменять тип создаваемых объектов.
***
## Основная идея

- Создание объектов выносится в отдельный метод
- Подклассы могут переопределять этот метод для создания объектов конкретных типов
- Избавляет клиентский код от привязки к конкретным классам
***

## Пример

```python
from abc import ABC, abstractmethod

# Абстрактный продукт
class Product(ABC):
    @abstractmethod
    def operation(self):
        """Операция, которую выполняет продукт."""
        pass

# Конкретные продукты
class ConcreteProductA(Product):
    def operation(self):
        return "Результат от Product A"

class ConcreteProductB(Product):
    def operation(self):
        return "Результат от Product B"

# Абстрактный создатель (фабрика)
class Creator(ABC):
    @abstractmethod
    def factory_method(self):
        """Фабричный метод для создания продукта."""
        pass

    def some_operation(self):
        """Операция создателя, которая использует продукт."""
        product = self.factory_method()  # Здесь вызывается фабричный метод
        result = f"Creator зависит от {product.operation()}"
        return result

# Конкретные создатели
class ConcreteCreatorA(Creator):
    def factory_method(self):
        return ConcreteProductA()

class ConcreteCreatorB(Creator):
    def factory_method(self):
        return ConcreteProductB()
```
