[[Структурный паттерн|Структурный паттерн проектирования]], который позволяет объектам с несовместимыми интерфейсами работать вместе. Он выступает в роли "переводчика" между двумя объектами.
***
Преобразует интерфейс класса к другому интерфейсу, на который рассчитан клиент. Адаптер [[Чистый код]]местную работу классов, невозможную в обычных условиях из-за несовместимости интерфейсов.
***

## Адаптер объектов
### Пример

```python
# Целевой интерфейс, который ожидает наша система
class PaymentProcessor:
    def pay(self, amount):
        pass

# Существующий класс PayPal с несовместимым интерфейсом
class PayPal:
    def send_money(self, amount, currency="USD"):
        return f"Отправлено {amount} {currency} через PayPal"

# Существующий класс Stripe с другим несовместимым интерфейсом
class Stripe:
    def create_charge(self, amount_cents):
        return f"Создан платеж на {amount_cents/100}$ через Stripe"

# Адаптер для PayPal
class PayPalAdapter(PaymentProcessor):
    def __init__(self, paypal):
        self.paypal = paypal
    
    def pay(self, amount):
        # Конвертируем интерфейс: доллары -> доллары
        return self.paypal.send_money(amount, "USD")

# Адаптер для Stripe
class StripeAdapter(PaymentProcessor):
    def __init__(self, stripe):
        self.stripe = stripe
    
    def pay(self, amount):
        # Конвертируем интерфейс: доллары -> центы
        amount_cents = int(amount * 100)
        return self.stripe.create_charge(amount_cents)

# Клиентский код, который работает с единым интерфейсом
def process_payment(processor: PaymentProcessor, amount):
    result = processor.pay(amount)
    print(f"Платеж обработан: {result}")
```

###  Когда использовать?

1. **Несовместимые интерфейсы:** Вы хотите использовать существующий класс, но его интерфейс не соответствует тому, что ожидает ваш код.
2. **Работа с legacy или сторонними библиотеками:** Вы не можете изменить код существующего класса (он закрыт, устарел или принадлежит другой библиотеке), но вам нужно интегрировать его в свою систему.
3. **Унификация интерфейсов:** У вас есть несколько классов с похожей функциональностью, но разными методами. Вы можете создать единый интерфейс и адаптеры для каждого класса, чтобы клиент работал с ними одинаково.

###  Структура паттерна

В паттерне участвуют 4 компонента:

1. **Цель (Target):** Интерфейс, который ожидает Клиент.
2. **Адаптируемый (Adaptee):** Существующий класс, который нужно приспособить (его интерфейс не подходит Клиенту).
3. **Адаптер (Adapter):** Класс, который реализует интерфейс `Target` и внутри использует `Adaptee`. Он преобразует вызовы.
4. **Клиент (Client):** Код, который работает с интерфейсом `Target`.
