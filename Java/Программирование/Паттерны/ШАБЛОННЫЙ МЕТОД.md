[[Поведенческий паттерн|Поведенческий паттерн проектирования]], который определяет скелет алгоритма в абстрактном базовом классе, позволяя подклассам переопределять отдельные шаги алгоритма, не меняя его общую структуру.
***
## Пример

```python
from abc import ABC, abstractmethod

class Game(ABC):
    def play(self):  # Шаблонный метод: фиксированный алгоритм
        self.initialize()
        self.start_play()
        self.end_play()

    @abstractmethod
    def initialize(self):
        pass  # Подклассы реализуют инициализацию

    @abstractmethod
    def start_play(self):
        pass  # Подклассы реализуют начало игры

    @abstractmethod
    def end_play(self):
        pass  # Подклассы реализуют конец игры

class Chess(Game):
    def initialize(self):
        print("Шахматы инициализированы! Разместите фигуры на доске.")

    def start_play(self):
        print("Начало игры в шахматы!")

    def end_play(self):
        print("Конец игры в шахматы! Спасибо за игру!")

class Soccer(Game):
    def initialize(self):
        print("Футбол инициализирован! Разместите мяч в центре поля.")

    def start_play(self):
        print("Начало игры в футбол!")

    def end_play(self):
        print("Конец игры в футбол! Спасибо за игру!")
```
