## Что такое ковариантность?

**Ковариантность** — это свойство, позволяющее использовать более конкретный (производный) тип там, где ожидается более общий (базовый) тип.

В Java ковариантность проявляется в трех основных областях:

---

## 1. Ковариантность массивов (с самого начала Java)

**Массивы в Java ковариантны** — это одно из самых известных и опасных мест в языке.

```java
Object[] objects = new String[10]; // Ковариантность: String[] → Object[]
objects[0] = "Hello"; // OK
objects[1] = 42; // Ошибка времени выполнения: ArrayStoreException!
```

Проблема: компилятор разрешает присвоение, но в runtime мы получаем ошибку, т.к. массив на самом деле `String[]`.

---

## 2. Ковариантность возвращаемого типа метода (с Java 5)

**Возвращаемые типы методов могут быть ковариантными** при переопределении:

```java
class Animal {
    public Animal getChild() {
        return new Animal();
    }
}

class Dog extends Animal {
    @Override
    public Dog getChild() { // Ковариантность: Dog → Animal
        return new Dog();
    }
}
```

Это безопасно и очень полезно!

---

## 3. Ковариантность в дженериках (Wildcards)

### Ковариантность с `? extends`

```java
List<? extends Number> numbers = new ArrayList<Integer>(); // OK
// numbers.add(new Integer(10)); // ОШИБКА компиляции!
Number n = numbers.get(0); // OK - можно читать как Number
```

**Важно:** `List<? extends Number>` — **read-only** в смысле добавления новых элементов (кроме `null`).

### Контрвариантность с `? super` (для полноты картины)

```java
List<? super Integer> list = new ArrayList<Number>();
list.add(new Integer(10)); // OK - можно добавлять Integer
// Integer i = list.get(0); // ОШИБКА - нельзя читать как Integer
Object obj = list.get(0); // OK - можно читать только как Object
```

---

## Сравнение вариантов

| Тип | Ковариантность | Безопасность | Пример |
|-----|---------------|--------------|---------|
| Массивы | Да (с рождения Java) | Нет (runtime ошибки) | `Object[] arr = new String[5]` |
| Возвращаемый тип | Да (с Java 5) | Да | `Dog getChild()` переопределяет `Animal getChild()` |
| Дженерики | С `? extends` | Да (compile-time) | `List<? extends Number>` |
| Дженерики (инвариантность) | Нет по умолчанию | Да | `List<Number> != List<Integer>` |

---

## PECS (Producer Extends, Consumer Super)

Полезная мнемоника для работы с wildcards:

```java
// Producer - только производит (отдает) значения
void printNumbers(List<? extends Number> numbers) {
    for (Number n : numbers) {
        System.out.println(n);
    }
}

// Consumer - только потребляет (принимает) значения
void addIntegers(List<? super Integer> list) {
    list.add(1);
    list.add(2);
    list.add(3);
}
```
