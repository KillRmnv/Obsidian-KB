## Что такое инвариантность?

**Инвариантность** — это свойство, при котором типы считаются совместимыми **только если они в точности совпадают**.

В Java **дженерики по умолчанию инварианны** — это одно из ключевых отличий от массивов!

---

## Пример инвариантности дженериков

```java
List<Animal> animals = new ArrayList<Animal>();
List<Dog> dogs = new ArrayList<Dog>();

// animals = dogs; // ОШИБКА компиляции!
// dogs = animals; // ОШИБКА компиляции!

// НЕЛЬЗЯ, несмотря на то что Dog extends Animal
// List<Animal> != List<Dog>
```

**Почему так?** Для безопасности типов в compile-time.

---

## Проблема, которую решает инвариантность

Представим, что дженерики были бы ковариантны как массивы:

```java
// ПСЕВДОКОД: если бы дженерики были ковариантны
List<Animal> animals = new ArrayList<Dog>(); // Предположим, разрешено
animals.add(new Cat()); // Компилятор бы разрешил (Cat тоже Animal)
// Но в runtime: ArrayList<Dog> не может содержать Cat!
```

Инвариантность предотвращает такие ошибки **на этапе компиляции**.

---

## Сравнение с ковариантностью массивов

```java
// МАССИВЫ (ковариантны - ОПАСНО)
Object[] objects = new String[3];
objects[0] = "Hello"; // OK
objects[1] = 123; // ArrayStoreException в RUNTIME!

// ДЖЕНЕРИКИ (инвариантны - БЕЗОПАСНО)
List<Object> objectsList = new ArrayList<String>(); // ОШИБКА компиляции
// Предотвращает проблему на этапе компиляции!
```

## Как обойти инвариантность? (Wildcards)

Хотя дженерики инвариантны, можно использовать wildcards для контролируемой вариантности:

### 1. Ковариантность через `? extends`
```java
List<? extends Animal> animals = new ArrayList<Dog>(); // OK
// animals.add(new Dog()); // ОШИБКА - нельзя добавлять
Animal a = animals.get(0); // OK - можно читать
```

### 2. Контравариантность через `? super`
```java
List<? super Dog> dogs = new ArrayList<Animal>(); // OK
dogs.add(new Dog()); // OK - можно добавлять Dog
// Dog d = dogs.get(0); // ОШИБКА - нельзя читать как Dog
Object o = dogs.get(0); // OK - только как Object
```

## Инвариантность в разных контекстах

### 1. **Коллекции (инвариантны)**
```java
Set<Number> numbers = new HashSet<Integer>(); // Ошибка
Map<String, Number> map = new HashMap<String, Integer>(); // Ошибка
```

### 2. **Параметры методов (инвариантны)**
```java
void process(List<Animal> animals) { }

List<Dog> dogs = new ArrayList<>();
// process(dogs); // Ошибка!
processWildcard(dogs); // OK, если параметр List<? extends Animal>
```

### 3. **Возвращаемые значения (ковариантны)**
```java
interface Factory<T> {
    T create();
}

class DogFactory implements Factory<Dog> {
    @Override
    public Dog create() { // Dog → Animal (ковариантность)
        return new Dog();
    }
}
```

## Когда нужна инвариантность?

### Пример безопасного класса с инвариантностью
```java
class Box<T> {
    private T value;
    
    public void set(T value) { this.value = value; }
    public T get() { return value; }
    
    // Инвариантность сохраняет безопасность
    public void copyFrom(Box<T> other) {
        this.value = other.value; // Типы гарантированно совпадают
    }
}

Box<Animal> animalBox = new Box<>();
Box<Dog> dogBox = new Box<>();
// animalBox.copyFrom(dogBox); // ОШИБКА - защита от несовместимости
```

## Сравнение вариантностей

| Вариантность | Дженерики по умолчанию | Массивы | Возвращаемый тип |
|-------------|----------------------|---------|------------------|
| **Инвариантность** | ✅ Да | ❌ Нет | ❌ Нет |
| **Ковариантность** | ❌ Нет (только с `? extends`) | ✅ Да | ✅ Да |
| **Контравариантность** | ❌ Нет (только с `? super`) | ❌ Нет | ❌ Нет |

---

## Практическое правило

```java
// ИНВАРИАНТНОСТЬ (по умолчанию) - строгая проверка
List<Animal> list1 = new ArrayList<Dog>(); // ОШИБКА

// Для гибкости используйте:
// 1. Ковариантность (только чтение)
List<? extends Animal> list2 = new ArrayList<Dog>(); // OK

// 2. Контравариантность (только запись)
List<? super Dog> list3 = new ArrayList<Animal>(); // OK

// 3. Инвариантность (чтение и запись)
List<Dog> list4 = new ArrayList<Dog>(); // OK - полный контроль
```

---

## Ключевые выводы

1. **Инвариантность дженериков** — дизайнерское решение Java для безопасности типов
2. **Предотвращает runtime ошибки**, перенося проверки в compile-time
3. **Отличается от поведения массивов** (историческая особенность)
4. **Можно ослабить** через wildcards (`? extends` / `? super`)
5. **Инвариантность по умолчанию** — это "лучшая защита по умолчанию"

Инвариантность делает код более предсказуемым и безопасным, хоть и требует иногда больше шаблонного кода для работы с иерархиями типов.
