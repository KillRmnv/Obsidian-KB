**Flux architecture**
***
Архитектурный шаблон для построения клиентских приложений, основанный на **однонаправленном потоке данных**.

Суть Flux заключается в строгом разделении ролей компонентов и запрете двунаправленных связей между ними. Все изменения состояния проходят по одному и тому же предсказуемому пути.

Основные элементы архитектуры:

- **Action** - описание события или намерения изменить состояние.
- **Dispatcher** - центральный механизм, передающий действия всем хранилищам.
- **Store** - хранит состояние и бизнес-логику приложения.
- **View** - пользовательский интерфейс, который отображает состояние и инициирует действия.

## Принцип работы

Пользовательское действие во View создаёт Action -> Action передаётся через Dispatcher в Store -> Store обновляет состояние -> View перерисовывается на основе нового состояния.

Преимуществами Flux являются предсказуемость изменений состояния, упрощённая отладка и ясный поток данных. 

Недостатком является усложнение архитектуры и избыточность для небольших приложений.
