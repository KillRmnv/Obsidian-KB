## **Монотонное время (Monotonic Time)**

### Определение:
Монотонное время — это **строго возрастающий** счетчик времени, который гарантированно никогда не идет назад и не подвержен корректировкам.

### Характеристики:
- **Только вперед**: всегда увеличивается, никогда не уменьшается
- **Не связано с реальным временем**: это условные "тики", а не часы/минуты
- **Не подвержено корректировкам**: NTP, часовые пояса, летнее время не влияют
- **Не обязательно начинается с эпохи**: может начинаться с загрузки системы

### Источники в системе:
- **TSC (Time Stamp Counter)**: счетчик тактов процессора
- **HPET (High Precision Event Timer)**: аппаратный таймер
- **jiffies**: в Linux, счетчик тиков таймера ядра
- **CLOCK_MONOTONIC**: в Linux/Unix системах

### Использование:
```c
// Linux пример
struct timespec ts;
clock_gettime(CLOCK_MONOTONIC, &ts);
// ts.tv_sec, ts.tv_nsec - монотонное время
```

```java
// Java
long start = System.nanoTime(); // Монотонное время
// ... выполнение операции
long elapsed = System.nanoTime() - start;
```

### Применение:
- **Измерение интервалов**: профилирование, бенчмаркинг
- **Таймауты и дедлайны**: сетевые соединения, алгоритмы
- **Планировщики задач**: отслеживание времени выполнения
- **Анимации и игры**: расчет дельты времени между кадрами

### Пример проблемы, которую решает:
```java
// Без монотонного времени возможна проблема:
long start = System.currentTimeMillis();
// ... выполнение
// Администратор меняет системное время назад
long end = System.currentTimeMillis();
long elapsed = end - start; // Может быть отрицательным!
```

## **Календарное время (Wall-clock Time)**

### Определение:
Календарное время — это **реальное астрономическое время**, соответствующее человеческому восприятию времени (год, месяц, день, час).

### Характеристики:
- **Соответствует реальному миру**: привязано к часовым поясам
- **Подвержено корректировкам**: NTP, летнее время, смена поясов
- **Может идти назад**: при синхронизации времени
- **Начинается с эпохи**: обычно Unix Epoch (1 января 1970)

### Источники:
- **Системные часы (RTC)**: реальное время компьютера
- **NTP серверы**: сетевые источники точного времени
- **CLOCK_REALTIME**: в Linux системах

### Использование:
```c
// Linux пример
struct timespec ts;
clock_gettime(CLOCK_REALTIME, &ts);
// ts содержит календарное время
```

```java
// Java
long now = System.currentTimeMillis(); // Календарное время
Date date = new Date(); // Тоже календарное
Instant instant = Instant.now(); // В Java 8+
```

### Применение:
- **Логирование**: временные метки событий
- **Расписания**: cron задачи, напоминания
- **Финансовые операции**: время транзакций
- **Юридические документы**: дата и время подписания

## **Сравнительная таблица**

| Аспект | Монотонное время | Календарное время |
|--------|-----------------|-------------------|
| **Направление** | Только вперед | Может идти назад/вперед |
| **Корректировки** | Невозможны | NTP, летнее время, ручная настройка |
| **Начало отсчета** | Загрузка системы/процессора | Эпоха (обычно 1970-01-01) |
| **Точность** | Очень высокая (наносекунды) | Зависит от синхронизации |
| **Использование** | Интервалы, измерения | Даты, логи, события |
| **Пример в Linux** | `CLOCK_MONOTONIC` | `CLOCK_REALTIME` |
| **Пример в Java** | `System.nanoTime()` | `System.currentTimeMillis()` |

## **Проблемы и решения**

### Проблема 1: Расхождение времен
```java
// Неправильно измерять интервалы календарным временем
long start = System.currentTimeMillis();
// ... код
long end = System.currentTimeMillis();
// Если системное время изменилось, результат неверный
```

**Решение:**
```java
long start = System.nanoTime(); // Монотонное
// ... код
long elapsedNanos = System.nanoTime() - start;
```

### Проблема 2: Синхронизация в распределенных системах
```java
// На разных серверах время может расходиться
long timestamp = System.currentTimeMillis();
// timestamp на сервере A != timestamp на сервере B
```

**Решение:**
- Использовать NTP для синхронизации
- Использовать логические часы (Lamport timestamps)
- Использовать централизованный источник времени

## **Реальные кейсы использования**

### 1. **Базы данных: MVCC (Multi-Version Concurrency Control)**
```sql
-- PostgreSQL использует монотонный XID для версий транзакций
SELECT xmin, xmax, * FROM table;
```

### 2. **Распределенные системы: логическое время**
```go
// Использование векторных часов вместо календарных
type VectorClock map[string]int64
```

### 3. **Анализ производительности**
```java
// Правильное измерение времени выполнения
long startTime = System.nanoTime();
performOperation();
long duration = System.nanoTime() - startTime;
```

### 4. **Таймауты в сетевых протоколах**
```c
// Использование монотонного времени для таймаутов
struct timespec deadline;
clock_gettime(CLOCK_MONOTONIC, &deadline);
deadline.tv_sec += 5; // Таймаут 5 секунд
```

## **Linux/Unix системные вызовы**

### Получение времени:
```c
#include <time.h>

struct timespec mono_time, real_time;

// Монотонное время (для измерений)
clock_gettime(CLOCK_MONOTONIC, &mono_time);

// Календарное время (для дат)
clock_gettime(CLOCK_REALTIME, &real_time);

// Raw монотонное (без NTP корректировок)
clock_gettime(CLOCK_MONOTONIC_RAW, &mono_time);
```

## **Java API**

### Правильное использование:
```java
import java.time.Instant;
import java.time.Duration;

// Для измерений интервалов - монотонное
long start = System.nanoTime();
// операция
Duration elapsed = Duration.ofNanos(System.nanoTime() - start);

// Для временных меток - календарное
Instant eventTime = Instant.now(); // Синхронизировано с NTP
```

### Частые ошибки:
```java
// ❌ НЕПРАВИЛЬНО - смешиваем типы времени
long startNano = System.nanoTime();
// ... код
long currentMillis = System.currentTimeMillis();
long elapsed = currentMillis - (startNano / 1_000_000); // Опасное преобразование!

// ✅ ПРАВИЛЬНО - используем один тип
long startNano = System.nanoTime();
// ... код
long elapsedNanos = System.nanoTime() - startNano;
```

## **Итог**

1. **Монотонное время** — для измерений, где важна **относительность** и **стабильность**
2. **Календарное время** — для событий, где важна **абсолютность** и **соответствие реальному миру**

**Золотое правило:**  
- **Измеряете интервалы?** → Используйте монотонное время  
- **Фиксируете события?** → Используйте календарное время  
- **Работаете в распределенной системе?** → Будьте осторожны с обоими!
