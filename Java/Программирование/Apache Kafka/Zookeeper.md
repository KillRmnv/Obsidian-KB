- Исторически Kafka зависела от **Apache ZooKeeper** для хранения метаданных кластера (списка брокеров, топиков, ACL) и выбора контроллера.
    
- Начиная с версии **Kafka 3.x** и окончательно в **Kafka 4.0**, ZooKeeper был полностью устранен (**KRaft mode**). Теперь Kafka использует встроенный механизм консенсуса на основе Raft для управления метаданными, что упрощает архитектуру и повышает стабильность.
