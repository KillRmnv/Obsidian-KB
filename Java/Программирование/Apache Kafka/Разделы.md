- Каждый топик делится на **разделы (partitions)** для горизонтального масштабирования и параллельной обработки.
    
- Разделы распределяются по серверам кластера Kafka (**брокерам**)(увеличивает пропускную способность системы еще больше)
    
- **События с одним и тем же ключом** всегда попадают в один и тот же раздел, что гарантирует порядок их обработки в рамках этого раздела.
    
- Каждый раздел — это упорядоченный, неизменяемый журнал событий.


Увеличивает пропускную способность

Но нарушает последовательность, если в разные разделы попали

Для попадания в один раздел использую специальный ключ


При распределении разделов по разным серверам происходит копирование этих разделов(они станут резервными), поэтому при выходе из строя одного сервера ответственного за обработку раздела топика данные не теряются(резервные становятся основными).

ACK отправляется в случае обновления всех разделов, в т.ч. и копий
