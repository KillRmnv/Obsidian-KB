- Каждый раздел реплицируется на несколько брокеров для отказоустойчивости.
    
- Одна реплика — **лидер (leader)**, она обрабатывает все запросы на чтение и запись. Остальные — **последователи (followers)**, они асинхронно копируют данные с лидера.
    
- Если брокер с лидером выходит из строя, одна из реплик-последователей автоматически становится новым лидером.
