Промежуточное звено между отправителем(продюсер) и получателем(консьюмер) с хранением на случай ошибки.
Гарантирует доставку сообщения.Нет необходимости на продюсере думать о возможных ошибках при отправлении по сети.
Один продюсер -- много потребителей.

один сервер Kafka (нода)

Отправляет ACK о получении записи отправителю.Если ACK не пришел отправитель обязан еще раз отправить запись.Но в ситуации, когда запись пришла на брокер,но ACK не дошел могут возникнуть дубликаты, ибо все записи иммутабельны в kafka.На получателе нужно фильтровать дубликаты.
