- Механизм для горизонтального масштабирования обработки.
    
- Все потребители в одной группе совместно обрабатывают данные одного топика: **каждому потребителю в группе назначается уникальный набор разделов**.
    
- Если потребителей в группе больше, чем разделов, часть потребителей будет без работы.
    
- Группа хранит свой общий оффсет для каждого раздела в специальном служебном топике `__consumer_offsets`.
