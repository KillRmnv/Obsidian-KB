

## 📋 Расшифровка бенчмарков кодинга

### 1. **HumanEval**
| Параметр | Описание |
|----------|----------|
| **Что тестирует** | Способность модели писать функции Python по docstring |
| **Формат** | 164 задачи с docstring → модель генерирует тело функции |
| **Метрика** | `pass@1` — % решений, прошедших юнит-тесты с первой попытки |
| **Пример задачи** | "Напиши функцию, которая возвращает n-е число Фибоначчи" |
| **Особенности** | Только Python, простые алгоритмы, проверяет базовый синтаксис и логику |

**Что проверяет:** Умение понимать описание на естественном языке и превращать его в рабочий код.

---

### 2. **MBPP (Mostly Basic Python Problems)**
| Параметр | Описание |
|----------|----------|
| **Что тестирует** | Решение практических задач Python средней сложности |
| **Формат** | 974 задачи с текстовым описанием + тесты |
| **Метрика** | `pass@1` — прохождение скрытых тестов |
| **Пример задачи** | "Напиши функцию для сортировки списка словарей по ключу" |
| **Особенности** | Больше задач, чем HumanEval, более разнообразные сценарии |

**Что проверяет:** Практическое программирование — работа со структурами данных, строками, файлами.

---

### 3. **LiveCodeBench**
| Параметр | Описание |
|----------|----------|
| **Что тестирует** | Решение конкурсных задач по программированию |
| **Формат** | Задачи с LeetCode/Codeforces — сложные алгоритмы |
| **Метрика** | `pass@1` — прохождение всех тестовых случаев |
| **Пример задачи** | Динамическое программирование, графы, оптимизация |
| **Особенности** | Самый сложный бенчмарк, требует глубокого понимания алгоритмов |

**Что проверяет:** "Инженерное мышление" — способность решать нетривиальные задачи, требующие планирования.

---

### 4. **BigCodeBench**
| Параметр | Описание |
|----------|----------|
| **Что тестирует** | Работа с реальными библиотеками и API |
| **Формат** | 1,140 задач с использованием 139 Python-библиотек |
| **Метрика** | `pass@1` — выполнение кода с реальными зависимостями |
| **Пример задачи** | "Используя pandas, проанализируй CSV и построй график" |
| **Особенности** | Проверяет знание популярных библиотек (NumPy, Pandas, Requests и т.д.) |

**Что проверяет:** Практическую применимость — умение использовать сторонние библиотеки как реальный разработчик.

---

### 5. **MultiPL-E**
| Параметр | Описание |
|----------|----------|
| **Что тестирует** | Мультиязычность — одинаковые задачи на разных языках |
| **Формат** | Перевод HumanEval на 18+ языков программирования |
| **Метрика** | Средний `pass@1` по всем языкам |
| **Языки** | Python, Java, C++, JavaScript, Go, Rust, Ruby и др. |

**Что проверяет:** Способность модели работать не только с Python, но и с другими языками — критично для Java-разработки.

---

### 6. **CRUXEval (Code Reasoning, Understanding, and eXecution Evaluation)**
| Параметр | Описание |
|----------|----------|
| **Что тестирует** | Понимание кода и предсказание его поведения |
| **Формат** | Два направления: input→output и output→input |
| **Метрика** | Точность предсказания результата выполнения |
| **Пример задачи** | "Дан код функции и её вывод, какой был вход?" |

**Что проверяет:** Не генерацию, а **понимание** кода — важно для ревью и отладки.

---

### 7. **Aider (Code Editing Benchmark)**
| Параметр | Описание |
|----------|----------|
| **Что тестирует** | Редактирование существующего кода |
| **Формат** | Дано: код + запрос на изменение → нужно внести правки |
| **Метрика** | `pass@2` — успех за 2 попытки |
| **Пример задачи** | "Добавь обработку ошибок в эту функцию" |

**Что проверяет:** Способность работать с legacy-кодом — реальный сценарий разработки.

---

### 8. **Spider / BIRD-SQL**
| Параметр | Описание |
|----------|----------|
| **Что тестирует** | Генерация SQL-запросов по текстовому описанию |
| **Формат** | Схема БД + вопрос на естественном языке → SQL |
| **Метрика** | Точность выполнения запроса |

**Что проверяет:** Работу с базами данных — полезно для backend-разработки.

---

### 9. **RepoBench (Repository-Level Benchmark)**
| Параметр | Описание |
|----------|----------|
| **Что тестирует** | Понимание контекста большого проекта |
| **Формат** | Дано: множество файлов проекта → нужно дополнить/исправить код |
| **Метрика** | Точность предсказания недостающего кода |

**Что проверяет:** Работу с реальными проектами, а не изолированными функциями.

---

### 10. **FIM (Fill-in-the-Middle)**
| Параметр | Описание |
|----------|----------|
| **Что тестирует** | Автодополнение кода в середине файла |
| **Формат** | Префикс + суффикс → модель генерирует середину |
| **Метрика** | Точность совпадения с оригинальным кодом |

**Что проверяет:** Именно то, что нужно для **автодополнения в IDE**!

---

## 🎯 Какие бенчмарки важны для твоих задач?

| Твоя задача                      | Ключевые бенчмарки         | Почему                                 |
| -------------------------------- | -------------------------- | -------------------------------------- |
| **Автодополнение в Zed/VS Code** | FIM, HumanEval             | Проверяют точность дополнения кода     |
| **Генерация Python/Java**        | HumanEval, MBPP, MultiPL-E | Базовое качество кода на нужных языках |
| **Генерация LaTeX**              | Нет специальных тестов     | Смотри общие языковые способности      |
| **Простые скрипты**              | MBPP, BigCodeBench         | Практические задачи                    |
| **Сложные алгоритмы**            | LiveCodeBench              | Конкурсное программирование            |
| **Работа с проектом**            | RepoBench, Aider           | Контекст большого codebase             |
