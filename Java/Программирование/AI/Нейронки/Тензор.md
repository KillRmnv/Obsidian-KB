
### Что это:
> Тензор — это многомерный массив чисел, который:
> - Может работать на GPU
> - Запоминает, *как он был получен*, чтобы потом посчитать градиенты

### Иерархия:
```
Скаляр (0D) → 5
Вектор (1D) → [1, 2, 3]
Матрица (2D) → [[1,2], [3,4]]
Тензор (3D+) → [[[1,2],[3,4]], [[5,6],[7,8]]]
```
# 🧊 ТЕНЗОРЫ: Что внутри этих чисел?

Тензор — это просто **контейнер для чисел**. Но смысл чисел внутри меняется в зависимости от того, **на каком этапе** работы сети находится этот тензор.

Вот конкретика для **вашего датасета погоды**:

---

## 1️⃣ Тензор ВХОДНЫХ ДАННЫХ (X)
**Где:** В самом начале, когда вы загружаете датасет.  
**Что внутри:** Реальные измерения погоды.

```python
# Пример для одного дня (одна строка таблицы)
tensor([15.6,  8.3,  3.0,  0.0,  1.0])
  │      │      │      │      └─ is_clear (1=да, 0=нет)
  │      │      │      └─ precipitation (мм)
  │      │      └─ wind (м/с)
  │      └─ temp_min (°C)
  └─ temp_max (°C)
```

> **Ответ на вопрос:** Это числа **измерений**. Градусы, метры в секунду, миллиметры осадков.  
> **Тип:** `torch.FloatTensor` (дробные числа).

---

## 2️⃣ Тензор ВЕСОВ МОДЕЛИ (Weights)
**Где:** Внутри нейросети (`model.parameters()`).  
**Что внутри:** «Сила связей» между нейронами. Это **знания** сети.

```python
# Пример весов для одного нейрона скрытого слоя
tensor([ 0.85, -0.42,  0.10,  0.05, -0.90])
  │       │       │       │       └─ Насколько важен is_clear для этого нейрона
  │       │       │       └─ Насколько важен precipitation
  │       │       └─ Насколько важен wind
  │       └─ Насколько важен temp_min
  └─ Насколько важен temp_max
```

> **Ответ на вопрос:** Это числа **важности**.  
> - `0.85` → «Температура максимум сильно влияет на решение этого нейрона».  
> - `-0.90` → «Если ясно (is_clear=1), этот нейрон сильно тормозит (возможно, это нейрон для класса 'дождь')».  
> **В начале обучения:** Эти числа случайные (сеть ничего не знает).  
> **В конце обучения:** Эти числа настроены так, чтобы правильно предсказывать погоду.  
> **Тип:** `torch.FloatTensor`.

---

## 3️⃣ Тензор ЦЕЛЕВЫХ МЕТОК (y)
**Где:** В датасете, правильные ответы.  
**Что внутри:** Индексы классов погоды.

```python
# Пример для 5 дней
tensor([4, 2, 2, 3, 0])
  │      │      │      │      └─ 0 = drizzle
  │      │      │      └─ 3 = snow
  │      │      └─ 2 = rain
  │      └─ 2 = rain
  └─ 4 = sun
```

> **Ответ на вопрос:** Это **номера категорий**. Не текст «sun», а число `4`.  
> **Тип:** `torch.LongTensor` (целые числа).  
> **Важно:** Методичка требует именно целые числа для `CrossEntropyLoss`.

---

## 4️⃣ Тензор ВЫХОДА СЕТИ (Logits)
**Где:** После прямого прохода (`output = model(x)`).  
**Что внутри:** «Оценки уверенности» для каждого класса.

```python
# Сеть посмотрела на вход и выдала:
tensor([ 0.2, -1.5,  1.8, -2.0,  3.5])
  │       │       │       │       └─ Оценка для класса 4 (sun) →Highest!
  │       │       │       └─ Оценка для класса 3 (snow)
  │       │       └─ Оценка для класса 2 (rain)
  │       └─ Оценка для класса 1 (fog)
  └─ Оценка для класса 0 (drizzle)
```

> **Ответ на вопрос:** Это **сырые баллы**.  
> - Чем больше число, тем выше вероятность этого класса.  
> - Здесь `3.5` (sun) больше всех → сеть думает, что будет солнце.  
> - Это ещё не вероятности (сумма не равна 1), это «логиты».  
> **Тип:** `torch.FloatTensor`.

---

## 5️⃣ Тензор ГРАДИЕНТОВ (Gradients)
**Где:** Появляется после `loss.backward()`.  
**Что внутри:** «Советы» по изменению весов.

```python
# Градиент для веса temp_max (был 0.85)
tensor([ +0.04 ])
  └─ Если увеличить вес на 0.01, ошибка УВЕЛИЧИТСЯ.
     Значит, оптимизатор чуть УМЕНЬШИТ этот вес.
```

> **Ответ на вопрос:** Это **направление коррекции**.  
> - Положительное число → вес слишком большой, надо уменьшить.  
> - Отрицательное число → вес слишком маленький, надо увеличить.  
> **Тип:** `torch.FloatTensor` (хранится в атрибуте `.grad` у тензора весов).

---

## ❓ Почему именно ТЕНЗОР, а не просто список?

Вы можете спросить: *«Ну и зачем мне тензор, если это просто массив чисел? Почему не list или numpy array?»*

**Две причины (из методички):**

| Обычный массив (numpy/list) | Тензор (PyTorch) |
|-----------------------------|------------------|
| Хранит числа. | Хранит числа + **историю операций**. |
| Работает только на CPU. | Может работать на **GPU** (видеокарта), что в 10-100 раз быстрее. |
| Не знает про градиенты. | Знает, **как** было получено число, чтобы автоматически посчитать градиент (`.backward()`). |

> **Главное:** Тензор — это единственная валюта, которую понимает PyTorch. Вы не можете передать в модель `numpy array` или `list`. Только тензор.

## 📍 Где это в вашей лабораторной (Чек-лист)

Согласно методичке (Раздел 1, шаг 1.4):

1.  **Признаки (X):**
    ```python
    X_tensor = torch.FloatTensor(X_numpy)  # Числа измерений
    ```
2.  **Метки (y):**
    ```python
    y_tensor = torch.LongTensor(y_numpy)   # Номера классов (0, 1, 2...)
    ```

Если вы перепутаете типы (например, сделаете метки `FloatTensor`), `CrossEntropyLoss` выдаст ошибку.

## ✅ ИТОГ

| Тензор | Числа внутри означают... | Аналогия из жизни |
|--------|--------------------------|-------------------|
| **Вход (X)** | Измерения погоды (°C, м/с) | Показания приборов |
| **Веса (W)** | Важность связей в мозге сети | Синапсы в мозге (сила связи) |
| **Метки (y)** | Номер правильного ответа | Ключ к тесту (1-А, 2-Б...) |
| **Выход** | Уверенность сети в каждом классе | Черновик ответов перед проверкой |
| **Градиент** | Инструкция по исправлению ошибок | Работа над ошибками (красная ручка) |

Тензор — это просто **коробка**, которая хранит эти числа и позволяет сети их обрабатывать и учиться.
