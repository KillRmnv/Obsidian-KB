Планировщик (scheduler) управляет **learning rate (LR)** — шагом, с которым оптимизатор обновляет веса модели.

## Простая аналогия

Представь, что ты спускаешься с горы (ищешь минимум функции потерь):


| Ситуация       | Что делаешь    | Аналогия в обучении                  |
| :------------- | :------------- | :----------------------------------- |
| Далеко от цели | Большие шаги   | **Высокий LR** — быстро приближаемся |
| Близко к цели  | Маленькие шаги | **Низкий LR** — точная настройка     |
| Ходишь кругами | Уменьшаешь шаг | **Уменьшаем LR** — выходим из плато  |
