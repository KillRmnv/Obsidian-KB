
##  Что такое `transform`?

Это **цепочка преобразований** изображений. PyTorch не умеет работать с JPG/PNG напрямую — нужно перевести в тензоры.

### Разбор `transform_train` (для обучения):

| Трансформация | Что делает | Зачем |
|-------------|-----------|-------|
| `RandomResizedCrop(64, scale=(0.8, 1.0))` | Случайно вырезает кусок 64×64 (от 80% до 100% исходного) | **Аугментация**: модель видит разные масштабы и позиции объекта |
| `RandomHorizontalFlip()` | Случайное горизонтальное отражение (50% вероятность) | **Аугментация**: фрукты симметричны, увеличиваем данные в 2 раза |
| `RandomRotation(15)` | Поворот на ±15 градусов | **Аугментация**: разные углы съёмки |
| `ColorJitter(...)` | Случайное изменение яркости, контраста, насыщенности, оттенка | **Аугментация**: разное освещение |
| `ToTensor()` | PIL Image → torch tensor [0, 1] | Нормализация для PyTorch |
| `Normalize(mean, std)` | `(x - mean) / std` для каждого канала | Приведение к стандартному распределению (предобученные сети ожидают это) |

**Mean и std**: `(0.485, 0.456, 0.406)` — это средние значения по ImageNet. Почти все предобученные модели (ResNet, VGG и т.д.) ожидают именно такую нормализацию.

### Разбор `transform_test` (для валидации и теста):

```python
transform_test = transforms.Compose([
    transforms.Resize((64, 64)),      # Просто ресайз, без случайных кропов
    transforms.ToTensor(),
    transforms.Normalize(...)             # Та же нормализация
])
```

**Почему нет аугментаций?** На val/test нужны **детерминированные** результаты. Если каждый раз случайно крутить тестовые картинки — метрики будут прыгать.
