## Разбор каждого компонента

### 1. Свёрточные слои (`nn.Conv2d`)

```python
nn.Conv2d(in_channels=3, out_channels=32, kernel_size=3, padding=1)
```

| Параметр | Значение | Смысл |
|---------|---------|-------|
| `in_channels=3` | RGB изображение | 3 канала (красный, зелёный, синий) |
| `out_channels=32` | 32 фильтра | Будет 32 feature maps на выходе |
| `kernel_size=3` | Фильтр 3×3 | Стандартный размер |
| `padding=1` | Дополнение 1 пиксель | Сохраняем размер: 64×64 → 64×64 (без padding было бы 62×62) |

**Почему `padding=1`?** Для фильтра 3×3: выход = вход - 3 + 1 = вход - 2. С padding +1 с каждой стороны: выход = вход + 2 - 3 + 1 = вход. Размер сохраняется!

### 2. Batch Normalization (`nn.BatchNorm2d`)

```python
nn.BatchNorm2d(32)  # после conv1 с 32 каналами
```

**Что делает:** Нормализует каждый канал по батчу:
```
x_norm = (x - mean_batch) / sqrt(var_batch + eps)
x_out = γ * x_norm + β  # масштабируем и сдвигаем (обучаемые γ, β)
```

**Зачем:**
- Ускоряет сходимость (можно ставить больший learning rate)
- Стабилизирует распределения (решает проблему «внутреннего ковариантного сдвига»)
- Лёгкая регуляризация (шум из статистик батча)

### 3. ReLU — уже знаешь, активация `max(0, x)`

### 4. Pooling (`nn.MaxPool2d` / `nn.AvgPool2d`)

```python
nn.MaxPool2d(kernel_size=2, stride=2)   # conv1, conv2
nn.AvgPool2d(kernel_size=2, stride=2)  # conv3 (последний)
```

| Тип | Что берёт | Эффект |
|-----|----------|--------|
| **MaxPool** | Максимум из 2×2 | Сохраняет самые «активные» признаки, инвариантность к сдвигам |
| **AvgPool** | Среднее из 2×2 | Более «мягкое» сглаживание, часто используется в конце |

**Почему AvgPool в конце?** Мода — перед полносвязными слоями использовать average pooling вместо max (Global Average Pooling), но здесь просто обычный AvgPool2d.

### 5. Dropout — регуляризация

```python
nn.Dropout2d(p=0.25)   # для свёрточных слоёв (каналы)
nn.Dropout(p=0.5)      # для полносвязных (нейроны)
```

| Тип | Где | Что выключает |
|-----|-----|--------------|
| `Dropout2d` | Свёрточные слои | **Целые каналы** feature map (с вероятностью 0.25) |
| `Dropout` | Полносвязные | Отдельные нейроны (с вероятностью 0.5) |

**Dropout2d:** Во время обучения случайно обнуляет целые каналы (все пиксели в канале). Это заставляет сеть не полагаться на отдельные каналы.

**Важно:** В `forward` вызывается автоматически. При `model.eval()` dropout отключается!

### 6. Flatten — «выпрямление»

```python
nn.Flatten()  # 32×128×8×8 → 32×8192
```

Просто меняет форму тензора без изменения данных. 128 каналов по 8×8 = 8192 числа.

### 7. Полносвязные слои (`nn.Linear`)

```python
nn.Linear(in_features=128 * 8 * 8, out_features=256)  # 8192 → 256
nn.Linear(in_features=256, out_features=num_classes)    # 256 → 50
```

Обычные полносвязные слои, как в MLP. BatchNorm1d и Dropout между ними.

## Почему именно такая архитектура?

| Паттерн | Обоснование |
|---------|------------|
| **Conv → BN → ReLU → Pool** | Стандартный блок (сначала VGG, потом повсеместно) |
| **Увеличение каналов** (32→64→128) | При уменьшении пространства увеличиваем глубину — сохраняем «ёмкость» |
| **Dropout2d в свёртках, Dropout в FC** | Разные типы регуляризации для разных слоёв |
| **AvgPool в конце** | Мягче, чем MaxPool, перед финальными слоями |
| **BatchNorm везде** | Стабильность и скорость |

## Вопросы для проверки:

1. Почему `padding=1`? — Чтобы размер не уменьшался после свёртки (только после пулинга).
2. Почему `Dropout2d`, а не обычный `Dropout` в свёртках? — Чтобы выключать целые каналы, а не случайные пиксели (пространственная корреляция важна).
3. Почему перед `fc1` именно `128 * 8 * 8`? — После 3 пулингов 64→32→16→8, каналов 128.
