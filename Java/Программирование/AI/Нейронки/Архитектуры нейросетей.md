# 🏗️ ВИДЫ АРХИТЕКТУР НЕЙРОННЫХ СЕТЕЙ

Отличный вопрос! Разные задачи требуют разных архитектур. Вот основные типы — от простого к сложному.

---

## 🔹 1. Полносвязная сеть (Fully Connected / Dense / MLP)

**Что это:** Каждый нейрон слоя N соединён с каждым нейроном слоя N+1.

```
Вход → [Linear + ReLU] × N → Выход
```

**Для чего:** 
- ✅ Табличные данные (ваш датасет погоды)
- ✅ Простые задачи классификации/регрессии
- ✅ Небольшое количество признаков (< 1000)

**Плюсы:**
- Простая в реализации
- Хорошо работает на структурированных данных
- Быстро обучается

**Минусы:**
- Не учитывает пространственные/временные зависимости
- Много параметров → склонна к переобучению
- Не масштабируется на изображения/текст

> 📌 **Ваша лабораторная:** Именно эту архитектуру вы реализуете (Раздел 2 методички).

---

## 🔹 2. Сверточная нейросеть (CNN / Convolutional Neural Network)

**Что это:** Использует **свертки** — фильтры, которые «скользят» по данным и ищут локальные паттерны.

```
Вход (изображение) → Conv + ReLU → Pooling → ... → FC → Выход
```

**Для чего:**
- ✅ Изображения (распознавание объектов, сегментация)
- ✅ Аудио (спектрограммы)
- ✅ Любые данные с **пространственной структурой**

**Ключевые идеи:**
| Компонент | Зачем нужен |
|-----------|-------------|
| **Conv2d** | Ищет локальные паттерны (края, текстуры) |
| **Pooling** | Уменьшает размер, сохраняет важное |
| **Stride/Padding** | Контролирует «шаг» и обработку краёв |

**Почему не в вашей лабе:** 
> Методичка (Раздел 1): *«Использование изображений, текстов и других неструктурированных данных в данной лабораторной работе не допускается»*

---

## 🔹 3. Рекуррентная нейросеть (RNN / LSTM / GRU)

**Что это:** Сеть с **памятью** — выход на шаге t зависит от входа на шаге t **и** от состояния на шаге t-1.

```
Вход: [x₁, x₂, x₃, ..., xₜ]  ← последовательность
        ↓    ↓    ↓         ↓
      [h₁ → h₂ → h₃ → ... → hₜ]  ← скрытые состояния (память)
                                  ↓
                                Выход
```

**Для чего:**
- ✅ Временные ряды (прогноз погоды, биржи)
- ✅ Текст (посимвольная генерация)
- ✅ Аудио, видео (последовательности кадров)

**Проблема простых RNN:** Затухание градиентов → не помнят длинные зависимости.

**Решения:**
| Архитектура | Как решает проблему |
|-------------|---------------------|
| **LSTM** | «Ворота» (gates) контролируют, что запомнить/забыть |
| **GRU** | Упрощённая версия LSTM, быстрее обучается |

---

## 🔹 4. Трансформер (Transformer / Attention)

**Что это:** Архитектура на основе **механизма внимания** (Attention) — сеть «смотрит» на все части входа одновременно и решает, что важно.

```
Вход → Embedding + Positional Encoding → [Attention + FFN] × N → Выход
```

**Ключевая идея — Self-Attention:**
```
Для каждого слова вычисляем:
• Query: «Что я ищу?»
• Key: «Что я могу предложить?»
• Value: «Какая во мне информация?»

Attention(Q,K,V) = softmax(Q·Kᵀ/√d) · V
```

**Для чего:**
- ✅ Текст (перевод, генерация, классификация)
- ✅ LLM (GPT, LLaMA, BERT)
- ✅ Мультимодальные задачи (текст + изображение)

**Почему доминирует:**
- Параллелизация (в отличие от RNN)
- Улавливает дальние зависимости в тексте
- Масштабируется до триллионов параметров

> 📌 **LLM**, о которых вы спрашивали — это трансформеры.

---

## 🔹 5. Автоэнкодер (Autoencoder)

**Что это:** Сеть, которая учится сжимать данные (энкодер) и восстанавливать их (декодер).

```
Вход → Энкодер → Bottleneck (сжатое представление) → Декодер → Реконструкция
```

**Для чего:**
- ✅ Снижение размерности (альтернатива PCA)
- ✅ Аномалии (плохо восстанавливает «непохожие» данные)
- ✅ Генерация (Variational Autoencoder)

**Особенность:** Обучается **без меток** (unsupervised) — минимизирует ошибку реконструкции.

---

## 🔹 6. Генеративно-состязательная сеть (GAN)

**Что это:** Две сети соревнуются:
- **Генератор**: создаёт поддельные данные
- **Дискриминатор**: отличает реальные от поддельных

```
Генератор: шум → фальшивое изображение
Дискриминатор: изображение → «реальное» / «фейк»

Обучение: минимаксная игра
```

**Для чего:**
- ✅ Генерация изображений (лица, арты)
- ✅ Стиль-трансфер, супер-разрешение
- ✅ Аугментация данных

**Сложность:** Нестабильное обучение, требует тонкой настройки.

---

## 🔹 7. Гибридные и специализированные архитектуры

| Архитектура | Суть | Применение |
|-------------|------|------------|
| **CNN + RNN** | Свертки извлекают признаки, RNN обрабатывает последовательность | Видео, подписи к изображениям |
| **Graph Neural Network (GNN)** | Работает с графами (узлы + рёбра) | Соцсети, молекулы, рекомендации |
| **Siamese Network** | Две одинаковые сети сравнивают пары входов | Распознавание лиц, верификация |
| **U-Net** | Encoder-Decoder с skip-connections | Сегментация медицинских изображений |

---

## 📊 СВОДНАЯ ТАБЛИЦА: Какую архитектуру выбрать?

| Тип данных | Задача | Рекомендуемая архитектура |
|------------|--------|---------------------------|
| **Таблица** (числа, категории) | Классификация / Регрессия | ✅ Fully Connected (ваша лаба) |
| **Изображение** | Классификация / Детекция | ✅ CNN (ResNet, EfficientNet) |
| **Текст** | Классификация / Генерация | ✅ Transformer (BERT, GPT) |
| **Временной ряд** | Прогноз / Классификация | ✅ LSTM / Transformer |
| **Граф** | Классификация узлов /_link prediction_ | ✅ GNN (GCN, GAT) |
| **Нет меток** | Сжатие / Аномалии | ✅ Autoencoder |
| **Генерация** | Создание новых данных | ✅ GAN / Diffusion / VAE |

## 🔍 ПОЧЕМУ В ВАШЕЙ ЛАБЕ ТОЛЬКО FULLY CONNECTED?

Согласно **Разделу 1 методички**:

> *«Использование изображений, текстов и других неструктурированных данных в данной лабораторной работе не допускается»*

**Причины:**
1.  **Фокус на базе:** Вы учитесь понимать тензоры, градиенты, цикл обучения — это одинаково для всех архитектур.
2.  **Простота отладки:** Табличные данные легче визуализировать и анализировать.
3.  **Сравнение с классикой:** Методы из прошлого семестра (логистическая регрессия, деревья) тоже работают на таблицах → честное сравнение.

> 🎯 **Вывод:** Fully Connected — это «азбука» глубокого обучения. Выучив её, вы сможете понять любую другую архитектуру.
