## 🔹 НЕЙРОН — атом нейросети

### Что это математически:
```
выход = активация( w₁·x₁ + w₂·x₂ + ... + wₙ·xₙ + b )
```

### Что это интуитивно:
> Нейрон — это «взвешенный голос». Он получает несколько входных сигналов, каждый умножает на свой «вес» (важность), складывает, добавляет смещение (bias) и пропускает через фильтр (функцию активации).

### Компоненты нейрона:
| Компонент | Что делает | Аналогия |
|-----------|-----------|----------|
| **Входы (x)** | Признаки данных | Вопросы, которые задают нейрону |
| **Веса (w)** | Важность каждого входа | Насколько нейрон «доверяет» каждому вопросу |
| **Bias (b)** | Сдвиг порога срабатывания | Насколько нейрон «склонен согласиться» по умолчанию |
| **Сумматор** | Σ(w·x) + b | Взвешенное голосование |
| **Активация** | Нелинейное преобразование | Фильтр: «решить» или «проигнорировать» сигнал |

> 🎯 **Один нейрон = линейный классификатор** (как логистическая регрессия). Сила появляется, когда нейронов много и они соединены в слои.

## 🔹 СЛОЙ — группа нейронов

![[Программирование/AI/Нейронки/Слой]]]

### Типы слоёв в полносвязной сети:
| Слой | Размер | Зачем нужен |
|------|--------|-------------|
| **Входной** | = числу признаков | Принимает данные, не вычисляет — просто передаёт |
| **Скрытый** | Выбираете вы (32, 64, 128...) | Извлекает паттерны, учится представлениям данных |
| **Выходной** | = числу классов | Выдаёт «сырые оценки» (логиты) для каждого класса |

### Почему «полносвязный» (Fully Connected)?
> Каждый нейрон слоя N соединён с **каждым** нейроном слоя N+1. Нет пропущенных связей — все входы влияют на все выходы.

```
Вход: [x₁, x₂, x₃]  →  Скрытый: [h₁, h₂, h₃, h₄]  →  Выход: [y₁, y₂]
         │                   │  │  │  │                  │   │
         └─────все связи─────┴──┴──┴──┴──────────────────┴───┘
```

> ⚠️ **Важно:** Слой без функции активации = просто линейное преобразование. Два таких слоя подряд = один слой. **Нелинейность — обязательна.**

## 🔹 ГРАДИЕНТ — направление улучшения

### Что это интуитивно:
> Градиент — это «стрелка», которая показывает: *если я чуть-чуть изменю этот вес, в какую сторону изменится ошибка?*

### Математически:
```
∂Loss/∂w = насколько изменится Loss, если увеличить w на 0.001
```

### Зачем нужен:
- Оптимизатор использует градиенты, чтобы понять: **какой вес уменьшить, какой увеличить**, чтобы ошибка стала меньше
- Без градиентов обучение = случайный подбор = не работает

> 🔁 **Обратное распространение (backpropagation)** — алгоритм, который эффективно считает градиенты для всех весов сразу, используя цепное правило.

## 🔹 ФУНКЦИЯ ПОТЕРЬ (LOSS) — «оценка ошибки»

### Что это:
> Число, которое показывает: *насколько предсказания модели отличаются от правильных ответов*.

### Для вашей задачи (многоклассовая классификация):
**CrossEntropyLoss** — стандарт

```
Чем больше модель «уверена» в неверном классе → тем больше штраф
```

### Почему не MSE (среднеквадратичная ошибка)?
| MSE | CrossEntropy |
|-----|--------------|
| Для регрессии (предсказать число) | Для классификации (предсказать класс) |
| Штрафует за расстояние | Штрафует за «уверенность не в том классе» |
| Не учитывает вероятности | Работает с распределением вероятностей |

> ⚠️ **CrossEntropyLoss ожидает:**
> - На вход: сырые логиты (не softmax!)
> - Целевые метки: целые числа 0, 1, 2... (не one-hot векторы)

## 🔹 ОПТИМИЗАТОР — «инженер», который настраивает веса

### Что он делает:
> Получает градиенты → решает, *насколько* и *в какую сторону* изменить каждый вес → обновляет модель.

### Основные алгоритмы:

#### SGD (Stochastic Gradient Descent)
```
w_new = w_old - lr * gradient
```
- Простой, понятный
- ❌ Может «скакать», медленно сходится

#### Adam (Adaptive Moment Estimation) — стандарт
- Для каждого веса свой шаг обучения
- Учитывает не только текущий градиент, но и историю
- ✅ Работает «из коробки», быстро сходится

### Learning Rate (скорость обучения) — главный гиперпараметр
| Слишком большой | Слишком маленький |
|-----------------|-------------------|
| Loss «скачет», не сходится | Обучение идёт вечность |
| Модель «перепрыгивает» минимум | Может застрять в локальном минимуме |

> 🎯 Начните с `lr=0.001` для Adam. Если loss нестабилен — уменьшите в 10 раз.

## 🔹 ПРЯМОЙ И ОБРАТНЫЙ ПРОХОД — цикл обучения

### Прямой проход (Forward):
```
Данные → Сеть → Предсказание → Сравнение с истиной → Loss
```
- Сеть «думает» и выдаёт ответ
- Считается ошибка

### Обратный проход (Backward):
```
Loss → Вычисление градиентов → Обновление весов
```
- Градиенты распространяются назад по сети
- Оптимизатор обновляет веса

> 🔁 Эти два шага повторяются на каждой итерации. Одна эпоха = один полный проход по всем данным.

## 🔹 ПЕРЕОБУЧЕНИЕ (OVERFITTING) — когда модель «зазубрила»

### Что это:
> Модель идеально работает на обучающих данных, но плохо — на новых.

### Как увидеть:
```
Эпоха 10:  Train Acc=85%,  Test Acc=83%  ✅
Эпоха 30:  Train Acc=98%,  Test Acc=79%  ❌ переобучение!
```

### Почему происходит:
- Слишком сложная модель (много параметров)
- Мало данных
- Слишком долго обучали

### Как бороться:
| Метод | Как работает |
|-------|--------------|
| **Упростить архитектуру** | Меньше слоёв/нейронов = меньше свободы «запоминать» |
| **Dropout** | Случайно «выключает» нейроны во время обучения — заставляет сеть быть устойчивее |
| **Ранняя остановка** | Остановить обучение, когда test loss перестал улучшаться |
| **L2-регуляризация** | Штрафует большие веса — модель становится «проще» |

## 🔹 МЕТРИКИ — как измерить качество

| Метрика | Формула | Что показывает | Когда важна |
|---------|---------|----------------|-------------|
| **Accuracy** | Правильные / Все | Общая точность | Когда классы сбалансированы |
| **Precision** | TP / (TP+FP) | Из предсказанных «А» — сколько реально А | Когда ложные срабатывания дороги |
| **Recall** | TP / (TP+FN) | Из реальных «А» — сколько модель нашла | Когда пропуск объекта опасен |
| **F1-score** | 2·(P·R)/(P+R) | Баланс Precision и Recall | При дисбалансе классов — основная метрика |
| **Loss** | Зависит от функции | «Расстояние» до истины | Для оптимизации, не для интерпретации |
