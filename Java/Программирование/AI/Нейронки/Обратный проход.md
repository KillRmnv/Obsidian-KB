# 🔄 ОБРАТНЫЙ ПРОХОД: Как меняются веса всех нейронов

Отличный вопрос! Это **самая важная часть** обучения нейросети. Давай разберём по шагам на твоём датасете погоды.

---

## 📊 КОНТЕКСТ: Твоя сеть

```
Вход (16 признаков) → Скрытый слой (64 нейрона) → Выход (5 классов)
```

**Веса в сети:**
| Слой | Сколько весов |
|------|---------------|
| Вход → Скрытый | 16 × 64 = **1024 веса** + 64 bias |
| Скрытый → Выход | 64 × 5 = **320 весов** + 5 bias |
| **Итого** | **~1400 параметров** для обновления |

---

## 🔁 ШАГ 1: Прямой проход (запоминание пути)

Перед обратным проходом сеть делает прямой проход и **запоминает все промежуточные значения**:

```
Вход: [temp=15.6, wind=3.0, ...]  ← 16 чисел
  ↓
Скрытый слой: [7.2, 0, 3.1, ..., 4.5]  ← 64 числа (после ReLU)
  ↓
Выход: [0.2, -1.5, 1.8, -2.0, 3.5]  ← 5 логитов
  ↓
Loss = 1.66  ← ошибка (правильный класс = "sun")
```

> 🎯 **Важно:** PyTorch строит «граф вычислений» — запоминает, *как* каждое число было получено. Это нужно для обратного счёта градиентов.

---

## 🔁 ШАГ 2: Обратный проход (распространение ошибки)

### Что происходит:

```
Loss → Выходной слой → Скрытый слой → Вход
```

Ошибка распространяется **назад** по сети, и для **каждого веса** считается градиент:

```
∂Loss/∂w = насколько изменится Loss, если увеличить вес w на 0.001
```

### Пример для одного веса:

**Вес:** связь между `temp_max` (вход) и `Нейрон_1` (скрытый слой)
```
Текущее значение веса: w = 0.85

Градиент: ∂Loss/∂w = +2.3

Интерпретация:
- Градиент положительный → если увеличить w, Loss вырастет
- Значит, вес нужно УМЕНЬШИТЬ
```

### Как считаются градиенты для ВСЕХ весов сразу?

**Цепное правило (Chain Rule):**

```
Loss зависит от выхода → выход зависит от активации → активация зависит от суммы → 
сумма зависит от веса

∂Loss/∂w = ∂Loss/∂output × ∂output/∂activation × ∂activation/∂sum × ∂sum/∂w
```

PyTorch считает это **автоматически** для всех ~1400 параметров:

```python
loss.backward()  # ← одна строка, и все градиенты посчитаны!
```

---

## 🔁 ШАГ 3: Обновление весов (optimizer.step())

После `backward()` у каждого веса есть градиент. Оптимизатор (Adam) использует их для обновления:

### Формула (упрощённо для SGD):
```
w_new = w_old - learning_rate × gradient
```

### Пример для 3 разных весов:

| Вес | Текущее значение | Градиент | Обновление (lr=0.001) | Новое значение |
|-----|------------------|----------|----------------------|----------------|
| w₁ (temp_max → Нейрон_1) | 0.85 | +2.3 | 0.85 - 0.001×2.3 = **0.8477** | Уменьшили |
| w₂ (wind → Нейрон_2) | -0.42 | -1.7 | -0.42 - 0.001×(-1.7) = **-0.4183** | Увеличили |
| w₃ (precip → Нейрон_3) | 0.10 | +0.1 | 0.10 - 0.001×0.1 = **0.0999** | Почти не изменили |

> 🎯 **Ключевой момент:** Каждый вес обновляется **по-разному**, потому что градиенты разные!

---

## 🔁 ШАГ 4: Почему все нейроны меняются по-разному?

### Причина 1: Разные градиенты

Каждый нейрон участвовал в ошибке **по-своему**:

```
Нейрон_1 (выдал 7.2):
  • Сильно активировался
  • Сильно повлиял на выход
  • → Большой градиент → вес сильно изменится

Нейрон_2 (выдал 0 после ReLU):
  • «Замолчал» (отрицательная сумма)
  • Не повлиял на выход
  • → Градиент = 0 → вес не изменится!
```

### Причина 2: Разные веса в следующем слое

Даже если два нейрона выдали одинаковый выход, их влияние на ошибку разное:

```
Нейрон_A → Выход «sun» имеет вес 0.9  → сильное влияние → большой градиент
Нейрон_B → Выход «sun» имеет вес 0.1  → слабое влияние → маленький градиент
```

### Причина 3: Adam адаптирует шаг для каждого веса

Adam не просто использует градиент, а учитывает **историю**:

```
Для веса w₁:
  • Прошлые градиенты были большими → Adam уменьшит шаг
  • Чтобы вес не «скакал»

Для веса w₂:
  • Прошлые градиенты были маленькими → Adam увеличит шаг
  • Чтобы обучение шло быстрее
```

```
┌─────────────────────────────────────────────────────────────────┐
│                      ОБРАТНЫЙ ПРОХОД                            │
└─────────────────────────────────────────────────────────────────┘

LOSS = 1.66
  │
  ▼
ВЫХОДНОЙ СЛОЙ (5 нейронов)
  │  Градиенты для 320 весов + 5 bias
  │  ∂Loss/∂w_out[0] = +1.2
  │  ∂Loss/∂w_out[1] = -0.8
  │  ...
  ▼
СКРЫТЫЙ СЛОЙ (64 нейрона)
  │  Градиенты для 1024 весов + 64 bias
  │  ∂Loss/∂w_hidden[0] = +2.3
  │  ∂Loss/∂w_hidden[1] = 0.0  ← нейрон «замолчал», градиент = 0
  │  ...
  ▼
ВХОДНОЙ СЛОЙ (16 признаков)
  │  Градиенты посчитаны для всех параметров
  │  optimizer.step() обновляет все веса
  ▼
ВЕСА ОБНОВЛЕНЫ → Сеть стала чуть точнее
```

## 🔍 ЧТО ПРОИСХОДИТ С «МОЛЧАЩИМИ» НЕЙРОНАМИ (ReLU = 0)?

Если нейрон выдал 0 после ReLU:

```
Прямой проход:
  sum = -2.1 → ReLU(-2.1) = 0  ← нейрон «замолчал»

Обратный проход:
  ∂ReLU/∂sum = 0  (производная ReLU при x<0 = 0)
  → Градиент не проходит дальше
  → Веса этого нейрона НЕ обновляются
```

> ⚠️ **Проблема:** Если нейрон слишком долго «молчит», он может никогда не обучиться («мёртвый нейрон»).  
> ✅ **Решение:** Leaky ReLU пропускает малый градиент даже при x<0.

## 📍 ГДЕ ЭТО В ТВОЁМ КОДЕ

```python
# Цикл обучения (из методички, раздел 3)

for epoch in range(50):
    for batch_X, batch_y in train_loader:
        # 1. Обнулить градиенты
        optimizer.zero_grad()  # ← Сбрасываем старые градиенты
        
        # 2. Прямой проход
        output = model(batch_X)  # ← Запоминается граф вычислений
        
        # 3. Посчитать loss
        loss = criterion(output, batch_y)
        
        # 4. Обратный проход
        loss.backward()  # ← Считаем градиенты для ВСЕХ весов
        
        # 5. Обновить веса
        optimizer.step()  # ← Применяем обновления ко ВСЕМ параметрам
```

## ✅ ИТОГ: Ответы на твои вопросы

| Вопрос | Ответ |
|--------|-------|
| **Как обратный проход меняет веса?** | Для каждого веса считается градиент → оптимизатор обновляет вес в направлении уменьшения ошибки |
| **Все ли нейроны обновляются?** | Нет! Если нейрон «замолчал» (ReLU=0), его градиент = 0 → вес не меняется |
| **Почему веса меняются по-разному?** | У каждого веса свой градиент (разное влияние на ошибку) + Adam адаптирует шаг для каждого |
| **Сколько весов обновляется за 1 итерацию?** | Все обучаемые параметры (~1400 в твоей сети) |
| **Где хранятся градиенты?** | В атрибуте `.grad` у каждого тензора веса после `loss.backward()` |

---

## 🎯 ГЛАВНАЯ МЫСЛЬ

> **Обратный проход — это «работа над ошибками» для всей сети одновременно.**
>
> 1. Loss показывает, насколько ошиблись
> 2. `backward()` считает, **кто виноват** (какой вес насколько повлиял на ошибку)
> 3. `step()` **исправляет** каждый вес пропорционально его «вине»
>
> Повторяем 50+ эпох → сеть постепенно минимизирует ошибку на всех примерах.
