В Java **интерфейс** и **абстрактный класс** решают похожую задачу — задают контракт для классов-наследников, но используются в разных ситуациях. Ниже — ключевые отличия и когда что выбирать.

---

## 1. Основное назначение

### Интерфейс (`interface`)

- Описывает **что класс должен делать**
    
- Не хранит состояние объекта
    
- Используется для задания **контракта поведения**
    

### Абстрактный класс (`abstract class`)

- Описывает **что класс делает и частично как**
    
- Может содержать логику и состояние
    
- Используется как **базовый класс**
    

---

## 2. Наследование

|Характеристика|Интерфейс|Абстрактный класс|
|---|---|---|
|Количество|можно реализовать **несколько**|можно наследовать **только один**|
|Ключевое слово|`implements`|`extends`|

```java
class A implements Runnable, AutoCloseable {}
class B extends Thread {}
```

👉 Это главное архитектурное отличие.

---

## 3. Методы

### Интерфейс

- До Java 8: только `public abstract`
    
- С Java 8:
    
    - `default` — с реализацией
        
    - `static`
        
- С Java 9:
    
    - `private` (для внутренних helper-методов)
        

```java
interface Logger {
    void log(String msg);

    default void info(String msg) {
        log("INFO: " + msg);
    }
}
```

### Абстрактный класс

- Может иметь:
    
    - абстрактные методы
        
    - обычные методы
        
- Любые модификаторы доступа (`protected`, `private` и т.д.)
    

```java
abstract class Logger {
    protected abstract void log(String msg);

    protected void info(String msg) {
        log("INFO: " + msg);
    }
}
```

---

## 4. Поля (состояние)

### Интерфейс

- **Только**:
    
    ```java
    public static final
    ```
    
- Фактически — константы
    

```java
interface Config {
    int MAX_SIZE = 100;
}
```

### Абстрактный класс

- Может иметь обычные поля
    
- Может хранить состояние объекта
    

```java
abstract class Config {
    protected int maxSize;
}
```

---

## 5. Конструкторы

||Интерфейс|Абстрактный класс|
|---|---|---|
|Конструкторы|❌ нет|✅ есть|

```java
abstract class User {
    protected String name;

    public User(String name) {
        this.name = name;
    }
}
```

---

## 6. Когда использовать что

### Используй **интерфейс**, если:

- нужен **контракт** (API)
    
- класс может иметь **несколько ролей**
    
- нет необходимости хранить состояние
    
- проектируешь архитектуру / библиотеку
    
- поведение может быть реализовано по-разному
    

📌 Пример:

```java
Comparable, Runnable, Serializable
```

---

### Используй **абстрактный класс**, если:

- есть **общая логика**
    
- есть **состояние**
    
- нужна частичная реализация
    
- логически есть отношение _is-a_
    

📌 Пример:

```java
AbstractList, HttpServlet
```

---

## 7. Краткое сравнение

|Критерий|Интерфейс|Абстрактный класс|
|---|---|---|
|Множественное наследование|✅|❌|
|Хранение состояния|❌|✅|
|Реализация методов|`default`|обычная|
|Конструкторы|❌|✅|
|Назначение|контракт|базовый класс|

---

## 8. Простое правило

> **Интерфейс — “что умеет”**  
> **Абстрактный класс — “кем является”**

Если хочешь — могу привести примеры из реального проекта (например, твой веб-проект на Micronaut или state machine) или разобрать типичную ошибку выбора 👍
