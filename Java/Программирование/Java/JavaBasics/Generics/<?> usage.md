
## Где можно использовать wildcard `?`

Wildcard `?` можно ставить **в сигнатурах типов**, но **не для объявления параметра типа**:

Разрешено (использование существующего generic-типа):

- Локальные переменные
    
    java
    
    `List<?> list; List<? extends Number> nums; List<? super Integer> ints;`
    
- Поля класса
    
    java
    
    `class A {     List<? extends Human> people; }`
    
- Параметры методов и возвращаемые типы
    
    java
    
    `void process(List<? extends Human> list) { ... }` 
    `List<? super Human> create() { ... }
    `
    

Именно про это говорят: wildcard используется в **параметрах, полях, локальных переменных и return‑типах**.hexlet+2​

## Что _нельзя_ с wildcard

Нельзя так:

java

`class Person<? extends Human> { ... }   // не компилируется`

Здесь как раз требуется **объявление параметра типа**, а не wildcard. Для объявления надо дать имя параметру:

java

`class Person<T extends Human> { ... }   // правильно`

Wildcard не даёт имени типа (`?` — безымянный), поэтому внутри класса вы не сможете написать, например, поле типа `?` или метод `? get()`. Для объявления generic‑класса, интерфейса или generic‑метода всегда используются **именованные параметры типа** (`<T>`, `<K, V>` и т.п.), а не `?`.sky+1​

## Итого в одном предложении

- **Параметризировать можно классы/интерфейсы/методы** только через **именованные параметры типа**: `<T extends Human>`.
    
- **Wildcard `?`** используют **только при использовании обобщённого типа** (в полях, аргументах и возвращаемых типах методов, локальных переменных), чтобы сказать «тут какой‑то подтип/надтип, но имя мне не нужно».ya+2​
    



Да, метод может возвращать тип с wildcard, это валидно и иногда полезно:

java

`List<? extends Number> getNumbers() { ... } Optional<? extends Human> findHuman() { ... }`

Но важно понимать последствия.

## Что это значит

- `? extends Human` — «какой‑то неизвестный подтип `Human`».
    
- Вызвавший код сможет **только читать** элементы как `Human`, но не сможет безопасно добавлять туда конкретных наследников.
    

Пример:

java

`List<? extends Human> list = getHumans(); Human h = list.get(0);   // ок list.add(new Student()); // ошибка компиляции`

Компилятор не знает, `list` — это `List<Student>` или `List<Teacher>`, поэтому запрещает запись, чтобы не сломать типобезопасность.

## Когда лучше не использовать wildcard в return type

Чаще для возвращаемых типов предпочитают **параметр типа**:

java

`<T extends Human> List<T> getHumans() { ... }`

Так вызывающий код может получить:

java

`List<Student> students = service.<Student>getHumans();`

или через выведение типа, если есть аргументы/контекст. Такой метод:

- сохраняет конкретный тип (`T`) наружу,
    
- позволяет и читать, и писать `T` в коллекцию.
    

Wildcard в return-type используют, когда вызывающему не нужен точный тип, а достаточно «какой‑то `Human`/`Number`», и коллекцию не планируется модифицировать.
