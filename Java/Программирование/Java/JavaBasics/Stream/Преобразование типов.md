Отличный уточняющий вопрос! Когда у нас **объект → объект**, ситуация другая. Давайте разберем подробно.

## `Stream<T>` и преобразование объектов

Когда у нас **уже есть** `Stream<T>` (объектный стрим), используется просто **`map()`** для преобразования объектов:

```java
// Stream<T>.map(Function<T, R>) - преобразование объектов
Stream<String> strings = Stream.of("apple", "banana", "cherry");
Stream<Integer> lengths = strings.map(String::length); // String → Integer
```

---

## Полная картина преобразований

| Исходный тип | Метод | Результат | Когда использовать |
|-------------|-------|-----------|-------------------|
| **`IntStream`** | `.map(IntUnaryOperator)` | `IntStream` | `int → int` (примитив → примитив) |
| **`IntStream`** | `.mapToObj(IntFunction<R>)` | `Stream<R>` | `int → объект` (примитив → объект) |
| **`Stream<T>`** | `.map(Function<T, R>)` | `Stream<R>` | `объект → объект` (объект → объект) |

---

## Примеры: объект → объект

### 1. Базовые преобразования
```java
// String → String (простая трансформация)
List<String> names = List.of("alice", "bob", "charlie");
List<String> upper = names.stream()
    .map(String::toUpperCase)  // String → String
    .toList(); // ["ALICE", "BOB", "CHARLIE"]

// String → Integer
List<String> numbers = List.of("1", "2", "3");
List<Integer> ints = numbers.stream()
    .map(Integer::parseInt)    // String → Integer
    .toList(); // [1, 2, 3]

// Integer → String
List<Integer> values = List.of(1, 2, 3);
List<String> strings = values.stream()
    .map(Object::toString)     // Integer → String
    .toList(); // ["1", "2", "3"]
```

### 2. Преобразование объектов DTO/Entity
```java
record UserDTO(String name, int age) {}
record UserEntity(Long id, String name, int age) {}

List<UserDTO> dtos = List.of(
    new UserDTO("Alice", 25),
    new UserDTO("Bob", 30)
);

// DTO → Entity
List<UserEntity> entities = dtos.stream()
    .map(dto -> new UserEntity(
        null,           // id генерится в БД
        dto.name(),
        dto.age()
    ))                 // UserDTO → UserEntity
    .toList();
```

### 3. Цепочки преобразований объектов
```java
List<String> words = List.of("  hello ", "  world  ", "  java  ");

List<String> clean = words.stream()
    .map(String::trim)         // String → String (убрать пробелы)
    .map(String::toUpperCase)  // String → String (в верхний регистр)
    .map(s -> "[" + s + "]")   // String → String (добавить скобки)
    .toList(); // ["[HELLO]", "[WORLD]", "[JAVA]"]
```

### 4. Извлечение вложенных данных
```java
record Order(Long id, List<OrderItem> items) {}
record OrderItem(String product, double price) {}

List<Order> orders = ...;

// Order → String (имена продуктов)
List<String> allProducts = orders.stream()
    .flatMap(order -> order.items().stream()) // Order → Stream<OrderItem>
    .map(OrderItem::product)                  // OrderItem → String
    .distinct()
    .toList();

// Order → Double (общая сумма заказа)
List<Double> orderTotals = orders.stream()
    .map(order -> order.items().stream()
        .mapToDouble(OrderItem::price)
        .sum())                              // Order → Double
    .toList();
```

## Особенности работы с `Stream<T>.map()`

### 1. **Нет ограничений на тип преобразования**
```java
// Можно преобразовать в любой тип
Stream<Object> objects = Stream.of("text", 42, 3.14);
Stream<String> strings = objects.map(Object::toString);

// Можно даже в совершенно другой иерархии
List<File> files = List.of(new File("a.txt"), new File("b.txt"));
List<Path> paths = files.stream()
    .map(File::toPath)  // File → Path (разные иерархии классов)
    .toList();
```

### 2. **Работа с Optional**
```java
List<String> inputs = List.of("1", "2", "abc", "3");

// Безопасный парсинг с Optional
List<Integer> numbers = inputs.stream()
    .map(s -> {
        try {
            return Optional.of(Integer.parseInt(s));
        } catch (NumberFormatException e) {
            return Optional.<Integer>empty();
        }
    })
    .filter(Optional::isPresent)
    .map(Optional::get)
    .toList(); // [1, 2, 3]

// Или с flatMap (более идиоматично)
List<Integer> numbers2 = inputs.stream()
    .flatMap(s -> {
        try {
            return Stream.of(Integer.parseInt(s));
        } catch (NumberFormatException e) {
            return Stream.empty();
        }
    })
    .toList();
```

### 3. **Композиция функций**
```java
// Можно комбинировать функции
Function<String, String> trim = String::trim;
Function<String, String> upper = String::toUpperCase;
Function<String, String> exclaim = s -> s + "!";

// Композиция вручную
List<String> result1 = words.stream()
    .map(trim.andThen(upper).andThen(exclaim))
    .toList();

// Или цепочкой map
List<String> result2 = words.stream()
    .map(trim)
    .map(upper)
    .map(exclaim)
    .toList(); // То же самое
```

## Сравнение с примитивными стримами

### Пример: один и тот же результат разными путями
```java
// Исходные данные: список строк с числами
List<String> stringNumbers = List.of("10", "20", "30");

// Способ 1: Объектный стрим (String → Integer → преобразование)
List<Integer> result1 = stringNumbers.stream()      // Stream<String>
    .map(Integer::parseInt)        // String → Integer
    .map(x -> x * 2)               // Integer → Integer (BOXING/UNBOXING!)
    .toList();                     // [20, 40, 60]

// Способ 2: Через примитивный стрим (оптимальнее)
List<Integer> result2 = stringNumbers.stream()      // Stream<String>
    .mapToInt(Integer::parseInt)   // String → int (примитив)
    .map(x -> x * 2)               // int → int (примитивная операция)
    .boxed()                       // int → Integer (BOXING только в конце)
    .toList();                     // [20, 40, 60] - БЫСТРЕЕ!

// Способ 3: Полностью примитивный (если не нужны объекты)
int sum = stringNumbers.stream()
    .mapToInt(Integer::parseInt)   // String → int
    .map(x -> x * 2)               // int → int
    .sum();                        // 120 - САМЫЙ БЫСТРЫЙ
```

## Паттерны для объектных преобразований

### Паттерн 1: Трансформация данных
```java
// CSV строки → объекты
List<String> csvLines = List.of(
    "John,25,Developer",
    "Alice,30,Manager",
    "Bob,28,Designer"
);

record Employee(String name, int age, String position) {}

List<Employee> employees = csvLines.stream()
    .map(line -> line.split(","))
    .map(parts -> new Employee(
        parts[0].trim(),
        Integer.parseInt(parts[1].trim()),
        parts[2].trim()
    ))
    .toList();
```

### Паттерн 2: Обогащение данных
```java
record Product(String id, double price) {}
record EnrichedProduct(String id, double price, double tax, double total) {}

List<Product> products = ...;

List<EnrichedProduct> enriched = products.stream()
    .map(p -> {
        double tax = p.price() * 0.20;
        double total = p.price() + tax;
        return new EnrichedProduct(p.id(), p.price(), tax, total);
    })
    .toList();
```

### Паттерн 3: Группировка и преобразование
```java
Map<String, List<Employee>> byDepartment = employees.stream()
    .collect(Collectors.groupingBy(Employee::department));

// Преобразуем Map<String, List<Employee>> в Map<String, List<String>>
Map<String, List<String>> namesByDept = byDepartment.entrySet().stream()
    .collect(Collectors.toMap(
        Map.Entry::getKey,                    // ключ оставляем
        entry -> entry.getValue().stream()    // List<Employee> →
            .map(Employee::name)              //   → Stream<String>
            .toList()                         //   → List<String>
    ));
```

## Производительность: объекты vs примитивы

```java
// Benchmark сравнения
int size = 1_000_000;

// Объекты: медленнее из-за накладных расходов
List<Integer> objectList = IntStream.range(0, size)
    .boxed()
    .toList();

long time1 = measure(() -> {
    int sum = objectList.stream()
        .map(x -> x * 2)      // Integer → Integer (unbox/box внутри)
        .reduce(0, Integer::sum);
});

// Примитивы: быстрее
int[] primitiveArray = IntStream.range(0, size).toArray();

long time2 = measure(() -> {
    int sum = Arrays.stream(primitiveArray)
        .map(x -> x * 2)      // int → int
        .sum();
});

// Разница может быть 3-10 раз в пользу примитивов!
```

## Когда использовать что?

### **Используйте `Stream<T>.map()` (объект → объект) когда:**
1. Уже работаете с объектным стримом (`Stream<String>`, `Stream<Employee>` и т.д.)
2. Нужно преобразовать один тип объекта в другой (`DTO → Entity`)
3. Преобразование сложное (вызов нескольких методов, создание новых объектов)
4. Работаете не с числами (строки, даты, кастомные объекты)

### **Используйте примитивные стримы (`IntStream.map()` или `.mapToObj()`) когда:**
1. Исходные данные примитивные (массивы `int[]`, `long[]`, `double[]`)
2. Нужны числовые операции и важна производительность
3. Делаете агрегацию (суммы, средние, статистика)
4. Работаете с большими объемами числовых данных

---

## Ключевые выводы

1. **`Stream<T>.map()`** — для преобразований **объект → объект**
2. **`IntStream.map()`** — для преобразований **примитив → примитив** (остаемся в примитивном стриме)
3. **`.mapToObj()`** — для переходов **примитив → объект** (выходим из примитивного стрима)
4. **Для чисел предпочитайте примитивные стримы** до последнего момента
5. **Для сложных объектов используйте `Stream<T>.map()`** — он гибче

Простое правило:  
**Смотрите на тип вашего стрима:**  
- `Stream<T>` → используйте `.map()`  
- `IntStream`/`LongStream`/`DoubleStream` → используйте `.map()` для примитивов, `.mapToObj()` для объектов
