- Это **логическая группировка классов и интерфейсов**.
    
- Нужны для:
    
    - организации кода (разделение по функционалу);
        
    - предотвращения конфликтов имён классов;
        
    - управления уровнем доступа (`public`, `protected`, _package-private_).
        
- Отражают **структуру каталогов** в проекте.
## Основные правила

1. **Строчные буквы**  
    Все имена пакетов пишутся только в нижнем регистре:
    
    `package com.example.myapp;`
    
2. **Уникальность за счёт домена**  
    Обычно название пакета начинается с **обратного интернет-домена организации**:
    
    - `com.companyname.project`
        
    - `org.openai.chatgpt`
        
    - `by.university.lab1`
        
3. **Иерархия директорий**  
    Пакеты отражают структуру папок:
    
    `src/ └── com/     └── example/         └── myapp/             └── Main.java   (package com.example.myapp;)`
    
4. **Без спецсимволов**  
    Только буквы и цифры, **никаких пробелов, дефисов или подчеркиваний**.  
    Разделение слов делается через "склеивание":
    
    - ❌ `my_app`
        
    - ✅ `myapp`
        
5. **Тематика пакета отражает назначение**  
    Часто используется структура:
    
    - `model` — классы сущностей (например, `User`, `Order`).
        
    - `dao` / `repository` — доступ к данным.
        
    - `service` — бизнес-логика.
        
    - `controller` — API или UI слой.
        
    - `util` / `common` — утилиты.
