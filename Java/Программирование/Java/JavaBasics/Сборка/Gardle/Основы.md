Gradle — это открытая система автоматизации сборки проектов, сочетающая гибкость Ant и декларативность Maven, но с использованием Groovy или Kotlin DSL вместо XML. Она особенно популярна для Java, Android, Kotlin и мультиязыковых проектов благодаря инкрементальной сборке, кэшированию и параллелизму задач.

Конфигурация хранится в **build.gradle** (или build.gradle.kts для Kotlin). Файл определяет плагины, зависимости, задачи и репозитории. Стандартная структура: src/main/java, src/test/java, build/ (выходные файлы). Gradle использует Directed Acyclic Graph (DAG) для задач, чтобы запускать только изменившиеся части, ускоряя сборку.

Жизненный цикл похож на Maven: phases как build, check, assemble. Задачи (tasks) — базовые единицы: compileJava, test, jar. Кастомные задачи пишутся на Groovy/Kotlin.
