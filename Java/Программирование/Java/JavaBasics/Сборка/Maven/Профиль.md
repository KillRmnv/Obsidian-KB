Maven Profiles позволяют настраивать сборку под разные среды (dev, prod, test), добавляя зависимости, свойства или плагины условно. Они определяются в `<profiles>` секции pom.xml и активируются через `-P`, переменные окружения или автоматически.knowledgebasement+1

## Активация профилей

- **Командная строка**: `mvn clean install -Pdev,prod` (несколько через запятую).
    
- **Автоматическая**: `<activeByDefault>true</activeByDefault>`, по JDK (`<jdk>1.8</jdk>`), OS (`<os><family>Windows</family></os>`), файлу (`<file>exists.txt</file>`).
    
- **Переменные**: `<activation><property><name>env</name><value>dev</value></property></activation>` + `mvn -Denv=dev`.
    

## Пример в pom.xml

xml

```
<profiles>
	 <profile> 
		 <id>dev</id> 
		 <activation> 
			 <activeByDefault>true</activeByDefault> 
		  </activation> 
		<properties> <db.url>jdbc:h2:mem:testdb</db.url> </properties>            <dependencies> 
			<dependency> 
			<groupId>com.h2database</groupId>
			 <artifactId>h2</artifactId> 
			 <scope>test</scope> 
		     </dependency> 
	      </dependencies> 
	</profile> 
	<profile> 
			<id>prod</id> <properties>     <db.url>jdbc:postgresql://prod/db</db.url> 
		</properties>
</profile> 
</profiles>
```
Активация: `mvn package -Pprod` применит prod-настройки.baeldung+2

## Полезные команды

- `mvn help:active-profiles` — список активных профилей.
    
- `mvn help:effective-pom -Pdev` — effective POM с профилем.[[maven.apache](https://maven.apache.org/guides/introduction/introduction-to-profiles.html)]​
