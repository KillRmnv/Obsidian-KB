Maven позволяет запускать Java-проекты через плагины `exec-maven-plugin` (для обычных приложений) или `spring-boot-maven-plugin` (для Spring Boot). В Zed IDE используйте встроенный терминал (Ctrl+`), так как нет встроенных run-конфигураций как в IntelliJ [web:27].

## Обычный запуск

Для простых Maven-проектов (с main-классом) используйте exec-плагин.

- `mvn exec:java -Dexec.mainClass="com.example.App"` — запуск main-класса.zetcode+1
    
- С аргументами: `mvn exec:java -Dexec.mainClass="com.example.App" -Dexec.args="arg1 arg2"`.[[zetcode](https://zetcode.com/maven/exec-plugin/)]​
    
- Для Spring Boot: `mvn spring-boot:run` — автоматический запуск.[[docs.redhat](https://docs.redhat.com/en/documentation/red_hat_support_for_spring_boot/2.3/html/spring_boot_runtime_guide/debugging-springboot-based-application_spring-boot)]​
    

Сначала соберите: `mvn clean compile` или `mvn package`.[[leoborai](https://leoborai.com/notes/use-zed-as-ide-for-java-on-macos)]​

## Отладка (debug)

Запускайте с JVM-аргументами для remote debug (порт 5005 по умолчанию).

- Обычный проект: `mvn exec:java -Dexec.mainClass="com.example.App" -Dexec.args="" -Dexec.jvmArguments="-Xdebug -Xrunjdwp:transport=dt_socket,server=y,suspend=n,address=5005"`.[[zetcode](https://zetcode.com/maven/exec-plugin/)]​
    
- Spring Boot: `mvn spring-boot:run -Drun.jvmArguments="-Xdebug -Xrunjdwp:transport=dt_socket,server=y,suspend=n,address=5005"`.spring+1
    
- С паузой до подключения: `suspend=y` (идеально для IDE).[[docs.redhat](https://docs.redhat.com/en/documentation/red_hat_support_for_spring_boot/2.3/html/spring_boot_runtime_guide/debugging-springboot-based-application_spring-boot)]​
