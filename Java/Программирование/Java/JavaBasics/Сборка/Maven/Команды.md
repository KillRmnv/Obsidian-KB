Maven предоставляет множество полезных команд для управления жизненным циклом проекта, зависимостями и диагностикой. Вот наиболее востребованные, сгруппированные по назначению.

## Команды clean lifecycle

Выполняют очистку проекта последовательно.

|Команда|Описание|
|---|---|
|mvn pre-clean|Подготовка к очистке.|
|mvn clean|Удаление `target/` (основная очистка).|
|mvn post-clean|Действия после очистки.|

## Команды default lifecycle

Полный цикл сборки (самый используемый). Запуск `mvn package` выполнит все до `package`.

| Команда                    | Описание                           |
| -------------------------- | ---------------------------------- |
| mvn validate               | Валидация pom.xml.                 |
| mvn initialize             | Инициализация.                     |
| mvn generate-sources       | Генерация исходников.              |
| mvn process-sources        | Обработка исходников.              |
| mvn generate-resources     | Генерация ресурсов.                |
| mvn process-resources      | Копирование ресурсов.              |
| mvn compile                | Компиляция основного кода.         |
| mvn process-test-sources   | Обработка тестовых исходников.     |
| mvn process-test-resources | Копирование тестовых ресурсов.     |
| mvn test-compile           | Компиляция тестов.                 |
| mvn test                   | Запуск unit-тестов.                |
| mvn prepare-package        | Подготовка к упаковке.             |
| mvn package                | Упаковка в JAR/WAR.                |
| mvn integration-test       | Интеграционные тесты.              |
| mvn post-integration-test  | После интеграционных тестов.       |
| mvn verify                 | Проверка сборки.                   |
| mvn install                | Установка в локальный репозиторий. |
| mvn deploy                 | Деплой в удалённый репозиторий.    |

## Команды site lifecycle

Генерация сайта и документации.

| Команда         | Описание                        |
| --------------- | ------------------------------- |
| mvn pre-site    | Подготовка сайта.               |
| mvn site        | Сборка сайта (Javadoc, отчёты). |
| mvn post-site   | После сборки сайта.             |
| mvn site-deploy | Деплой сайта.                   |
## Комбинированные сборки

Часто используются для быстрой автоматизации.

- `mvn clean compile` — очистка + компиляция.[[arenda-server](https://arenda-server.cloud/blog/shpargalka-po-komandam-i-opcijam-maven/)]​
    
- `mvn clean test` — очистка + тесты.[[arenda-server](https://arenda-server.cloud/blog/shpargalka-po-komandam-i-opcijam-maven/)]​
    
- `mvn clean package` — очистка + JAR/WAR.[[arenda-server](https://arenda-server.cloud/blog/shpargalka-po-komandam-i-opcijam-maven/)]​
    
- `mvn clean install` — очистка + установка локально.[[arenda-server](https://arenda-server.cloud/blog/shpargalka-po-komandam-i-opcijam-maven/)]​
    

## Зависимости и анализ

Для работы с `pom.xml` и внешними библиотеками.

- `mvn dependency:tree` — дерево всех зависимостей.[[arenda-server](https://arenda-server.cloud/blog/shpargalka-po-komandam-i-opcijam-maven/)]​
    
- `mvn dependency:resolve` — загрузка всех зависимостей.[[arenda-server](https://arenda-server.cloud/blog/shpargalka-po-komandam-i-opcijam-maven/)]​
    
- `mvn versions:display-dependency-updates` — устаревшие версии зависимостей.[[arenda-server](https://arenda-server.cloud/blog/shpargalka-po-komandam-i-opcijam-maven/)]​
    
- `mvn help:effective-pom` — итоговый POM после профилей.education-wiki+1
    

## Пропуск тестов и профили

Опции для ускорения или кастомизации.

- `mvn clean package -DskipTests` — сборка без тестов.[[arenda-server](https://arenda-server.cloud/blog/shpargalka-po-komandam-i-opcijam-maven/)]​
    
- `mvn clean package -Dmaven.test.skip=true` — без компиляции/тестов.[[arenda-server](https://arenda-server.cloud/blog/shpargalka-po-komandam-i-opcijam-maven/)]​
    
- `mvn clean package -Pdev` — активация профиля `dev`.[[arenda-server](https://arenda-server.cloud/blog/shpargalka-po-komandam-i-opcijam-maven/)]​
    

## Диагностика и плагины

Полезно для отладки.

- `mvn help:active-profiles` — активные профили.[[arenda-server](https://arenda-server.cloud/blog/shpargalka-po-komandam-i-opcijam-maven/)]​
    
- `mvn archetype:generate` — генерация нового проекта.[[wiki.merionet](https://wiki.merionet.ru/articles/20-komand-i-parametrov-maven)]​
    
- `mvn dependency:purge-local-repository` — очистка локального кэша.[[arenda-server](https://arenda-server.cloud/blog/shpargalka-po-komandam-i-opcijam-maven/)]​
    

## Запуск приложений

Для Spring Boot или простых JAR.

- `mvn spring-boot:run` — запуск Spring Boot.[[arenda-server](https://arenda-server.cloud/blog/shpargalka-po-komandam-i-opcijam-maven/)]​
    
- `mvn exec:java -Dexec.mainClass="com.example.Main"` — запуск класса.[[arenda-server](https://arenda-server.cloud/blog/shpargalka-po-komandam-i-opcijam-maven/)]​
    

Команды запускаются из корня проекта с `pom.xml`. Для Zed IDE используйте встроенный терминал (Ctrl+`).
