Maven плагины — это расширения, подключаемые в `pom.xml`, для автоматизации задач сборки, запуска, анализа и деплоя. Они определяют цели (goals), вызываемые командами вроде `mvn plugin:goal`.[[arenda-server](https://arenda-server.cloud/blog/shpargalka-po-komandam-i-opcijam-maven/)]​

## Exec Maven Plugin

Плагин для запуска Java-программ или системных команд.

- Цели: `exec:java` (Java в той же JVM), `exec:exec` (отдельный процесс).digitalocean+1
    
- Подключение в pom.xml:
    
    text
    
    `<plugin>   <groupId>org.codehaus.mojo</groupId>  <artifactId>exec-maven-plugin</artifactId>  <version>3.1.0</version> </plugin>`
    
- Использование: `mvn exec:java -Dexec.mainClass="com.example.App"`.[[mojohaus](https://www.mojohaus.org/exec-maven-plugin/usage.html)]​
    

## Spring Boot Maven Plugin

Для Spring Boot: repackage в executable JAR, запуск, build образа.

- Цели: `spring-boot:run`, `spring-boot:build-image`, `spring-boot:repackage`.sonatype+1
    
- Подключение (авто в Spring Initializr):
    
    text
    
    `<plugin>   <groupId>org.springframework.boot</groupId>  <artifactId>spring-boot-maven-plugin</artifactId> </plugin>`
    
- Debug: `mvn spring-boot:run -Drun.jvmArguments="-Xdebug -agentlib:jdwp=transport=dt_socket,server=y,suspend=n,address=*:5005"`.[[docs.spring](https://docs.spring.io/spring-boot/docs/2.1.0.RELEASE/maven-plugin/examples/run-debug.html)]​
