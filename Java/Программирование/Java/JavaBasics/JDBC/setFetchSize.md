### **`setFetchSize()` (JDBC) — Техническое ограничение при передаче**

Это настройка **JDBC-драйвера** (клиента), которая говорит, **как передавать данные** от сервера к приложению.

- **Суть**: Вы говорите драйверу: "Запрос может вернуть миллион строк, но не присылай мне их все сразу, чтобы не занять всю оперативную память. Присылай пачками по 1000 строк по мере того, как я буду их перебирать в цикле `while(rs.next())`".
    
- **Результат**: База данных **находит все строки**, удовлетворяющие запросу (если нет `LIMIT`), и держит курсор открытым. Драйвер забирает результаты порциями. Общий объем переданных данных в итоге будет **одинаковым** (будут переданы все строки), но пиковое потребление памяти снижается.
    
- **Цель**: Оптимизировать потребление памяти клиентского приложения и уменьшить количество сетевых вызовов при работе с **очень большими наборами данных**.
    

**Аналогия**: Вы берете ту же книгу "Война и мир" и решаете прочитать её целиком, но у вас маленькая сумочка, в которую влезает только 50 страниц. Вы просите библиотекаря: "Дай мне всю книгу на неделю, но я буду приходить к тебе каждый час и забирать по 50 страниц, а прочитанные листы буду отдавать". В итоге вы прочитаете всю книгу (все строки), но физически носите с собой только по 50 страниц.

java

// Пример в коде
Statement stmt = connection.createStatement();
// Говорим драйверу: "Тащи данные пачками по 100 строк"
stmt.setFetchSize(100);
ResultSet rs = stmt.executeQuery("SELECT * FROM orders");
// В памяти сейчас максимум 100 строк из миллиона.
while (rs.next()) {
    // Когда заканчивается текущая пачка (100 строк),
    // драйвер под капотом идет в БД и тащит следующие 100.
}

## Сравнительная таблица

|Характеристика|`LIMIT` / `OFFSET` (SQL)|`setFetchSize()` (JDBC)|
|---|---|---|
|**Где выполняется**|На сервере базы данных|На клиенте (драйвером) и частично на сервере|
|**Что происходит**|БД **не выбирает** лишние строки из таблицы. Она ищет только нужный диапазон.|БД выбирает **все строки** (создает полный результирующий набор), но отдает их клиенту частями.|
|**Конечный результат**|Клиент получает только несколько записей. **Объем данных мал**.|Клиент получает **все записи**, но по частям. **Объем данных большой**.|
|**Цель**|**Пагинация** (переключение страниц), отдача небольшого среза данных пользователю.|**Потоковая обработка** больших данных (отчеты, выгрузки) без переполнения оперативной памяти (OutOfMemoryError).|
|**Нагрузка на сеть**|Низкая (передается мало данных).|Высокая (передаются все данные, просто по частям).|
|**Нагрузка на RAM клиента**|Низкая (данных мало).|Регулируется (не больше `fetchSize` в пике).|
|**Нагрузка на БД (CPU/IO)**|Может быть высокой, если `OFFSET` большой (БД всё равно приходится "прыгать" по таблице, но в современных БД это оптимизировано).|Высокая, если не использовать `LIMIT`, так как БД вынуждена сканировать всю таблицу/индекс.|
