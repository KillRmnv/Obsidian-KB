**Apache Tomcat** (часто просто **Tomcat**) — это один из самых популярных серверов приложений с открытым исходным кодом.

Вот подробный гайд: что это, зачем нужно и как с этим работать.

---

### 1. Что такое Tomcat?

Простыми словами: **Tomcat — это «двигатель» для запуска Java-веб-приложений.**

*   **Техническое определение:** Это реализация спецификаций **Jakarta Servlet** и **Jakarta Server Pages** (ранее Java EE).
*   **В чем разница с Nginx или Apache HTTP Server?**
    *   *Nginx/Apache HTTP:* Отдают статические файлы (HTML, картинки, CSS). Они не «понимают» Java-код.
    *   *Tomcat:* Умеет выполнять Java-код (Servlets, JSP), обрабатывать логику приложения, работать с базами данных и генерировать динамический контент.
*   **Зачем он нужен:** Если вы написали сайт на Java (Spring Boot, Struts, чистые Servlets), вам нужен Tomcat, чтобы этот сайт открылся в браузере.

---

### 2. Подготовка и Установка

Tomcat написан на Java, поэтому перед установкой нужна сама Java.

#### Шаг 1: Установка Java (JDK/JRE)
Tomcat требует установленную Java.
1.  Проверьте версию: `java -version`.
2.  Для Tomcat 9 нужна Java 8+. Для Tomcat 10+ лучше Java 11 или 17.
3.  Убедитесь, что переменная окружения `JAVA_HOME` указывает на папку с Java.

#### Шаг 2: Скачивание Tomcat
1.  Зайдите на официальный сайт: [tomcat.apache.org](https://tomcat.apache.org/).
2.  Скачайте архив (Core -> **zip** или **tar.gz**).
    *   *Совет:* Выбирайте версию, соответствующую вашей Java. (Например, Tomcat 9 — самый стабильный и популярный сейчас).
3.  Распакуйте архив в удобную папку (например, `/opt/tomcat` на Linux или `C:\tomcat` на Windows).

---

### 3. Структура папок (Важно!)

После распаковки вы увидите несколько папок. Вот главные из них:

*   **`bin/`** — Скрипты для запуска и остановки сервера.
*   **`conf/`** — Файлы конфигурации (здесь настраивают порты, пользователей и т.д.).
*   **`webapps/`** — **Самая важная папка.** Сюда кладут файлы приложений (`.war`), чтобы они заработали.
*   **`logs/`** — Логи работы сервера (если что-то сломалось, смотрим сюда).
*   **`lib/`** — Библиотеки, необходимые самому Tomcat.

---

### 4. Как запустить и остановить

В папке `bin` есть скрипты.

**На Windows:**
*   Запуск: `startup.bat`
*   Остановка: `shutdown.bat`
*   *Примечание:* Лучше запускать от имени администратора, если нужны порты ниже 1024.

**На Linux / macOS:**
*   Запуск: `./startup.sh`
*   Остановка: `./shutdown.sh`
*   *Примечание:* Скриптам нужно дать права на выполнение: `chmod +x *.sh`.

**Проверка:**
Откройте браузер и введите: `http://localhost:8080`.
Если вы увидите страницу с логотипом Tomcat и котом — **сервер работает!**

---

### 5. Как разместить свое приложение (Deploy)

Есть два основных способа:

#### Способ А: Автоматический (через папку webapps)
1.  Соберите свое Java-приложение в файл с расширением **`.war`** (Web ARchive).
2.  Скопируйте этот файл в папку `webapps`.
    *   Например: `my-app.war`.
3.  Tomcat автоматически распакует его и запустит.
4.  Приложение будет доступно по адресу: `http://localhost:8080/my-app/`.

#### Способ Б: Через Manager App (Веб-интерфейс)
1.  По умолчанию доступ закрыт. Нужно настроить пользователя в файле `conf/tomcat-users.xml`.
2.  Добавьте роль `manager-gui` и пользователя:
    ```xml
    <role rolename="manager-gui"/>
    <user username="admin" password="password123" roles="manager-gui"/>
    ```
3.  Перезапустите Tomcat.
4.  Зайдите на `http://localhost:8080/manager/html`, введите логин/пароль и загрузите `.war` файл через интерфейс.

---

### 6. Основная конфигурация

Все настройки находятся в папке `conf/`. Главный файл — **`server.xml`**.

**Частые задачи:**

1.  **Смена порта:**
    По умолчанию Tomcat слушает порт **8080**. Чтобы изменить на стандартный веб-порт **80**:
    *   Откройте `conf/server.xml`.
    *   Найдите строку `<Connector port="8080" ... />`.
    *   Поменяйте на `<Connector port="80" ... />`.
    *   *Внимание:* На Linux для порта 80 нужны права root (не рекомендуется запускать Tomcat от root!).

2.  **Настройка памяти (Java Heap):**
    Если приложение «тяжелое», нужно выделить больше памяти.
    *   В Linux: отредактируйте `bin/setenv.sh` (создайте, если нет).
    *   Добавьте: `export CATALINA_OPTS="-Xms512M -Xmx1024M"` (минимум 512 Мб, максимум 1 Гб).

---

### 7. Работа в Production (Боевой режим)

Никогда не используйте «голый» Tomcat в продакшене без защиты.

1.  **Не запускайте от root:** Создайте специального пользователя (например, `tomcat`) и запускайте сервер от его имени.
2.  **Скройте версию:** В логах и заголовках не должно быть видно, что у вас Tomcat 9.0.50 (это помогает хакерам искать уязвимости).
3.  **Используйте Reverse Proxy (Nginx/Apache):**
    *   Ставьте перед Tomcat Nginx.
    *   Nginx принимает запросы от пользователей (порт 80/443 SSL).
    *   Nginx передает динамические запросы на Tomcat (порт 8080).
    *   Nginx раздает статику (картинки, css), разгружая Tomcat.
4.  **Удалите лишнее:** В папке `webapps` удалите стандартные приложения (`docs`, `examples`, `manager`, `host-manager`), если они не нужны. Это уменьшает поверхность атаки.
