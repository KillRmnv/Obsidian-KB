**JSP (JavaServer Pages)** — это технология для создания динамических веб-страниц на языке Java.

Простыми словами: **JSP — это HTML-файл, в который можно вставлять Java-код.**

---

### 1. Зачем нужен JSP?

В статическом HTML вы не можете показать данные из базы данных (например, "Привет, Иван!" или список товаров).
JSP позволяет генерировать HTML на лету в зависимости от данных.

*   **Было (Servlet):** Вы пишете Java-код, который вручную собирает HTML строками:
    ```java
    out.println("<html><body>");
    out.println("<h1>Привет, " + userName + "</h1>");
    out.println("</body></html>");
    ```
    *Это неудобно: верстать в Java сложно.*

*   **Стало (JSP):** Вы пишете обычный HTML, а Java-код вставляете только там, где нужны данные:
    ```html
    <html><body>
        <h1>Привет, ${userName}</h1>
    </body></html>
    ```
    *Это удобно: верстальщик работает с HTML, программист — с данными.*

---

### 2. Как это работает внутри Tomcat?

Вспомните архитектуру Tomcat (компонент **Jasper**).
Когда вы запрашиваете страницу `index.jsp`:

1.  **Translation (Трансляция):** Tomcat берет файл `.jsp` и превращает его в обычный Java-класс (Servlet).
2.  **Compilation (Компиляция):** Этот класс компилируется в `.class`.
3.  **Execution (Выполнение):** Servlet выполняется, генерирует чистый HTML.
4.  **Response (Ответ):** Чистый HTML отправляется в браузер.

**Важно:** Браузер **не видит** Java-код. Он получает только результат (HTML).

### 3. Синтаксис JSP

В JSP есть несколько способов вставки кода:

#### А. Выражения (Expression Language — EL)
Современный и рекомендуемый способ. Использует фигурные скобки с долларом.
```jsp
<p>Имя пользователя: ${user.name}</p>
<p>Сумма: ${price * quantity}</p>
```

#### Б. Скриптлеты (Scriptlets) — *Устаревший способ*
Прямая вставка Java-кода. **Не рекомендуется использовать** в новых проектах, так как смешивает логику и вид.
```jsp
<%
    int x = 10;
    if (x > 5) {
%>
    <p>Число больше 5</p>
<%
    }
%>
```

#### В. Директивы (Directives)
Настройки для самой страницы (импорт классов, кодировка).
```jsp
<%@ page contentType="text/html;charset=UTF-8" %>
<%@ taglib uri="http://java.sun.com/jsp/jstl/core" prefix="c" %>
```

#### Г. JSTL (Tag Library)
Стандартные теги для логики (циклы, условия), чтобы не писать Java-скриптлеты.
```jsp
<c:forEach var="item" items="${productList}">
    <div>${item.name}</div>
</c:forEach>
```

### 4. JSP в архитектуре MVC

В классической Java-веб разработке (Model-View-Controller) JSP занимает роль **View (Представление)**.

1.  **Controller (Servlet / Spring Controller):** Принимает запрос, получает данные из Базы Данных.
2.  **Model (Java Objects):** Данные (например, объект `User`).
3.  **View (JSP):** Получает данные от Контроллера и рисует их в HTML.

---

### 5. Актуальность: Использовать или нет?

Это самый важный вопрос.

| Плюсы JSP | Минусы JSP |
| :--- | :--- |
| ✅ Быстро для простых проектов | ❌ Смешивает логику и верстку (плохая архитектура) |
| ✅ Мощная экосистема тегов | ❌ Сложно тестировать автоматически |
| ✅ Работает "из коробки" в Tomcat | ❌ Требует перезагрузки контекста при изменении (часто) |
| ✅ Много готового кода в интернете | ❌ **Устаревающая технология** |

#### Современный подход (2024+)
В новых проектах на **Spring Boot** JSP используют **редко**.

1.  **REST API + Frontend (React/Vue/Angular):**
    *   Spring отдает только данные (JSON).
    *   Отдельное JS-приложение рисует интерфейс.
    *   JSP не нужен вообще.

2.  **Шаблонизаторы (Thymeleaf, Freemarker):**
    *   Если нужно генерировать HTML на сервере (например, для SEO или простых админок), используют **Thymeleaf**.
    *   Он лучше интегрируется со Spring, не требует компиляции в Servlet и выглядит чище.

### 6. Пример файла `index.jsp`

```jsp
<%@ page contentType="text/html;charset=UTF-8" language="java" %>
<html>
<head>
    <title>Пример JSP</title>
</head>
<body>
    <h1>Привет, ${username}!</h1>

    <h2>Список задач:</h2>
    <ul>
        <!-- Цикл по списку, который пришел из Controller -->
        <c:forEach var="task" items="${tasks}">
            <li>${task.name} (Статус: ${task.done})</li>
        </c:forEach>
    </ul>

    <footer>
        &copy; 2024 Моя Компания
    </footer>
</body>
</html>
```

### Итог
*   **JSP** — это технология для генерации HTML на стороне сервера с помощью Java.
*   **Tomcat** автоматически превращает JSP в Servlets.
*   **Для обучения:** Полезно понять, как работает веб "под капотом".
*   **Для работы:** В новых проектах лучше использовать **Spring Boot + Thymeleaf** (для серверного рендеринга) или **Spring Boot REST + React/Vue** (для раздельного фронтенда). JSP считается легаси (устаревшим) решением.
