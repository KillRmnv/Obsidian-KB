Java EE (Java Platform, Enterprise Edition) — это мощная платформа для разработки корпоративных приложений, которая значительно расширяет возможности стандартной Java (Java SE) . Вместо того чтобы изобретать велосипед для типовых задач, Java EE предоставляет разработчику готовый набор проверенных решений и API . Она определяет стандарты для компонентов, сервисов и взаимодействия, что позволяет создавать масштабируемые, надежные и безопасные приложения .

Ключевая идея Java EE — компонентно-ориентированный подход. Приложение собирается из готовых модулей, а всю рутинную работу по управлению транзакциями, безопасности, многопоточности и подключению к базам данных берет на себя **сервер приложений** (например, GlassFish, WildFly, WebLogic) . Разработчик может сосредоточиться на бизнес-логике.

Технологии Java EE можно логически сгруппировать по их назначению .

###  Веб-технологии (Представление)
Этот слой отвечает за взаимодействие с пользователем, будь то веб-страница или API для мобильного приложения.

*   **Сервлеты (Servlets)**: Фундаментальная технология для обработки HTTP-запросов и формирования ответов. Это "сердце" любого веб-приложения на Java . Вы уже знакомы с ними.
*   **JSP (JavaServer Pages)**: Технология для создания динамических веб-страниц, позволяющая встраивать Java-код прямо в HTML . JSP-страницы в итоге транслируются в сервлеты.
*   **JSF (JavaServer Faces)**: Фреймворк для построения пользовательских интерфейсов из готовых компонентов. Он упрощает создание сложных интерфейсов, управляя состоянием компонентов и навигацией .
*   **JSPL (JSP Standard Tag Library)**: Стандартная библиотека тегов для JSP. Она позволяет избежать написания Java-кода в JSP, используя готовые теги для циклов, условий и форматирования .

###  Бизнес-логика (Сервисы)
Здесь сосредоточена основная функциональность приложения.

*   **EJB (Enterprise JavaBeans)**: Компоненты для реализации бизнес-логики. Они бывают сессионными (для разового взаимодействия с клиентом), управляемыми сообщениями (для асинхронной обработки) и компонентами сущностей (сегодня для этого чаще используется JPA) .
*   **CDI (Contexts and Dependency Injection)**: Спецификация, которая облегчает связывание компонентов друг с другом. Благодаря CDI вы можете просто "попросить" внедрить нужный сервис, а не создавать его вручную .

### Взаимодействие с данными и системами
Технологии для работы с базами данных, обмена сообщениями и интеграции с другими системами.

*   **JPA (Java Persistence API)**: Стандарт для объектно-реляционного отображения (ORM). Он позволяет работать с реляционными базами данных через объекты Java, скрывая сложность SQL-запросов .
*   **JMS (Java Message Service)**: API для асинхронного обмена сообщениями между компонентами приложения или различными системами. Сообщения могут отправляться в очереди или по подписке .
*   **JDBC (Java Database Connectivity)**: Более низкоуровневый, но стандартный API для выполнения SQL-запросов к базам данных .
*   **JavaMail**: API для отправки электронной почты из приложения .

###  Веб-сервисы и распределенные вычисления
Технологии для создания API и обеспечения взаимодействия между удаленными компонентами.

*   **JAX-RS (Java API for RESTful Web Services)**: Упрощает создание RESTful веб-сервисов с помощью удобных аннотаций .
*   **JAX-WS (Java API for XML Web Services)**: Используется для создания более строгих SOAP-веб-сервисов на основе XML .
*   **JNDI (Java Naming and Directory Interface)**: Служба каталогов, которая позволяет находить и получать ссылки на различные ресурсы (базы данных, EJB-компоненты) по имени .

###  Системные сервисы (Транзакции, Безопасность)
Сервисы, которые управляют ключевыми аспектами работы приложения.

*   **JTA (Java Transaction API)**: API для управления транзакциями. Позволяет гарантировать, что несколько операций с данными будут выполнены целиком или не выполнены вовсе .
*   **JAAS (Java Authentication and Authorization Service)**: Сервис для аутентификации (кто ты?) и авторизации (что тебе можно делать?) .
*   **Bean Validation**: Позволяет задавать правила валидации данных (например, "поле не может быть пустым" или "email должен быть корректным") прямо в коде с помощью аннотаций .

###  Итог
Java EE — это не просто набор технологий, а целостная экосистема, которая покрывает все слои корпоративного приложения . Ее главная цель — дать разработчику стандартизированные инструменты для решения типовых задач, чтобы он мог сосредоточиться на уникальной бизнес-логике.
