Отлично! 🎉 Вы установили **полноценный пак конфигов "KooL's Hyprland Dotfiles"**. Это очень грамотно организованная структура — не один огромный файл, а модульная система.

Давайте разберём, **как в этом ориентироваться** и **где безопасно вносить правки**.

---

## 🗂️ Структура папки `~/.config/hypr/`

```
~/.config/hypr/
├── hyprland.conf          # 🔑 ГЛАВНЫЙ ФАЙЛ (точка входа)
├── configs/               # 📦 Системные конфиги (не трогайте)
├── UserConfigs/           # ✏️ ВАШИ настройки (редактируйте здесь!)
├── animations/            # 🎬 Пресеты анимаций
├── scripts/               # ⚙️ Скрипты для виджетов и функций
├── Monitor_Profiles/      # 🖥️ Настройки для нескольких мониторов
├── hyprlock.conf          # 🔒 Экран блокировки
├── hypridle.conf          # 😴 Управление сном/энергосбережением
└── ...
```

---

## 🔑 Главный принцип: `source = ...`

Файл `hyprland.conf` не содержит всех настроек напрямую. Вместо этого он **подключает** другие файлы:

```ini
# Пример из hyprland.conf:
source = ~/.config/hypr/configs/Keybinds.conf
source = ~/.config/hypr/UserConfigs/UserKeybinds.conf  # ← Ваши правки здесь!
```

> 💡 **Порядок важен:** Если один и тот же параметр задан в двух подключённых файлах, **победит тот, что подключён позже**. Поэтому `UserConfigs` подключаются в конце — ваши настройки переопределяют системные.

---

## ✏️ Где безопасно редактировать (папка `UserConfigs/`)

| Файл | Что настраивает | Пример правки |
|------|----------------|---------------|
| `UserKeybinds.conf` | Горячие клавиши | `bind = SUPER, T, exec, kitty` |
| `UserSettings.conf` | Общие настройки (курсор, декорации) | `general:border_size = 3` |
| `UserAnimations.conf` | Анимации окон | `animation = fade, 1, 4, easeout` |
| `UserDecorations.conf` | Тени, скругления, размытие | `decoration:rounding = 10` |
| `Startup_Apps.conf` | Программы при запуске | `exec-once = waybar` |
| `WindowRules.conf` | Правила для конкретных окон | `windowrulev2 = float, title:(Open File)` |

> ✅ **Правило:** Всегда правьте файлы в `UserConfigs/`, а не в `configs/`. При обновлении дотфилов системные файлы перезапишутся, а ваши — останутся.

## 🚀 С чего начать: 3 первых шага

### Шаг 1: Откройте главный конфиг для ознакомления
```bash
cat ~/.config/hypr/hyprland.conf
```
Посмотрите, какие файлы подключаются через `source =`. Это даст понимание архитектуры.

### Шаг 2: Откройте `UserKeybinds.conf` и добавьте свою клавишу
```bash
nano ~/.config/hypr/UserConfigs/UserKeybinds.conf
```

Добавьте в конец (примеры):
```ini
# Открыть терминал
bind = SUPER, Return, exec, kitty

# Закрыть активное окно
bind = SUPER, Q, killactive,

# Перезагрузить конфиг (после правок!)
bind = SUPER, Shift, C, exec, hyprctl reload

# Скриншот области
bind = SUPER Shift, S, exec, grim -g "$(slurp)" - | swappy -f -
```

### Шаг 3: Примените изменения
После правок **не нужно перезагружаться**. Просто нажмите:
```
Super + Shift + C
```
Или выполните в терминале:
```bash
hyprctl reload
```

## 🔍 Полезные команды для отладки

| Команда | Что делает |
|---------|-----------|
| `hyprctl reload` | Перезагружает конфиг без выхода из сессии |
| `hyprctl monitors` | Показывает информацию о мониторах |
| `hyprctl clients` | Список открытых окон |
| `hyprctl activewindow` | Информация об активном окне |
| `journalctl -b | grep hypr` | Логи Hyprland (если что-то не работает) |
| `hyprctl -j activewindow | jq` | Красивый вывод в JSON (нужен `jq`) |

## 🎨 Как поменять тему / обои

### Обои:
1.  Положите картинку в `~/.config/hypr/wallpapers/`
2.  Откройте `~/.config/hypr/wallust/wallust-hyprland.conf` или `hyprpaper.conf`
3.  Укажите путь к новому файлу
4.  Примените: `Super + Shift + C`

Или используйте скрипт из пакета:
```bash
~/.config/hypr/scripts/WallpaperSelect.sh
```

### Тема виджетов (Waybar):
```bash
~/.config/hypr/scripts/WaybarStyles.sh
```
Этот скрипт переключает предустановленные стили панели.

---

## ⚠️ Чего НЕ делать

| ❌ Не делайте | ✅ Делайте вместо этого |
|--------------|------------------------|
| Править файлы в `configs/` | Править только `UserConfigs/` |
| Удалять `source = ...` из `hyprland.conf` | Добавлять свои `source` в конец файла |
| Менять имена системных скриптов в `scripts/` | Создавать свои скрипты в `UserScripts/` |
| Забывать делать `hyprctl reload` | Привыкать к `Super+Shift+C` после каждой правки |

## 🧭 Карта для новичка: что смотреть в первую очередь

1.  **`~/.config/hypr/UserConfigs/UserKeybinds.conf`** — выучите горячие клавиши
2.  **`~/.config/hypr/UserConfigs/UserSettings.conf`** — поменяйте размер рамок, шрифты
3.  **`~/.config/hypr/configs/Startup_Apps.conf`** — добавьте автозапуск нужных программ
4.  **`~/.config/waybar/`** — настройка панели (отдельная папка, но связана с Hyprland)

## 💡 Совет на будущее: ведите свой лог изменений

Создайте файл `~/.config/hypr/UserConfigs/CHANGES.md` и записывайте туда:
```markdown
## 2026-03-07
- Добавил горячую клавишу Super+T для терминала
- Увеличил размер рамок до 3px
- Отключил анимацию сворачивания
```

Это спасёт вас, когда:
- Вы забудете, что меняли
- После обновления что-то сломается
- Вы захотите перенести настройки на другой ПК

---

## 🆘 Если что-то сломалось

1.  **Отмените последнюю правку** в файле `UserConfigs/`
2.  **Перезагрузите конфиг**: `Super + Shift + C`
3.  **Если не помогло** — посмотрите логи:
    ```bash
    journalctl -b | grep -i hypr | tail -50
    ```
4.  **Если совсем плохо** — восстановите из бэкапа (они в `~/.config/hypr/backup/` или `~/.config/backup/`)
