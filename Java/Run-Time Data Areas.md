**Run-Time Data Areas** — это области памяти, которые выделяются JVM (Java Virtual Machine) во время выполнения программы. Спецификация Java четко разделяет эти области в зависимости от их назначения и времени жизни.

Их можно разделить на две большие группы: те, что создаются для **всей JVM** (общие), и те, что создаются для **каждого потока** индивидуально.

---

### 1. Области памяти уровня потока (Thread-level)

Эти области создаются при запуске потока и уничтожаются при его завершении. Они изолированы: один поток не может залезть в стек другого.

- **PC Register (Program Counter):** «Счетчик команд». Каждый поток имеет свой PC, в котором хранится адрес текущей инструкции, выполняемой процессором. Если метод нативный, значение счетчика не определено.
    
- **JVM Stack (Стек):** Здесь хранятся локальные переменные, результаты промежуточных вычислений и данные для вызова методов.
    
    - Данные в стеке организованы в виде **Frames** (фреймов). Новый фрейм создается при каждом вызове метода и уничтожается при выходе из него.
        
    - Если стек переполняется, вылетает знаменитая `StackOverflowError`.
        
- **Native Method Stack:** Аналогичен обычному стеку, но используется для методов, написанных на других языках (например, C/C++), которые вызываются через JNI (Java Native Interface).
    

---

### 2. Области памяти уровня JVM (Shared)

Эти области доступны всем потокам приложения. Именно здесь происходит основная магия управления данными.

- **Heap (Куча):** Самая большая и важная область. Здесь «живут» все объекты и массивы, созданные через оператор `new`.
    
    - Именно кучу чистит **Garbage Collector**.
        
    - Если в куче не остается места для новых объектов, возникает `OutOfMemoryError`.
        
- **Method Area (Область методов):** Хранит структуру классов: пулы констант, данные о полях, методы, код конструкторов и статические переменные.
    
    - В HotSpot JVM (начиная с Java 8) эта область физически реализована в **Metaspace**, которая вынесена из основной кучи в нативную память системы.
        
- **Run-Time Constant Pool:** Часть Method Area. Это таблица констант (числа, строки, ссылки на методы и поля), на которые ссылается байт-код.

|**Область**|**Кто имеет доступ?**|**Что хранит?**|**Ошибка при нехватке**|
|---|---|---|---|
|**PC Register**|Один поток|Адрес текущей команды|—|
|**JVM Stack**|Один поток|Локальные переменные, фреймы|`StackOverflowError`|
|**Heap**|Все потоки|Объекты, массивы|`OutOfMemoryError`|
|**Method Area**|Все потоки|Метаданные классов, статику|`OutOfMemoryError: Metaspace`|

---

### Почему это важно знать?

Понимание этих областей помогает при тюнинге производительности:

1. **-Xss:** Устанавливает размер стека (влияет на глубину рекурсии).
    
2. **-Xmx / -Xms:** Устанавливает размер кучи (влияет на то, сколько объектов выдержит программа).
    
3. **Metaspace:** Помогает понять, почему память может «течь» за пределами кучи, если динамически создается слишком много классов.
