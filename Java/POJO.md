**POJO** расшифровывается как **Plain Old Java Object** («старый добрый Java-объект»).
### Основные характеристики POJO

Класс считается POJO, если он соблюдает следующие правила:

1. **Не наследуется** от сторонних классов (кроме `Object`).
    
2. **Не реализует** специфические интерфейсы сторонних фреймворков (JMS, EJB и т.д.).
    
3. **Не содержит** аннотаций, которые диктуют поведение сторонним библиотекам (хотя в современном понимании это правило часто смягчается).
    
4. **Автономен:** его можно скомпилировать и запустить в любой среде, где есть Java.
