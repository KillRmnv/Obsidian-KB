**Happens-before** — это логическое отношение в Java Memory Model (JMM), которое гарантирует, что изменения, внесенные одним потоком в память, будут **видны** другому потоку.

Простыми словами: если событие A находится в отношении _happens-before_ с событием B, то Java гарантирует, что всё, что сделал поток в событии A, будет гарантированно «замечено» потоком в событии B.

---

### Почему это вообще нужно?

Современные процессоры и компиляторы для оптимизации скорости делают две вещи:

1. **Кэширование:** Потоки могут хранить переменные в локальных кэшах ядер, а не в основной памяти.
    
2. **Переупорядочивание (Reordering):** Компилятор или процессор может менять порядок инструкций, если считает, что это не изменит результат в рамках _одного_ потока.
    

Без правил _happens-before_ второй поток может увидеть данные в непредсказуемом порядке или не увидеть их вовсе.

---

### Основные правила (Rules of Happens-Before)

Java определяет несколько сценариев, где это отношение возникает автоматически:

#### 1. Single Thread Rule (Внутри одного потока)

Внутри одного потока каждое действие _happens-before_ любому действию, идущему позже по тексту программы.

> _Результат:_ В рамках одного потока вы всегда видите свои предыдущие записи.

#### 2. Volatile Variable Rule

Запись в `volatile` переменную всегда _happens-before_ чтению из этой же переменной.

> _Результат:_ Это гарантирует «свежесть» данных. Как только поток записал значение в `volatile`, любой другой поток при чтении увидит это значение и **все остальные изменения**, сделанные до записи.

#### 3. Synchronized / Lock Rule

Освобождение монитора (выход из `synchronized` блока или `lock.unlock()`) _happens-before_ захвата этого же монитора другим потоком.

> _Результат:_ Всё, что сделал первый поток внутри критической секции, станет видимым второму потоку, когда тот войдет в свою критическую секцию.

#### 4. Thread Start & Join Rules

- Вызов `thread.start()` _happens-before_ любого действия в запущенном потоке.
    
- Любое действие в потоке _happens-before_ возврата из вызова `thread.join()` в другом потоке.
    

#### 5. Transitivity (Транзитивность)

Если $A \to B$ и $B \to C$, то $A \to C$.

> Это позволяет выстраивать цепочки видимости. Например: записали данные в обычное поле -> записали в volatile -> другой поток прочитал этот volatile -> он увидел и то обычное поле тоже.
