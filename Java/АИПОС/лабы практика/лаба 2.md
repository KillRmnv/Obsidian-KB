## 1. Packet Tracer - Identify MAC and IP Addresses (5.1.4.4)

### Цель
Научиться собирать PDU информацию в режиме симуляции и идентифицировать MAC и IP адреса устройств на каждом этапе передачи данных.

### Алгоритм выполнения

#### Часть 1: Сбор PDU информации

**Шаг 1: Подготовка к анализу**
- Откройте предложенный файл в Packet Tracer
- Определите компьютеры и их IP адреса из топологии
- Найдите компьютер с IP адресом 172.16.31.2

**Шаг 2: Отправка ping в Simulation режиме**
```
1. Кликните на компьютер 172.16.31.2
2. Перейдите на вкладку Desktop → Command Prompt
3. Переключитесь в Simulation режим (правый нижний угол)
4. Введите команду: ping 10.10.10.3
5. Нажмите Enter
```

**Шаг 3: Анализ первого PDU (на компьютере-источнике)**
```
1. В Simulation режиме появится PDU (цветной конверт)
2. Кликните на PDU
3. Перейдите на вкладку "Outbound PDU Layer"
4. Запишите информацию:
   - At Device: Computer (172.16.31.2)
   - Destination MAC Address: например, 00D0:BA8E:741A
   - Source MAC Address: например, 000C:85CC:1DA7
   - Source IP Address: 172.16.31.2
   - Destination IP Address: 10.10.10.3
```

**Шаг 4: Перемещение PDU по сети**
```
1. Нажмите кнопку "Capture / Forward" (стрелка вправо)
2. PDU переместится на следующее устройство
3. Снова кликните на PDU
4. Запишите информацию с "Outbound PDU Layer"
5. Повторяйте шаги 1-4, пока PDU не достигнет места назначения
```

**Шаг 5: Заполнение таблицы**

Создайте таблицу с колонками:
```
At Device | Dest. MAC | Src MAC | Src IPv4 | Dest IPv4
```

**Важно понимать**:
- **MAC адреса изменяются** на каждом устройстве L3 (маршрутизаторе)
- **IP адреса НЕ изменяются** на протяжении всего пути
- На коммутаторах (L2) MAC адреса остаются теми же

#### Часть 2: Вопросы для размышления

**Вопрос 1**: Изменялись ли MAC адреса на пути от источника к назначению?
- **Ответ**: Да, MAC адреса изменяются на каждом маршрутизаторе

**Вопрос 2**: Изменялись ли IP адреса?
- **Ответ**: Нет, IP адреса источника и назначения остаются постоянными

**Вопрос 3**: На каких устройствах изменяются MAC адреса?
- **Ответ**: На маршрутизаторах (Layer 3 устройствах)

### Теория, необходимая для выполнения
- Различие между Layer 2 (MAC) и Layer 3 (IP) адресацией
- Понятие PDU (Protocol Data Unit)
- Работа коммутаторов vs маршрутизаторов
- Inbound vs Outbound PDU

## 2. Packet Tracer - Examine the ARP Table (5.2.1.7)

### Цель
Изучить процесс ARP (Address Resolution Protocol), ARP запросы, таблицы ARP и MAC-адресов коммутаторов.

### Алгоритм выполнения

#### Часть 1: Анализ запроса ARP

**Шаг 1: Генерация ARP запросов**
```
1. Кликните на компьютер 172.16.31.2
2. Desktop → Command Prompt
3. Введите команду: arp -d  (очистка ARP таблицы)
4. Переключитесь в Simulation режим
5. Введите команду: ping 172.16.31.3
```

*Теория:* `arp -d` удаляет все записи из ARP таблицы. Это заставляет компьютер послать ARP запрос.

**Шаг 2: Наблюдение за ARP и ICMP пакетами**
```
В Simulation режиме появятся ДВА типа PDU:
1. ARP (обычно желтый или зеленый)
2. ICMP (обычно синий - ping пакеты)
```

**Шаг 3: Анализ ARP запроса (broadcast)**
```
1. Кликните на ARP PDU
2. Перейдите на вкладку "Outbound PDU Details"
3. Обратите внимание на Destination MAC:
   - Destination MAC Address: FFFF.FFFF.FFFF (broadcast)
   
Это значит, что ARP запрос отправляется ВСЕМ устройствам в локальной сети.
```

**Шаг 4: Продвижение ARP PDU**
```
1. Нажимайте "Capture / Forward"
2. Наблюдайте, как коммутатор отправляет ARP запрос на ВСЕ порты (кроме того, откуда пришел)
3. Вопрос: Сколько копий PDU создал коммутатор для ответа на ARP-запрос?
   Ответ: 1 (только устройство с правильным IP отвечает)
```

**Шаг 5: Проверка ARP таблицы**
```
1. Переключитесь обратно в Realtime режим
2. На компьютере 172.16.31.2 в Command Prompt введите:
   arp -a
3. Проверьте: появилась ли новая запись?
   Должна появиться запись с IP: 172.16.31.3 и его MAC адресом
```

#### Часть 2: Анализ таблицы MAC-адресов коммутатора

**Шаг 1: Просмотр MAC таблицы на коммутаторе**
```
1. Кликните на Switch1
2. Перейдите на вкладку CLI
3. Войдите в привилегированный режим:
   Switch> enable
   Switch#
4. Введите команду:
   Switch# show mac-address-table
```

*Теория:* Коммутатор "учится" MAC адресам, наблюдая за трафиком, проходящим через его порты.

**Шаг 2: Анализ записей**
```
Таблица показывает:
- VLAN (обычно VLAN 1 по умолчанию)
- Mac Address (MAC адрес устройства)
- Type (DYNAMIC - выучен автоматически, STATIC - настроен вручную)
- Ports (на каком порту находится устройство)
```

#### Часть 3: Анализ процесса ARP в удаленных подключениях

**Шаг 1: Ping через маршрутизатор**
```
1. На компьютере 172.16.31.2 очистите ARP таблицу: arp -d
2. Переключитесь в Simulation режим
3. Введите команду: ping 10.10.10.1  (адрес за маршрутизатором)
4. Сколько пакетов PDU появилось? 
   Ответ: 2 (ARP + ICMP)
```

**Шаг 2: Анализ ARP для шлюза по умолчанию**
```
Когда компьютер пингует адрес в другой сети:
1. Компьютер проверяет, что IP назначения НЕ в его сети
2. Компьютер отправляет ARP запрос для ШЛЮЗА ПО УМОЛЧАНИЮ (маршрутизатора)
3. Получает MAC адрес маршрутизатора
4. Отправляет ICMP пакет маршрутизатору
```

**Шаг 3: Проверка ARP таблицы маршрутизатора**
```
1. Кликните на Router1
2. CLI → enable
3. Введите команду:
   Router# show ip arp
   
Вы увидите MAC адреса устройств, с которыми маршрутизатор общался.
```

### Важные команды

```
Команда             | Где использовать  | Назначение
--------------------|-------------------|----------------------------------
arp -a              | ПК (Command Prompt) | Просмотр ARP таблицы
arp -d              | ПК (Command Prompt) | Очистка ARP таблицы
show mac-address-table | Коммутатор (CLI) | Просмотр таблицы MAC адресов
show ip arp         | Маршрутизатор (CLI) | Просмотр ARP таблицы маршрутизатора
```

### Теория, необходимая для выполнения
- Протокол ARP и его назначение
- Broadcast адрес (FF:FF:FF:FF:FF:FF)
- Как коммутатор строит таблицу MAC адресов
- Шлюз по умолчанию (default gateway)
- Различие между локальной и удаленной сетью

## 3. Packet Tracer - Configure Layer 3 Switches (5.3.3.5)

### Цель
Заменить маршрутизатор и коммутатор на Layer 3 коммутатор (многоуровневый коммутатор).

### Алгоритм выполнения

#### Часть 1: Документирование текущей конфигурации сети

**Шаг 1: Сбор информации с маршрутизатора R1**
```
1. Кликните на R1
2. Перейдите на вкладку CLI
3. Введите команды:
   Router> enable
   Router# show running-config
   
   или сокращенно:
   Router# show run
```

**Шаг 2: Заполнение таблицы адресации**
```
Найдите в конфигурации секции:
interface GigabitEthernet0/0
 ip address 172.16.31.1 255.255.255.0
 
interface GigabitEthernet0/1
 ip address 192.168.0.2 255.255.255.0

Запишите эти адреса в таблицу.
```

#### Часть 2: Конфигурация, развертывание и тестирование нового многоуровневого коммутатора

**Шаг 1: Конфигурация MLSw1 с адресацией из R1**

```
1. Кликните на MLSw1
2. Перейдите на вкладку CLI
3. Войдите в режим конфигурации:

Switch> enable
Switch# configure terminal
Switch(config)#
```

**Шаг 2: Конфигурация GigabitEthernet 0/1 (WAN интерфейс)**

*ВАЖНО:* По умолчанию все порты коммутатора работают в режиме коммутации (Layer 2). Чтобы назначить IP адрес, нужно перевести порт в режим маршрутизации (Layer 3).

```
Switch(config)# interface gigabitEthernet 0/1
Switch(config-if)# no switchport    ← Переводит порт в режим маршрутизации (Layer 3)
Switch(config-if)# ip address 192.168.0.2 255.255.255.0
Switch(config-if)# no shutdown
Switch(config-if)# exit
```

*Теория:* Команда `no switchport` делает интерфейс роутированным (Layer 3). Теперь на нём можно настроить IP адрес.

**Шаг 3: Конфигурация VLAN 1 (LAN интерфейс)**

```
Switch(config)# interface vlan 1
Switch(config-if)# ip address 172.16.31.1 255.255.255.0
Switch(config-if)# no shutdown
Switch(config-if)# exit
```

*Теория:* VLAN 1 - это виртуальный интерфейс коммутатора, используемый для управления.

**Шаг 4: Сохранение конфигурации**
```
Switch(config)# exit
Switch# copy running-config startup-config
или:
Switch# write memory
```

#### Часть 3: Развертывание нового оборудования

**Шаг 1: Удаление старого оборудования**

*ВАЖНО:* Эти шаги обычно выполняются в нерабочее время.

```
1. Кликните на пустую область, чтобы отменить выбор
2. Используйте инструмент Delete (красный крестик)
3. Удалите все соединения между устройствами
   ИЛИ
   Просто удалите R1, S1 и S2 полностью
```

**Шаг 2: Подключение MLSw1**

```
1. Выберите инструмент Connections (молния)
2. Выберите подходящий кабель (Copper Straight-Through)
3. Подключите:
   - MLSw1 GigabitEthernet 0/1 ↔ Edge GigabitEthernet 0/0
   - PC1 FastEthernet0 ↔ MLSw1 FastEthernet (любой порт, например 0/1)
   - PC2 FastEthernet0 ↔ MLSw1 FastEthernet (например 0/2)
   - PC3 FastEthernet0 ↔ MLSw1 FastEthernet (например 0/3)
   - PC4 FastEthernet0 ↔ MLSw1 FastEthernet (например 0/4)
```

**Шаг 3: Проверка связности**

*ВАЖНО:* Подождите, пока оранжевые индикаторы связи станут зелеными.

```
1. Кликните на любой PC (например, PC1)
2. Desktop → Command Prompt
3. Введите команду:
   ping 192.168.0.1  (IP адрес Edge маршрутизатора)
4. Повторите для остальных ПК
5. Все пинги должны быть успешными
```

### Полная последовательность команд для MLSw1

### Теория, необходимая для выполнения
- Различие между Layer 2 и Layer 3 коммутаторами
- Команда `no switchport` и её назначение
- VLAN 1 как управляющий интерфейс
- Преимущества Layer 3 коммутатора перед маршрутизатором

## 4. Packet Tracer - Configuring IPv6 Addressing (8.2.5.3)

### Цель
Настроить IPv6 адресацию на маршрутизаторе, серверах и клиентах.

### Алгоритм выполнения

#### Часть 1: Конфигурация IPv6 адресации на маршрутизаторе

**Шаг 1: Включение IPv6 маршрутизации (КРИТИЧНО!)**

*ОЧЕНЬ ВАЖНО:* Эта команда ОБЯЗАТЕЛЬНА, чтобы маршрутизатор мог пересылать IPv6 пакеты.

```
R1> enable
R1# configure terminal
R1(config)# ipv6 unicast-routing    ← БЕЗ этой команды IPv6 НЕ будет работать!
```

*Теория:* По умолчанию маршрутизаторы Cisco НЕ пересылают IPv6 пакеты. Команда `ipv6 unicast-routing` включает эту функцию.

**Шаг 2: Конфигурация GigabitEthernet0/0**

```
R1(config)# interface gigabitEthernet 0/0
R1(config-if)# ipv6 address 2001:DB8:1:1::1/64
R1(config-if)# ipv6 address FE80::1 link-local
R1(config-if)# no shutdown
R1(config-if)# exit
```

*Теория:* 
- `2001:DB8:1:1::1/64` - это глобальный unicast адрес IPv6
- `FE80::1` - это link-local адрес (используется для локальной связи)
- `/64` означает, что первые 64 бита - сетевая часть

**Шаг 3: Конфигурация GigabitEthernet0/1**

```
R1(config)# interface gigabitEthernet 0/1
R1(config-if)# ipv6 address 2001:DB8:1:2::1/64
R1(config-if)# ipv6 address FE80::1 link-local
R1(config-if)# no shutdown
R1(config-if)# exit
```

**Шаг 4: Конфигурация Serial0/0/0**

```
R1(config)# interface serial 0/0/0
R1(config-if)# ipv6 address 2001:DB8:1:A001::2/64
R1(config-if)# ipv6 address FE80::1 link-local
R1(config-if)# no shutdown
R1(config-if)# exit
R1(config)# exit
```

**Шаг 5: Проверка конфигурации IPv6**

```
R1# show ipv6 interface brief

Вы должны увидеть все интерфейсы со статусом up/up:
GigabitEthernet0/0    [up/up]
  FE80::1
  2001:DB8:1:1::1
GigabitEthernet0/1    [up/up]
  FE80::1
  2001:DB8:1:2::1
Serial0/0/0           [up/up]
  FE80::1
  2001:DB8:1:A001::2
```

#### Часть 2: Конфигурация IPv6 адресации на серверах

**Шаг 1: Конфигурация Accounting Server**

```
1. Кликните на Accounting Server
2. Перейдите на вкладку Desktop
3. Кликните на IP Configuration
4. В разделе IPv6 Configuration выберите Static
5. Введите:
   IPv6 Address: 2001:DB8:1:1::4
   Prefix Length: 64   (НЕ путать с /64 - здесь вводится только число)
   IPv6 Gateway: FE80::1   (link-local адрес маршрутизатора)
```

*Теория:* В качестве шлюза (gateway) используется link-local адрес маршрутизатора, а не глобальный адрес.

**Шаг 2: Конфигурация CAD Server**

```
1. Кликните на CAD Server
2. Desktop → IP Configuration
3. IPv6 Configuration: Static
4. Введите:
   IPv6 Address: 2001:DB8:1:2::4
   Prefix Length: 64
   IPv6 Gateway: FE80::1
```

```
Для Sales:
1. Кликните на Sales

   Prefix Length: 64
   IPv6 Gateway: FE80::1

Для Billing:

3. IPv6 Address: 2001:DB8:1:1::3
   Prefix Length: 64
   IPv6 Gateway: FE80::1
```

**Шаг 2: Конфигурация Engineering и Design (в сети 2001:DB8:1:2::)**

```
Для Engineering:
1. Кликните на Engineering
2. Desktop → IP Configuration → Static
3. IPv6 Address: 2001:DB8:1:2::3
   Prefix Length: 64
   IPv6 Gateway: FE80::1

Для Design:
1. Кликните на Design

   Prefix Length: 64
```

#### Часть 4: Тестирование и проверка сетевой связности

**Шаг 1: Открытие веб-страниц серверов**

```
1. Кликните на Sales
2. Desktop → Web Browser
3. В строке URL введите: 2001:DB8:1:1::4  (IP Accounting Server)
4. Нажмите Go
5. Должна открыться веб-страница Accounting
6. Теперь введите: 2001:DB8:1:2::4  (IP CAD Server)
7. Должна открыться веб-страница CAD
8. Повторите для остальных клиентов
```

**Шаг 2: Ping к ISP**

```
1. Откройте любой клиент (например, Sales)
2. Desktop → Command Prompt
3. Введите команду:
   ping 2001:DB8:1:A001::1   (IPv6 адрес ISP)
4. Должны приходить ответы (Reply)
5. Повторите на других клиентах для полной проверки
```


### Важные команды для IPv6

```
Команда                          | Назначение
---------------------------------|---------------------------------------
ipv6 unicast-routing             | Включить IPv6 маршрутизацию (ОБЯЗАТЕЛЬНО)
ipv6 address [адрес]/[префикс]   | Назначить IPv6 адрес интерфейсу
ipv6 address [адрес] link-local  | Назначить link-local адрес
show ipv6 interface brief        | Показать все IPv6 интерфейсы
ping [IPv6 адрес]                | Проверить связность по IPv6
```

### Теория, необходимая для выполнения
- Формат IPv6 адресов (двоеточие, группы по 4 hex цифры)
- Link-local адреса (FE80::) и их назначение
- Префикс /64 в IPv6
- Зачем нужна команда `ipv6 unicast-routing`
- Различие между глобальным unicast и link-local адресами

## 5. Packet Tracer - Verifying IPv4 and IPv6 Addressing (8.3.2.5)

### Цель
Изучить dual-stack конфигурацию (IPv4 + IPv6 на одной сети) и научиться проверять адресацию обоих протоколов.

### Алгоритм выполнения

#### Часть 1: Завершение документации таблицы адресации

**Шаг 1: Использование ipconfig для проверки IPv4 адресации**

```
1. Кликните на PC1
2. Desktop → Command Prompt
3. Введите команду:
   ipconfig /all
   
4. Запишите в таблицу:
   - IPv4 Address: например, 10.10.1.100
   - Subnet Mask: например, 255.255.255.224
   - Default Gateway: например, 10.10.1.97
```

*Теория:* Команда `ipconfig /all` показывает полную информацию о сетевой конфигурации IPv4.

## 9. Packet Tracer - Subnetting Scenario 1 (9.1.4.6)

- Связь между R1 и R2 требует IP адреса на каждом конце

### Алгоритм выполнения

#### Часть 1: Проектирование схемы IP адресации

**Шаг 1: Разделение сети 192.168.100.0/24 на подсети**

**Вопрос a**: На основе топологии, сколько подсетей необходимо?
```

- LAN подключенная к R1 G0/0 → подсеть 1
- LAN подключенная к R1 G0/1 → подсеть 2
- LAN подключенная к R2 G0/0 → подсеть 3
- LAN подключенная к R2 G0/1 → подсеть 4
- WAN связь между R1 и R2 → подсеть 5

Ответ: 5 подсетей
```

**Вопрос b**: Сколько битов нужно заимствовать для поддержки количества подсетей?
```
Формула: 2^n ≥ количество подсетей
2^1 = 2 (недостаточно)
2^2 = 4 (недостаточно)
```

```
Количество адресов хостов: 2^5 = 32
Используемых адресов: 32 - 2 = 30 (вычитаем адрес сети и broadcast)
```

### Ключевые концепции

**MAC и IP адресация**:
- MAC адреса изменяются на маршрутизаторах (L3)
- IP адреса НЕ изменяются на всем пути
- ARP связывает IP с MAC адресами

**ARP**:
- ARP запрос отправляется broadcast (FF:FF:FF:FF:FF:FF)
- Только устройство с правильным IP отвечает
- Коммутатор пересылает broadcast на все порты

**IPv6**:
- Обязательна команда `ipv6 unicast-routing`
- Link-local адреса (FE80::) используются как шлюзы
- Префикс /64 стандартный для LAN
- Двоеточие `:` разделяет группы
- `::` сокращает последовательные нули

**Layer 3 коммутатор**:
- Команда `no switchport` делает порт роутированным
- VLAN 1 - управляющий интерфейс
- Может заменить маршрутизатор + коммутатор

**Dual-stack**:
- IPv4 и IPv6 работают одновременно
- Разные команды: ipconfig vs ipv6config
- Разные протоколы ping и tracert

**Subnetting**:
- Заимствование битов: 2^n = количество подсетей
- Хосты: 2^h - 2
- Размер блока: 256 - значение октета маски
- Первый используемый: Network + 1
- Последний используемый: Broadcast - 1

**Troubleshooting**:
1. Тестирование (ping)
2. Локализация (tracert)
3. Анализ (show команды)
4. Решение (исправление конфигурации)
5. Проверка (повторный ping)
