## 1. Packet Tracer - Representing the Network (1.2.4.4)

### Цель
Ознакомиться с интерфейсом Packet Tracer, научиться переключаться между режимами отображения сети и наблюдать за работой сетевых устройств.

### Алгоритм выполнения

#### Часть 1: Обзор программы Packet Tracer
**Шаг 1: Изучение компонентов сети**
- Откройте предложенный файл в Packet Tracer
- В левом нижнем углу найдите панель Icon Toolbar (иконки устройств)
- Определите категории: End Devices, Intermediate Devices, Connections, Multiuser Connection

**Шаг 2: Работа с топологией сети**
- Изучите предоставленную топологию сети
- Определите количество компьютеров, серверов, коммутаторов и маршрутизаторов
- Разберитесь с типами кабелей, используемых в сети

#### Часть 2: Переключение между режимами

**Шаг 1: Realtime и Simulation режимы**
- В правом нижнем углу найдите переключатель **Realtime** / **Simulation**
- В режиме Realtime сеть работает как реальная сеть в реальном времени
- В режиме Simulation можно наблюдать пакеты и изучать их маршруты медленнее

*Выполните:*
```
Realtime режим → Simulation режим → нажмите Auto Capture/Play
```

**Шаг 2: Изучение PDU (Packet Data Units)**
- В Simulation режиме нажмите **Auto Capture / Play**
- Наблюдайте за цветными конвертами (пакеты) между устройствами
- Кликните на пакет, чтобы увидеть детали (Inbound/Outbound PDU Details)

#### Часть 3: Логический и физический виды

**Шаг 1: Переключение между видами**
- В центральной части интерфейса найдите вкладку **Logical** (по умолчанию)
- Рядом находится вкладка **Physical** 
- Кликните на **Physical**, чтобы увидеть физическое расположение устройств

**Шаг 2: Работа с физическим видом**
- В Physical режиме видны физические места расположения коммутаторов
- Используйте zoom для приближения/отдаления
- Вернитесь в Logical режим

### Теория, необходимая для выполнения
- Концепция LAN (Local Area Network)
- Концепция WAN (Wide Area Network)
- Модель OSI и PDU в разных слоях
- Основные типы кабелей (Ethernet, Serial, Fiber Optic)

## 2. Packet Tracer - Navigating the IOS (2.1.4.8)

### Цель
Освоить навигацию по Cisco IOS, различные режимы доступа, работу со справкой и базовые команды.

### Алгоритм выполнения

#### Часть 1: Базовое подключение и работа с CLI

**Шаг 1: Подключение консольного кабеля**
```
1. Кликните на иконку Connections (молния) внизу слева
2. Выберите светло-голубой консольный кабель (Console)
3. Кликните на PC1 → выберите RS-232
4. Потяните кабель к коммутатору S1 → выберите Console порт
```

**Шаг 2: Открытие терминального сеанса**
```
1. Кликните на PC1 → вкладка Desktop
2. Дважды кликните на Terminal
3. Проверьте: Bits per second = 9600 → OK
4. Нажмите ENTER
```

**Шаг 3: Изучение справки по IOS**
```
Команды для изучения справки:
S1> ?                    # Показывает все доступные команды в User EXEC режиме
S1> t?                   # Показывает команды, начинающиеся на 't' (telnet, terminal, traceroute)
S1> te?                  # Уточняет команды на 'te' (telnet, terminal)
```

**Очень важно:** Вопросительный знак (?) работает в любом режиме и показывает доступные команды или параметры.

#### Часть 2: Изучение режимов EXEC

**Шаг 1: Переход в привилегированный режим (Privileged EXEC)**
```
S1> enable               # Переход в привилегированный режим
S1#                      # Символ # указывает на привилегированный режим
```

*Теория:* User EXEC режим (>) имеет ограниченный набор команд для мониторинга. Privileged EXEC режим (#) позволяет настраивать устройство.

**Шаг 2: Переход в режим глобальной конфигурации**
```
S1# configure terminal
# или сокращенно:
S1# conf t
S1(config)#              # Prompt изменился - вы в режиме конфигурации
```

*Теория:* Mode Global Configuration используется для настройки параметров, которые применяются ко всему устройству.

**Шаг 3: Возврат из режимов**
```
S1(config)# exit         # Вернуться на уровень выше
# или
S1(config)# end
# или
Ctrl + Z                 # Быстрый выход в привилегированный режим
```

#### Часть 3: Команда clock (настройка времени)

**Шаг 1: Просмотр текущего времени**
```
S1# show clock           # Отображает текущее время на устройстве
```

**Шаг 2: Установка времени**
```
S1# clock ?              # Справка по команде clock
S1# clock set ?          # Показывает требуемый формат: hh:mm:ss
S1# clock set 15:30:00 ? # Запрашивает день месяца
S1# clock set 15:30:00 25 January 2024
# или
S1# clock set 15:30:00 25 Jan 2024
```

*Теория:* Используется 24-часовой формат времени. Формат даты: день, месяц, год.

**Шаг 3: Проверка установки**
```
S1# show clock          # Проверяет установленное время
```

### Важные команды для справки
```
Команда          | Назначение
-----------------+-----------------------------------------
?                | Справка (любой режим)
en <TAB>         | Автодополнение команды (нажать TAB)
enable           | Переход в привилегированный режим
configure terminal | Переход в режим глобальной конфигурации
exit             | Выход на уровень выше
end              | Выход в привилегированный режим
Ctrl + Z         | Выход в привилегированный режим
show clock       | Просмотр времени устройства
```

### Теория, необходимая для выполнения
- Иерархия режимов IOS (User EXEC → Privileged EXEC → Config modes)
- Контекстная справка (?)
- Автодополнение команд (TAB)
- Сообщения об ошибках IOS

## 3. Packet Tracer - Configuring Initial Switch Settings (2.2.3.3)

### Цель
Научиться выполнять базовую конфигурацию коммутатора: задавать имя, настраивать пароли, создавать MOTD и сохранять конфигурацию.

### Алгоритм выполнения

#### Часть 1: Проверка текущей конфигурации

**Шаг 1: Вход в привилегированный режим**
```
Switch> enable          # Вход в привилегированный режим
Switch#
```

**Шаг 2: Просмотр текущей конфигурации**
```
Switch# show running-config    # Полная текущая конфигурация (находится в RAM)
Switch# show startup-config    # Сохраненная конфигурация (находится в NVRAM)
# или сокращенно:
Switch# show run
Switch# show start
```

#### Часть 2: Базовая конфигурация коммутатора

**Шаг 1: Присвоение имени коммутатору**
```
Switch# configure terminal
Switch(config)# hostname S1     # Задание имени коммутатору
S1(config)# exit
S1#
```

*Теория:* Имя устройства помогает идентифицировать его в сети. Это отражается в приглашении командной строки.

**Шаг 2: Защита консольного порта паролем**
```
S1# configure terminal
S1(config)# line console 0      # Вход в режим конфигурации консольной линии
S1(config-line)# password letmein  # Установка пароля
S1(config-line)# login          # Активация запроса пароля при входе (ВАЖНО!)
S1(config-line)# exit
S1(config)# exit
S1#
```

*Теория:* Без команды `login` пароль не будет проверяться. `line console 0` — это физический консольный порт коммутатора.

**Шаг 3: Защита привилегированного режима паролем (Enable Password)**
```
S1# configure terminal
S1(config)# enable password c1$c0    # Установка простого пароля привилегированного режима
S1(config)# exit
S1#
```

*Теория:* Этот пароль устаревший и хранится в открытом виде. Используйте вместо него `enable secret`.

**Шаг 4: Защита привилегированного режима шифрованным паролем (Enable Secret)**
```
S1# configure terminal
S1(config)# enable secret itsasecret    # Зашифрованный пароль (тип 5)
S1(config)# exit
S1#
```

*Теория:* Если настроены оба пароля, `enable secret` имеет приоритет. Все пароли используют одноразовое хеширование.

#### Часть 3: Шифрование открытых паролей

**Шаг 1: Включение шифрования всех паролей**
```
S1# configure terminal
S1(config)# service password-encryption    # Шифрует все существующие и будущие пароли
S1(config)# exit
S1#
```

*Теория:* После этой команды все пароли (console, vty, enable) отображаются в зашифрованном виде в файле конфигурации.

#### Часть 4: Конфигурация MOTD (Message of the Day)

**Шаг 1: Создание MOTD баннера**
```
S1# configure terminal
S1(config)# banner motd #This is a secure system. Authorized Access Only!#
# или с другим разделителем:
S1(config)# banner motd "This is a secure system. Authorized Access Only!"
S1(config)# exit
S1#
```

*Теория:* Разделители (# или ") должны быть одинаковыми в начале и конце. Выбирайте разделитель, которого нет в тексте сообщения.

#### Часть 5: Сохранение конфигурации

**Шаг 1: Просмотр текущей конфигурации**
```
S1# show running-config        # Просмотр того, что находится в RAM
```

**Шаг 2: Сохранение в NVRAM (постоянную память)**
```
S1# copy running-config startup-config
# Destination filename [startup-config]? [ENTER]
# или сокращенно:
S1# copy run start
# или:
S1# wr              # write memory
```

*Теория:* `Running-config` находится в RAM и теряется при перезагрузке. `Startup-config` находится в NVRAM и сохраняется после перезагрузки.

**Шаг 3: Проверка сохраненной конфигурации**
```
S1# show startup-config        # Проверяет сохраненную конфигурацию
```

#### Часть 6: Конфигурация второго коммутатора (S2)

Повторите все шаги 1-5 для коммутатора S2, используя те же или другие пароли/имена в зависимости от требований.

### Полная последовательность команд для S1

```
Switch> enable
Switch# configure terminal
Switch(config)# hostname S1
S1(config)# line console 0
S1(config-line)# password letmein
S1(config-line)# login
S1(config-line)# exit
S1(config)# enable password c1$c0
S1(config)# enable secret itsasecret
S1(config)# service password-encryption
S1(config)# banner motd #This is a secure system. Authorized Access Only!#
S1(config)# exit
S1# copy running-config startup-config
```

### Теория, необходимая для выполнения
- Иерархия режимов конфигурации
- Различие между `enable password` и `enable secret`
- Назначение `line console 0` и `line vty 0 4`
- Различие между `running-config` (RAM) и `startup-config` (NVRAM)
- Назначение `login` команды

## 4. Packet Tracer - Skills Integration Challenge (2.4.1.2)

### Цель
Интеграция всех полученных навыков: конфигурация двух коммутаторов с присвоением IP-адресов и проверка связности между ПК.

### Алгоритм выполнения

#### Часть 1: Конфигурация коммутатора ASw-1

**Шаг 1: Подключение и вход**
```
Switch> enable
Switch# configure terminal
Switch(config)# hostname ASw-1        # Имя из таблицы адресации
```

**Шаг 2: Защита консольной линии**
```
ASw-1(config)# line console 0
ASw-1(config-line)# password R4Xe3    # Пароль для всех линий
ASw-1(config-line)# login
ASw-1(config-line)# exit
```

**Шаг 3: Защита VTY линий (удаленный доступ Telnet/SSH)**
```
ASw-1(config)# line vty 0 4           # Конфигурирует линии VTY 0-4
ASw-1(config-line)# password R4Xe3
ASw-1(config-line)# login
ASw-1(config-line)# exit
```

*Теория:* VTY линии (Virtual TeleTYpe) используются для удаленного доступа к устройству через Telnet или SSH. `line vty 0 4` означает линии с 0 по 4 (5 линий).

**Шаг 4: Установка enable secret**
```
ASw-1(config)# enable secret C4aJa      # Зашифрованный пароль привилегированного режима
```

**Шаг 5: Шифрование всех паролей**
```
ASw-1(config)# service password-encryption
```

**Шаг 6: Конфигурация MOTD с ключевым словом "warning"**
```
ASw-1(config)# banner motd #warning#    # Обязательно содержит слово "warning"
```

**Шаг 7: Назначение IP-адреса на VLAN 1**
```
ASw-1(config)# interface vlan 1         # Вход в режим конфигурации виртуального интерфейса VLAN 1
ASw-1(config-if)# ip address 10.10.10.100 255.255.255.0  # IP адрес и маска из таблицы
ASw-1(config-if)# no shutdown           # Включение интерфейса
ASw-1(config-if)# exit
ASw-1(config)# exit
```

*Теория:* VLAN 1 — это управляющий VLAN коммутатора. Это логический интерфейс, используемый для удаленного управления. Команда `no shutdown` включает интерфейс (интерфейсы выключены по умолчанию).

**Шаг 8: Сохранение конфигурации**
```
ASw-1# copy running-config startup-config
```

#### Часть 2: Конфигурация коммутатора ASw-2

Повторите все шаги для ASw-2, используя другой IP-адрес из таблицы адресации (например, 10.10.10.150).

```
Switch> enable
Switch# configure terminal
Switch(config)# hostname ASw-2
ASw-2(config)# line console 0
ASw-2(config-line)# password R4Xe3
ASw-2(config-line)# login
ASw-2(config-line)# exit
ASw-2(config)# line vty 0 4
ASw-2(config-line)# password R4Xe3
ASw-2(config-line)# login
ASw-2(config-line)# exit
ASw-2(config)# enable secret C4aJa
ASw-2(config)# service password-encryption
ASw-2(config)# banner motd #warning#
ASw-2(config)# interface vlan 1
ASw-2(config-if)# ip address 10.10.10.150 255.255.255.0
ASw-2(config-if)# no shutdown
ASw-2(config-if)# exit
ASw-2(config)# exit
ASw-2# copy running-config startup-config
```

#### Часть 3: Конфигурация ПК

**Шаг 1: Конфигурация PC1**
```
1. Кликните на PC1
2. Вкладка Desktop → IP Configuration
3. Выберите Static (статический адрес)
4. IP Address: 10.10.10.1 (из таблицы адресации)
5. Subnet Mask: 255.255.255.0
```

**Шаг 2: Конфигурация PC2**
```
1. Кликните на PC2
2. Вкладка Desktop → IP Configuration
3. Выберите Static
4. IP Address: 10.10.10.2 (из таблицы адресации)
5. Subnet Mask: 255.255.255.0
```

*Теория:* Все ПК и коммутаторы должны быть в одной подсети (10.10.10.0/24) для работы.

#### Часть 4: Проверка связности

**Шаг 1: Проверка с помощью ping**
```
1. Кликните на PC1 → вкладка Desktop
2. Откройте Command Prompt (Командная строка)
3. Введите: ping 10.10.10.2
   или: ping 10.10.10.150 (IP коммутатора ASw-2)
```

*Теория:* Если ping успешен (Reply from...), это означает, что есть сетевая связность между устройствами.

**Шаг 2: Проверка конфигурации коммутатора**
```
1. Кликните на коммутатор ASw-1 → вкладка CLI
2. Введите: show running-config
3. Проверьте наличие всех настроек: hostname, пароли, IP, MOTD
```

### Важные отличия от предыдущих лабораторных

- В этой лабораторной нужно конфигурировать **две линии**: console (0) и vty (0 4)
- Нужно присвоить **IP-адреса** на интерфейсах VLAN
- Нужно включить интерфейсы командой **no shutdown**
- Нужно проверить связность с помощью **ping**

### Теория, необходимая для выполнения
- Назначение VTY линий и удаленного доступа
- Концепция VLAN и интерфейса VLAN 1
- IP-адресация и маски подсети
- Команда `no shutdown` для включения интерфейсов
- Использование ping для проверки связности

## 5. Packet Tracer - Exploring Internetworking Devices (6.3.1.10)

### Цель
Изучить физические характеристики маршрутизаторов и коммутаторов, модули расширения, типы кабелей и подключение устройств.

### Алгоритм выполнения

#### Часть 1: Идентификация физических характеристик устройств

**Шаг 1: Исследование маршрутизатора East**

1. Кликните на маршрутизатор **East**
2. Перейдите на вкладку **Physical** (не Configuration)
3. Используйте кнопки Zoom In / Zoom Out для приближения

**Шаг 2: Определение портов управления**
```
Вопрос: Какие порты управления доступны?
Ответ: AUX (Auxiliary) и Console порты

Команда для проверки:
East# show ip interface brief    # Показывает все интерфейсы
```

*Теория:* **Console** порт используется для прямого подключения кабелем (обычно при первой настройке). **AUX** порт был исторически использован для модема.

**Шаг 3: Определение LAN и WAN интерфейсов**
```
Маршрутизаторы обычно имеют:
- LAN интерфейсы: GigabitEthernet (для локальной сети)
- WAN интерфейсы: Serial (для глобальной сети)

Команда для проверки:
East> show ip interface brief
East> show interface gigabitethernet 0/0    # Информация о GigaEthernet
East> show interface serial 0/0/0           # Информация о Serial
```

*Теория:* Bandwidth на GigabitEthernet обычно 1000000 Kbit (1 Gbps). Bandwidth на Serial интерфейсах (например, 1544 Kbit) используется алгоритмами маршрутизации, а не обозначает реальную скорость.

**Шаг 4: Определение слотов расширения**
```
В Physical режиме посчитайте открытые слоты для модулей:
- East маршрутизатор: обычно 1 слот
- Switch2 и Switch3: обычно 5 слотов каждый
```

#### Часть 2: Выбор и установка модулей расширения

**Шаг 1: Определение требуемых модулей**

Задача: Подключить 3 ПК к маршрутизатору East (нет денег на новый коммутатор)

1. Кликните на East → вкладка **Physical**
2. На левой панели найдите **Modules**
3. Кликните на каждый модуль, чтобы увидеть описание:
   - **HWIC-4ESW** — модуль Ethernet коммутатора (4 порта Fast Ethernet)

*Теория:* HWIC = High-speed WAN Interface Card. Некоторые HWIC содержат встроенные Ethernet коммутаторы.

**Шаг 2: Выбор модулей для подключения к другим устройствам**

Для подключения Switch2 к Switch3 через оптоволокно:
- Требуется модуль **PTSWITCH-NM-1FGE** (1 порт Gigabit Fiber Optic)

**Шаг 3: Установка модулей (важно!)**

```
1. Кликните на маршрутизатор/коммутатор
2. Посмотрите на Physical режиме расположение слотов
3. Если видите сообщение "Cannot add module when power is on":
   - Отключите устройство кнопкой Power (справа от логотипа)
   - Перетащите модуль в пустой слот
   - Включите устройство обратно
4. Если нужно удалить неправильный модуль:
   - Перетащите его вниз на его изображение в нижней части окна
```

*Теория:* Интерфейсы не являются горячозаменяемыми (hot-swappable), поэтому нужно выключать устройство.

**Шаг 4: Проверка установки модулей**

```
После установки модуля введите:
East# show ip interface brief

Вы должны увидеть новые интерфейсы:
```

#### Часть 3: Подключение устройств кабелями

**Шаг 1: Выбор типов кабелей**

```
Тип кабеля          | Когда использовать
--------------------|----------------------------------------
Copper Straight     | Между разными типами устройств:
-Through            | - Маршрутизатор ↔ Коммутатор
                    | - Маршрутизатор ↔ ПК
                    | - Коммутатор ↔ ПК
                    |
Copper Cross-Over   | Между одинаковыми устройствами:
                    | - Коммутатор ↔ Коммутатор
                    | - ПК ↔ ПК
                    | (в новых коммутаторах часто работает Straight)
                    |
Serial DCE/DTE      | Между маршрутизаторами через Serial:
                    | - Маршрутизатор ↔ Маршрутизатор (WAN)
                    |
Fiber Optic         | Оптоволоконные соединения между коммутаторами
```

*Теория:* Straight-Through кабель используется между разными уровнями (например, коммутатор и маршрутизатор — разные уровни). Cross-Over используется между одинаковыми уровнями.

**Шаг 2: Подключение устройств**

Для каждого соединения выполните:

```
1. Кликните на иконку Connections (молния) внизу слева
2. Выберите нужный тип кабеля (см. таблицу выше)
3. Кликните на первое устройство
4. Выберите нужный интерфейс из списка
5. Кликните на второе устройство
6. Выберите его интерфейс
7. Если соединение правильное, загорится зеленый индикатор и баллы увеличатся
```

**Шаг 3: Пример подключения East → Switch1**

```
1. Выберите "Copper Straight-Through"
2. Кликните East → выберите GigabitEthernet0/0
3. Кликните Switch1 → выберите GigabitEthernet1/1
4. Должно появиться сообщение об увеличении очков
```

## Общие советы по экзамену

### Важные команды для всех лабораторных работ

```
Режимы доступа:
  ?                          # Справка (работает везде)
  enable                     # Вход в привилегированный режим
  configure terminal         # Вход в режим конфигурации
  exit / end                 # Выход из режима
  Ctrl + Z                   # Быстрый выход в привилегированный режим

Просмотр информации:
  show running-config        # Текущая конфигурация (RAM)
  show startup-config        # Сохраненная конфигурация (NVRAM)
  show ip interface brief    # Все интерфейсы устройства
  show interface [name]      # Информация об интерфейсе
  show clock                 # Текущее время на устройстве

Сохранение:
  copy running-config startup-config  # Сохраняет конфигурацию
  write memory                        # Альтернатива (wr)

Пароли и безопасность:
  hostname [name]                # Задание имени устройства
  line console 0                 # Конфигурация консольной линии
  line vty 0 4                   # Конфигурация VTY линий
  password [pass]                # Установка пароля
  login                          # Включение проверки пароля
  enable password [pass]         # Простой пароль привилегированного режима
  enable secret [pass]           # Зашифрованный пароль привилегированного режима
  service password-encryption    # Шифрование всех паролей

Конфигурация интерфейсов:
  interface vlan 1               # Конфигурация VLAN интерфейса
  ip address [IP] [маска]        # Назначение IP адреса
  no shutdown                    # Включение интерфейса
  shutdown                       # Отключение интерфейса

Баннеры:
  banner motd #[текст]#         # Сообщение при входе
  banner login #[текст]#        # Сообщение перед паролем

Часы:
  clock set [hh:mm:ss] [день] [месяц] [год]  # Установка времени
```

### Последовательность вопросов на экзамене может быть такой:

1. **Где находится консольный порт маршрутизатора?** → На маршрутизаторе (Physical вид, RJ-45 порт помеченный как "Console")

2. **Как перейти в привилегированный режим?** → `enable`

3. **Как войти в режим конфигурации?** → `configure terminal` или `conf t`

4. **Какая команда для установки пароля на консольный порт?** → `line console 0` → `password [пароль]` → `login`

5. **Как зашифровать пароли?** → `service password-encryption`

6. **Как установить IP адрес на VLAN 1?** → `interface vlan 1` → `ip address [IP] [маска]` → `no shutdown`

7. **Какая разница между running-config и startup-config?** → Running в RAM (теряется при перезагрузке), Startup в NVRAM (сохраняется)

8. **Как сохранить конфигурацию?** → `copy running-config startup-config`

9. **Какой кабель использовать между коммутаторами?** → Copper Cross-Over (или Straight если MDIX поддерживается)

10. **Как проверить IP адреса на всех интерфейсах?** → `show ip interface brief`

## Ключевые принципы, которые нужно помнить

✅ **Всегда проверяйте prompt (>  или #)** — это показывает ваш текущий режим
✅ **? вопросительный знак** — ваш лучший друг для справки в любой момент
✅ **exit vs end** — exit выходит на один уровень выше, end прыгает в привилегированный режим
✅ **login команда ОБЯЗАТЕЛЬНА** — пароль не сработает без неё
✅ **no shutdown** — без этой команды интерфейс остается отключенным
✅ **service password-encryption** — зашифровывает ВСЕ пароли, включая те, что уже настроены
✅ **Сохраняйте конфигурацию** — иначе все настройки потеряются при перезагрузке
