# Протокол связующего дерева (STP)

**STP (Spanning Tree Protocol)** — сетевой протокол канального уровня (Layer 2), предназначенный для **автоматического устранения петель (loops)** в топологии коммутируемых сетей Ethernet.[computernetworkingnotes](https://www.computernetworkingnotes.com/ccna-study-guide/stp-port-states-blocking-listing-learning-and-forwarding.html)​Lektsiia-3-5.-Kanalnyi-uroven.pdf​

Петли возникают при наличии избыточных связей (резервных каналов) между коммутаторами. Без STP петля приводит к «широковещательному шторму» (Broadcast Storm), который за секунды парализует работу всей сети.Lektsiia-3-5.-Kanalnyi-uroven.pdf​

**Стандарт:** IEEE 802.1D (классический STP).

---

## 1. Принцип работы STP

STP строит логическую **древовидную топологию** (граф без циклов), блокируя избыточные порты. В сети остается только один активный путь между любыми двумя коммутаторами.

Алгоритм работы состоит из 4 шагов:youtube​[networkacademy+1](https://www.networkacademy.io/ccna/spanning-tree/spanning-tree-port-roles)​Lektsiia-3-5.-Kanalnyi-uroven.pdf​

## Шаг 1. Выбор Корневого моста (Root Bridge)

В сети выбирается один главный коммутатор — **Root Bridge**. Он становится «корнем» дерева, от которого строятся все пути.

- **Критерий выбора:** Коммутатор с **наименьшим Bridge ID (BID)**.[ipcisco](https://ipcisco.com/lesson/stp-operation/)​youtube​
    
- **Bridge ID (8 байт):** = `Приоритет (2 байта)` + `MAC-адрес (6 байт)`.youtube​Lektsiia-3-5.-Kanalnyi-uroven.pdf​
    
- По умолчанию приоритет у всех равен **32768**. Если приоритеты равны, корнем становится коммутатор с **наименьшим MAC-адресом**.[ipcisco](https://ipcisco.com/lesson/stp-operation/)​youtube​
    

## Шаг 2. Выбор Корневых портов (Root Ports)

На каждом **некорневом** коммутаторе выбирается **один** порт, который ведет к Корневому мосту самым коротким путем (имеет наименьшую стоимость).[networkacademy+1](https://www.networkacademy.io/ccna/spanning-tree/spanning-tree-port-roles)​

- **Стоимость пути (Path Cost):** Зависит от скорости канала (чем выше скорость, тем ниже стоимость).
    
    - 10 Гбит/с = стоимость 2
        
    - 1 Гбит/с = стоимость 4
        
    - 100 Мбит/с = стоимость 19
        
- Этот порт называется **Root Port (RP)** и находится в состоянии пересылки (Forwarding).
    

## Шаг 3. Выбор Назначенных портов (Designated Ports)

В каждом сегменте сети (на каждом кабеле между коммутаторами) выбирается **один** порт, который будет передавать трафик в сторону корневого моста.[ipcisco](https://ipcisco.com/lesson/stp-operation/)​

- Это порт коммутатора, который находится «ближе» к корню (имеет меньшую стоимость пути).
    
- Этот порт называется **Designated Port (DP)** и также пересылает данные.
    
- _Примечание:_ Все порты на Корневом мосту являются Назначенными (Designated).[ipcisco](https://ipcisco.com/lesson/stp-operation/)​
    

## Шаг 4. Блокировка остальных портов

Все порты, которые не стали ни Root, ни Designated, переходят в состояние **Блокировки (Blocking)**.[computernetworkingnotes+1](https://www.computernetworkingnotes.com/ccna-study-guide/stp-port-states-blocking-listing-learning-and-forwarding.html)​

- Они **не передают** пользовательский трафик.
    
- Они **продолжают слушать** служебные кадры BPDU, чтобы узнать, если основной канал упадет, и тогда разблокироваться.
    

---

## 2. Кадры BPDU

Коммутаторы обмениваются информацией с помощью специальных кадров — **BPDU (Bridge Protocol Data Units)**.[infrastructure-engineer+1](https://infrastructure-engineer.com/stp-bpdu-frame-format-001/)​Lektsiia-3-5.-Kanalnyi-uroven.pdf​

- Они рассылаются каждые **2 секунды** (по умолчанию).
    
- Содержат: Bridge ID отправителя, Root Bridge ID, стоимость пути до корня и таймеры.[infrastructure-engineer](https://infrastructure-engineer.com/stp-bpdu-frame-format-001/)​
    
- Именно на основе BPDU коммутаторы узнают о топологии и выбирают роли портов.
    

---

## 3. Состояния портов в STP

При включении или изменении топологии порт проходит несколько стадий, прежде чем начать передавать данные (чтобы убедиться, что не создаст петлю):[geeksforgeeks+2](https://www.geeksforgeeks.org/computer-networks/spanning-tree-port-states/)​

1. **Blocking (Блокировка):** Порт не передает данные, только слушает BPDU. (20 сек макс. возраст).[geeksforgeeks](https://www.geeksforgeeks.org/computer-networks/spanning-tree-port-states/)​
    
2. **Listening (Прослушивание):** Порт готовится к передаче, участвует в выборах корня, но не передает данные и не учит MAC-адреса. (15 сек).[computernetworkingnotes+1](https://www.computernetworkingnotes.com/ccna-study-guide/stp-port-states-blocking-listing-learning-and-forwarding.html)​
    
3. **Learning (Обучение):** Порт не передает данные, но **начинает заполнять таблицу MAC-адресов**, слушая входящий трафик. (15 сек).[geeksforgeeks+1](https://www.geeksforgeeks.org/computer-networks/spanning-tree-port-states/)​
    
4. **Forwarding (Пересылка):** Нормальный режим работы. Порт передает данные.[computernetworkingnotes](https://www.computernetworkingnotes.com/ccna-study-guide/stp-port-states-blocking-listing-learning-and-forwarding.html)​
    
5. **Disabled:** Порт административно выключен.
    

**Итого:** Классическому STP требуется **30–50 секунд** (Convergence time) для восстановления связи после сбоя.[pynetlabs](https://www.pynetlabs.com/stp-vs-rstp-whats-the-difference/)​

---

## 4. Развитие протокола (RSTP)

Из-за медленной сходимости классического STP (50 секунд — это очень долго для современных сетей) был разработан **RSTP (Rapid STP, IEEE 802.1w)**.[linkedin+1](https://www.linkedin.com/pulse/what-key-differences-between-stp-rstp-terms-other-atul-sharma-)​

- **Время сходимости:** Снижено до **нескольких секунд** (или даже миллисекунд).[linkedin+1](https://www.linkedin.com/pulse/what-key-differences-between-stp-rstp-terms-other-atul-sharma-)​
    
- Совместим с классическим STP.
    
- Сегодня де-факто используется именно RSTP (или MSTP), а не старый 802.1D.
    

## Резюме для ответа:

STP предотвращает петли в L2-сетях, блокируя избыточные порты. Выбирается главный коммутатор (Root Bridge), к нему строятся кратчайшие пути от остальных. Остальные линки «замораживаются» в резерве. Если основной канал падает, резервный разблокируется. Обмен информацией идет через кадры BPDU.
