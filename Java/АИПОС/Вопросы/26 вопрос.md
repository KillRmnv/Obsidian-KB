RIP (Routing Information Protocol) — это один из самых ранних и простых протоколов динамической маршрутизации, которые используют алгоритм distance-vector для автоматического заполнения таблиц маршрутизации и обеспечения связности между сетями, однако он имеет жёсткие ограничения по масштабируемости и медленную сходимость при изменении топологии. Протокол существует в двух версиях и остаётся актуален в учебных целях и в старых сетях.[habr](https://habr.com/ru/articles/335090/)​Lektsiia-8-11.-Marshrutizatsiia.-Protokoly-dinamicheskoi-marshrutizatsii.pdf​

## Определение и назначение RIP

RIP — это протокол, входящий в категорию IGP (Interior Gateway Protocol) — протоколов маршрутизации внутри автономной системы или корпоративной сети. На основе информации от соседних маршрутизаторов RIP автоматически строит таблицу маршрутизации, определяя лучший путь к целевым сетям, исходя из метрики дистанции (количество хопов).Lektsiia-8-11.-Marshrutizatsiia.-Protokoly-dinamicheskoi-marshrutizatsii.pdf​[wikipedia+2](https://ru.wikipedia.org/wiki/%D0%9F%D1%80%D0%BE%D1%82%D0%BE%D0%BA%D0%BE%D0%BB_%D0%BC%D0%B0%D1%80%D1%88%D1%80%D1%83%D1%82%D0%B8%D0%B7%D0%B0%D1%86%D0%B8%D0%B8)​

## Две версии RIP

## RIPv1 (Routing Information Protocol version 1)

RIPv1 — исторически первая версия протокола, разработанная на основе алгоритма Беллмана-Форда.[habr](https://habr.com/ru/articles/335090/)​Lektsiia-8-11.-Marshrutizatsiia.-Protokoly-dinamicheskoi-marshrutizatsii.pdf​

**Характеристики:**Lektsiia-8-11.-Marshrutizatsiia.-Protokoly-dinamicheskoi-marshrutizatsii.pdf​[habr](https://habr.com/ru/articles/335090/)​

- Работает с классовой адресацией (классы A, B, C).
    
- Автоматическое суммирование подсетей по границам классов — маршруты суммируются до класса, в пределах которого находятся.
    
- Использует broadcast (255.255.255.255) для рассылки своих таблиц маршрутизации.
    
- **Не передаёт маску подсети** в сообщениях; получатель вынужден угадывать маску на основе классовой принадлежности адреса.
    
- Метрика: количество хопов (0–15).
    
- Отправляет полную таблицу маршрутизации каждые 30 секунд по мультикасту (в реальности broadcast).
    

Сейчас используется крайне редко из-за отсутствия поддержки CIDR и других современных техник адресации.

## RIPv2 (Routing Information Protocol version 2) — рекомендуется

RIPv2 — улучшенная версия, которая решила многие проблемы первой версии.[habr](https://habr.com/ru/articles/335090/)​Lektsiia-8-11.-Marshrutizatsiia.-Protokoly-dinamicheskoi-marshrutizatsii.pdf​

**Ключевые улучшения:**Lektsiia-8-11.-Marshrutizatsiia.-Protokoly-dinamicheskoi-marshrutizatsii.pdf​[habr](https://habr.com/ru/articles/335090/)​

- Поддержка бесклассовой адресации (CIDR, VLSM) — в каждом маршруте передаётся **маска подсети**.
    
- Использует multicast (224.0.0.9) для рассылки обновлений вместо broadcast, что снижает нагрузку на другие устройства в сегменте.
    
- Поддержка аутентификации маршрутов (механизм защиты от ложных обновлений).
    

**Конфигурация RIPv2 (Cisco):**[habr](https://habr.com/ru/articles/335090/)​

text

`router rip   version 2              <- включает версию 2  no auto-summary        <- отключает автоматическое суммирование  network 192.168.1.0    <- активирует RIP на интерфейсах этой сети  network 10.1.1.0       <- ещё одна сеть для анонса`

## Принципы работы: Distance-Vector алгоритм

RIP использует алгоритм **distance-vector** (дистанционно-векторный), в котором каждый маршрутизатор принимает решения на основе информации от соседей.[neerc.ifmo+1](https://neerc.ifmo.ru/wiki/index.php?title=%D0%90%D0%BB%D0%B3%D0%BE%D1%80%D0%B8%D1%82%D0%BC%D1%8B_%D0%BC%D0%B0%D1%80%D1%88%D1%80%D1%83%D1%82%D0%B8%D0%B7%D0%B0%D1%86%D0%B8%D0%B8)​

**Два ключевых понятия:**[habr](https://habr.com/ru/articles/335090/)​

- **Distance** (дистанция) — метрика пути до целевой сети (количество хопов).
    
- **Vector** (вектор) — направление следующего маршрутизатора (next-hop), через который можно достичь целевой сети.
    

**Процесс обучения:**[habr](https://habr.com/ru/articles/335090/)​

1. Маршрутизатор при старте объявляет о сетях, напрямую подключённых к его интерфейсам (с метрикой 0).
    
2. Соседние маршрутизаторы получают эту информацию и добавляют 1 к метрике.
    
3. Каждый маршрутизатор выбирает маршрут с **наименьшей метрикой** для каждой целевой сети.
    
4. Информация распространяется дальше по сети шаг за шагом.
    

Процесс повторяется каждые **30 секунд**, поэтому новая информация распространяется медленно.

## Типы и формат сообщений RIP

RIP использует два основных типа сообщений:Lektsiia-8-11.-Marshrutizatsiia.-Protokoly-dinamicheskoi-marshrutizatsii.pdf​[habr](https://habr.com/ru/articles/335090/)​

## Request (запрос)

Маршрутизатор отправляет запрос, когда:

- Включается впервые (просит соседей прислать все известные маршруты).
    
- Хочет обновить информацию о конкретной сети.
    

При инициальном запросе метрика установлена в **16** (недостижимо), и список маршрутов пуст.[habr](https://habr.com/ru/articles/335090/)​

## Response (ответ)

Содержит полный список известных маршрутизатору сетей с их метриками.[habr](https://habr.com/ru/articles/335090/)​

**Структура одной записи в RIPv2 сообщении:**Lektsiia-8-11.-Marshrutizatsiia.-Protokoly-dinamicheskoi-marshrutizatsii.pdf​[habr](https://habr.com/ru/articles/335090/)​

text

`Address Family:  0x2          (IPv4) Network:         192.168.1.0  (целевая сеть) Subnet Mask:     255.255.255.0 (маска подсети — только v2) Next Hop:        192.168.1.1  (адрес следующего маршрутизатора) Metric:          0x1          (1 хоп, до сети с нулевой дистанцией 0)`

## Таблица маршрутизации и административная дистанция

## Коды в таблице маршрутизации

Каждый маршрут в таблице имеет букву, обозначающую источник маршрута:[habr](https://habr.com/ru/articles/335090/)​

|Код|Значение|Пример|
|---|---|---|
|**C**|Connected (напрямую подключённая сеть)|C 192.168.1.0/24 is directly connected|
|**R**|RIP маршрут|R 10.2.2.0 [120/1] via 192.168.1.2|
|**S**|Static маршрут|S 10.0.0.0 [1/0] via 192.168.3.1|

## Формат записи маршрута RIP

text

`R 10.2.2.0/24 [120/1] via 192.168.1.2 on FastEthernet0/0   ↑  ↑         ↑↑↑↑↑↑  |  |         AD / Metric  |  Целевая сеть  Источник (RIP) AD (Administrative Distance) = 120 для RIP Metric = 1 (количество хопов)`

- **AD (Administrative Distance)** — приоритет протокола при наличии нескольких источников маршрутов; RIP = 120 (низкий приоритет).
    
- **Metric** — значение метрики, в RIP это количество хопов (от 1 до 15).
    
- **Next-hop** — адрес следующего маршрутизатора для достижения целевой сети.

## Метрика и ограничения RIP

## Метрика: количество хопов

Метрика в RIP — это просто количество маршрутизаторов (хопов) на пути до целевой сети.Lektsiia-8-11.-Marshrutizatsiia.-Protokoly-dinamicheskoi-marshrutizatsii.pdf​[habr](https://habr.com/ru/articles/335090/)​

- Сетьс нулевой дистанцией (напрямую подключённая) имеет метрику **0**.
    
- Сеть через одного соседа — метрика **1**.
    
- Максимально допустимая метрика — **15 хопов**.
    
- Метрика **16 означает «недостижимо»** (poison reverse, отравление маршрута).[habr](https://habr.com/ru/articles/335090/)​
    

## Максимум 15 хопов

Это жёсткое ограничение делает RIP непригодным для крупных сетей.[habr+1](https://habr.com/ru/companies/ua-hosting/articles/466437/)​

Пример: если в сети больше 15 последовательных маршрутизаторов, то сети за 15-м маршрутизатором просто не будут достижимы через RIP.

## Таймеры RIP

RIP использует несколько таймеров для управления жизненным циклом маршрутов:Lektsiia-8-11.-Marshrutizatsiia.-Protokoly-dinamicheskoi-marshrutizatsii.pdf​[habr](https://habr.com/ru/articles/335090/)​

|Таймер|Стандартное значение|Назначение|
|---|---|---|
|**Update timer**|30 сек|Период отправки полных обновлений маршрутов соседям|
|**Invalid timer**|180 сек|Если маршрут не обновляется 180 сек, он помечается как Invalid|
|**Flush timer**|240 сек|Если маршрут остаётся Invalid до 240 сек, он удаляется из таблицы|
|**Holddown timer**|180 сек|Время, в течение которого маршрут игнорируется после получения информации об отказе|

Из-за этих таймеров сходимость при изменении топологии может занимать **3–4 минуты**, что неприемлемо для высоконадёжных сетей.

## Пример работы RIP

Представим топологию с тремя маршрутизаторами:[habr](https://habr.com/ru/articles/335090/)​

text

`Router0 (Loopback: 10.1.1.0/24)  ←→  Router1  ←→  Router2 (Loopback: 10.2.2.0/24)                 192.168.1.0/24       192.168.1.0/24  192.168.3.0/24`

После включения RIPv2 на всех маршрутизаторах произойдёт следующее:[habr](https://habr.com/ru/articles/335090/)​

1. **Первое обновление (t=0 сек):** Router0 отправляет пакет с информацией о 10.1.1.0/24 и 192.168.1.0/24 с метрикой 0.
    
2. **Router1 получает обновление:** Узнаёт о 10.1.1.0/24 через Router0 (метрика становится 1).
    
3. **Router1 передаёт соседям:** Отправляет информацию о 10.1.1.0/24 Router2 с метрикой 2.
    
4. **Второе обновление (t=30 сек):** Процесс повторяется.
    

**Таблица Router0 после сходимости:**[habr](https://habr.com/ru/articles/335090/)​

text

`C 10.1.1.0/24 is directly connected, Loopback1 C 192.168.1.0/24 is directly connected, FastEthernet0/0 R 10.2.2.0/24 [120/2] via 192.168.1.2, FastEthernet0/0 R 192.168.3.0/24 [120/3] via 192.168.1.2, FastEthernet0/0`

## Балансировка нагрузки (ECMP)

Если несколько маршрутов до одной целевой сети имеют одинаковую метрику, оба маршрута помещаются в таблицу маршрутизации, и трафик распределяется между ними (equal-cost multipath).[habr](https://habr.com/ru/articles/335090/)​

text

`R 192.168.4.0 [120/1] via 192.168.1.2, FastEthernet0/0                       via 192.168.3.2, FastEthernet0/1`

## Преимущества и недостатки RIP

|Характеристика|Статус|
|---|---|
|**Простота конфигурации**|✅ Очень простой протокол|
|**Автоматическое обнаружение соседей**|✅ Не требует вручную указывать соседей|
|**Быстрая сходимость**|❌ Медленная (180–240 сек)|
|**Масштабируемость**|❌ Максимум 15 хопов|
|**Резервные маршруты**|❌ Нет; при отказе основного маршрута происходит перестройка|
|**Служебный трафик**|❌ Высокий (полная таблица каждые 30 сек)|
|**Count-to-infinity проблема**|❌ При разрыве канала маршруты долго удаляются|
|**Открытый стандарт**|✅ RFC 2453 (RIPv2)|

## Практическое применение

На практике RIP почти не используется в современных сетях, но:[habr+1](https://habr.com/ru/companies/ua-hosting/articles/466437/)​

- Остаётся стандартом в учебных материалах CCNA для понимания принципов динамической маршрутизации.
    
- Иногда встречается в старых системах, которые так и не обновили конфигурацию.
    
- Полезен как первый протокол для обучения концепции distance-vector.
    

Большинство современных сетей используют OSPF (открытый стандарт, быстрая сходимость, масштабируемость) или EIGRP (если используется оборудование только Cisco).
