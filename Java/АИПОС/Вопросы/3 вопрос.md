# 1. Эталонная модель OSI/ISO

**Модель OSI** (Open Systems Interconnection — взаимодействие открытых систем) — это теоретическая эталонная модель, разработанная организацией **ISO** (International Organization for Standardization).

**Ее главная цель:** Стандартизировать сетевое взаимодействие, разбив сложный процесс передачи данных на 7 независимых уровней (слоев). Это позволяет оборудованию и ПО от разных производителей работать совместно.

> **Важно для понимания:** Модель OSI — это _теоретический стандарт_. В реальной жизни используется стек протоколов **TCP/IP**, который немного отличается, но терминология (уровни, адресация) всегда берется из OSI.

## 7 Уровней модели OSI (снизу вверх)

Традиционно уровни нумеруются с 1-го (нижнего) по 7-й (верхний). Студент обязан знать их порядок наизусть.

## Группа нижних уровней (Media Layers) — отвечают за доставку данных:

**1. Физический уровень (Physical Layer / L1)**

- **Задача:** Передача неструктурированного потока **битов** по физической среде (кабелю, радиоэфиру).
    
- **Что определяет:** Напряжение, частоты, типы разъемов (RJ-45), модуляцию сигнала.
    
- **Оборудование:** Концентраторы (Hub), репитеры, кабели, медиаконвертеры.
    

**2. Канальный уровень (Data Link Layer / L2)**

- **Задача:** Обеспечение надежной передачи данных внутри **одного сегмента сети** (между соседними узлами) и физическая адресация.
    
- **Подуровни:**
    
    - _MAC (Media Access Control)_ — управление доступом к среде и адресация (MAC-адреса).
        
    - _LLC (Logical Link Control)_ — связь с сетевым уровнем, контроль ошибок.
        
- **Оборудование:** Коммутаторы (Switch), мосты (Bridge), сетевые карты (NIC).
    
- **Адресация:** MAC-адрес (физический адрес).
    

**3. Сетевой уровень (Network Layer / L3)**

- **Задача:** Определение маршрута доставки данных между **разными сетями** (маршрутизация) и логическая адресация.
    
- **Протоколы:** IPv4, IPv6, ICMP, IPsec.
    
- **Оборудование:** Маршрутизаторы (Router), L3-коммутаторы.
    
- **Адресация:** IP-адрес (логический адрес).
    

## Группа верхних уровней (Host Layers) — отвечают за обработку данных:

**4. Транспортный уровень (Transport Layer / L4)**

- **Задача:** Передача данных между приложениями на хостах. Обеспечивает надежность (или быстроту) и правильный порядок доставки.
    
- **Ключевое понятие:** Порты (позволяют различать трафик разных приложений, например, веб-браузера и Skype, на одном компьютере).
    
- **Протоколы:**
    
    - _TCP_ (гарантированная доставка, установка соединения).
        
    - _UDP_ (быстрая передача без гарантий, например, для видео/голоса).
        

**5. Сеансовый уровень (Session Layer / L5)**

- **Задача:** Управление сеансом связи (создание, поддержание, синхронизация и разрыв сессии) между приложениями. Определяет, кто говорит сейчас, а кто слушает (полудуплекс/дуплекс).
    

**6. Представительский уровень (Presentation Layer / L6)**

- **Задача:** Преобразование данных в формат, понятный приложению или сети. Работает как «переводчик».
    
- **Функции:** Кодировка (ASCII, UTF-8), сжатие данных (ZIP, JPEG), шифрование/дешифрование (SSL/TLS часто относят сюда, хотя они работают и на L4/L5).
    

**7. Прикладной уровень (Application Layer / L7)**

- **Задача:** Обеспечение интерфейса между пользователем (приложением) и сетью. Это не само приложение (не Chrome и не Outlook), а сетевые протоколы, которые эти приложения используют.
    
- **Протоколы:** HTTP/HTTPS (веб), FTP (файлы), SMTP/POP3 (почта), DNS, DHCP.

# 2. Инкапсуляция и Декапсуляция

Эти термины описывают процесс прохождения данных по уровням модели OSI.

## Инкапсуляция (Encapsulation)

Процесс «упаковки» данных при их движении **сверху вниз** (от Прикладного уровня к Физическому) на стороне **отправителя**.

1. Пользователь отправляет данные (например, фото в мессенджере).
    
2. Каждый нижестоящий уровень берет данные с уровня выше, считает их просто «полезной нагрузкой» (Payload) и добавляет к ним свой **заголовок (Header)** с служебной информацией (адреса, контрольные суммы, флаги).
    
3. На канальном уровне (L2) часто добавляется еще и **концевик (Trailer)** для проверки целостности (FCS).
    
4. В итоге на физический уровень попадает длинная последовательность битов.
    

## Декапсуляция (Decapsulation)

Обратный процесс «распаковки» данных при их движении **снизу вверх** (от Физического уровня к Прикладному) на стороне **получателя**.

1. Сетевая карта принимает биты.
    
2. Уровень L2 читает заголовок кадра, проверяет MAC-адрес (мне ли это?) и контрольную сумму. Если всё ок — сбрасывает заголовок L2 и передает "начинку" на L3.
    
3. Уровень L3 проверяет IP-адрес. Сбрасывает заголовок L3, передает выше.
    
4. Процесс повторяется, пока «чистые» данные не дойдут до приложения получателя.

# 3. Протокольные единицы данных (PDU)

**PDU (Protocol Data Unit)** — это название блока данных на конкретном уровне модели OSI. Студент должен четко знать соответствие «Уровень — Название PDU». Это частый вопрос на засыпку.

|Уровень OSI|Название PDU (Рус)|Название PDU (Eng)|Что содержит (упрощенно)|
|---|---|---|---|
|**7-5. Прикладной, Представительский, Сеансовый**|**Данные**|**Data**|Полезная информация пользователя (текст, картинка, запрос).|
|**4. Транспортный**|**Сегмент** (для TCP)  <br>**Датаграмма** (для UDP)|**Segment** / **Datagram**|Данные + Порты отправителя/получателя + Порядковый номер (для TCP).|
|**3. Сетевой**|**Пакет**|**Packet**|Сегмент + IP-адреса (Source IP, Dest IP).|
|**2. Канальный**|**Кадр** (или Фрейм)|**Frame**|Пакет + MAC-адреса + FCS (контрольная сумма).|
|**1. Физический**|**Бит**|**Bit**|Электрические, световые или радиоимпульсы.|

## Как это звучит на экзамене (Пример ответа):

> _"При передаче данных происходит процесс инкапсуляции. На транспортном уровне данные разбиваются на **Сегменты**. Сетевой уровень добавляет к сегменту IP-адреса, формируя **Пакет**. Канальный уровень помещает пакет внутрь **Кадра**, добавляя MAC-адреса и контрольную сумму. На физическом уровне кадр передается в виде потока **Битов**."_
