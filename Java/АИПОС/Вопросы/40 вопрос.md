
## **ЭКЗАМЕНАЦИОННЫЙ ОТВЕТ: Протокол FTP (File Transfer Protocol)**

## **1. ОПРЕДЕЛЕНИЕ И НАЗНАЧЕНИЕ FTP**

**FTP (File Transfer Protocol)** — это протокол прикладного уровня для передачи файлов. FTP используется для обмена файлами между компьютерами в сети, обеспечивая надежную и эффективную передачу данных по протоколу TCP.Lektsiia-13-14.-Prikladnoi-uroven.pdf​

## **Основные характеристики FTP**

|Характеристика|Описание|
|---|---|
|**Уровень сети**|Протокол прикладного уровняLektsiia-13-14.-Prikladnoi-uroven.pdf​|
|**Порт для команд**|Порт 21Lektsiia-13-14.-Prikladnoi-uroven.pdf​|
|**Порт для передачи данных**|Порт 20Lektsiia-13-14.-Prikladnoi-uroven.pdf​|
|**Транспортный протокол**|TCP (протокол надежной доставки)Lektsiia-13-14.-Prikladnoi-uroven.pdf​|
|**Архитектура**|Клиент-серверная[timeweb](https://timeweb.cloud/blog/chto-takoe-ftp-protokol)​|
|**Безопасность**|Небезопасен из-за отсутствия шифрованияLektsiia-13-14.-Prikladnoi-uroven.pdf​|
|**Применение**|Передача файлов по сети, обновление веб-сайтов, резервное копирование|

## **2. АРХИТЕКТУРА FTP**

## **Двухканальная архитектура**

FTP отличается от других протоколов (например, HTTP) своей **двухканальной архитектурой**:[sky](https://sky.pro/wiki/sql/protokol-ftp-chto-eto-i-kak-rabotaet/)​

1. **Канал управления (Control Channel)**
    
    - Порт 21
        
    - Используется для отправки команд и получения ответов
        
    - Остается открытым на протяжении всей сессии[timeweb](https://timeweb.cloud/blog/chto-takoe-ftp-protokol)​
        
    - Постоянное соединение TCP
        
2. **Канал данных (Data Channel)**
    
    - Порт 20 (в активном режиме)
        
    - Порты > 1024 (в пассивном режиме)
        
    - Используется для передачи файлов
        
    - Открывается и закрывается для каждой операции передачи
        

## **Преимущество двухканальной архитектуры**

Разделение управления и передачи данных позволяет управлять процессом передачи независимо от самих данных.[sky](https://sky.pro/wiki/sql/protokol-ftp-chto-eto-i-kak-rabotaet/)​

## **3. РЕЖИМЫ РАБОТЫ FTP**

FTP работает в двух основных режимах, различающихся направлением инициирования соединения для передачи данных:[wikipedia+1](https://ru.wikipedia.org/wiki/FTP)​

## **АКТИВНЫЙ РЕЖИМ (Active Mode)**

**Принцип работы**:[vps+1](https://vps.ua/wiki/ftp-mode/)​

1. Клиент подключается к серверу на **порт 21** (управляющее соединение)
    
2. Клиент создает временный порт в диапазоне **1024-65535** для приема данных
    
3. Клиент отправляет серверу команду **PORT N+1**, указывая:
    
    - Свой IP-адрес
        
    - Номер порта для передачи данных
        
4. **Сервер инициирует соединение** с клиентом на указанный порт с **порта 20**
    
5. Происходит передача файлов
    

**Диаграмма активного режима**:[vps](https://vps.ua/wiki/ftp-mode/)​

```
Клиент                          Сервер
  ↓                              ↓
  +----(управление, порт 21)---→ :21
  ←----(ответ)---------------←
  ↑ (слушает порт 1037)
  |  +----← (соединение данных, порт 20)
  |       ↓ (порт 1037)
  +----(данные)---->

```

**Проблемы активного режима**:[vps+1](https://vps.ua/wiki/ftp-mode/)​

- Если клиент находится за брандмауэром или NAT, входящие соединения от сервера блокируются
    
- Невозможна передача файлов через некоторые файрволы
    
- Требует открытия входящих портов на клиентской стороне
    

---

## **ПАССИВНЫЙ РЕЖИМ (Passive Mode)**

**Принцип работы**:[interface31+1](https://interface31.ru/tech_it/2013/10/osobennosti-raboty-protokola-ftp.html)​

1. Клиент подключается к серверу на **порт 21** (управляющее соединение)
    
2. Клиент отправляет серверу команду **PASV**
    
3. **Сервер отвечает** с указанием:
    
    - Своего IP-адреса
        
    - Номера порта для передачи данных (обычно в диапазоне 1024-65535)
        
4. **Клиент инициирует соединение** с сервером на указанный порт
    
5. Происходит передача файлов
    

**Диаграмма пассивного режима**:[vps](https://vps.ua/wiki/ftp-mode/)​

```text
Клиент                          Сервер
  ↓                              ↓
  +----(управление, порт 21)---→ :21
  ←----(ответ)---------------←
  |
  +----(соединение данных)---→ :2024
  ←----(данные)---------------←

```

**Преимущества пассивного режима**:[interface31+1](https://interface31.ru/tech_it/2013/10/osobennosti-raboty-protokola-ftp.html)​

- Все соединения инициирует клиент
    
- Клиент может находиться за NAT и брандмауэром
    
- Клиент не требует выделенного IP-адреса
    
- **Основной режим работы FTP в современных условиях**[interface31](https://interface31.ru/tech_it/2013/10/osobennosti-raboty-protokola-ftp.html)​
    

## **Сравнение режимов**

|Аспект|Активный режим|Пассивный режим|
|---|---|---|
|**Инициатор соединения данных**|Сервер|Клиент|
|**Порт сервера для данных**|Порт 20|Случайный (> 1024)|
|**Требует NAT/брандмауэр**|Нет|Да|
|**Клиент за брандмауэром**|Проблематично|Работает|
|**Современное использование**|Редко|Часто|

## **4. РЕЖИМЫ ПЕРЕДАЧИ ДАННЫХ**

FTP поддерживает несколько режимов передачи данных:[wikipedia](https://ru.wikipedia.org/wiki/FTP)​

|Режим|Описание|
|---|---|
|**Поточный режим (Stream Mode)**|Данные посылаются в виде непрерывного потока; TCP выполняет всю обработку; индикатор конца файла не требуется[wikipedia](https://ru.wikipedia.org/wiki/FTP)​|
|**Блочный режим (Block Mode)**|FTP разбивает данные на блоки (блок заголовка, количество байт, поле данных) и передает их TCP[wikipedia](https://ru.wikipedia.org/wiki/FTP)​|
|**Сжатый режим (Compressed Mode)**|Данные сжимаются перед передачей для уменьшения объема|

## **5. ОСНОВНЫЕ КОМАНДЫ FTP**

Клиент отправляет команды на сервер через канал управления:[timeweb+1](https://timeweb.cloud/blog/chto-takoe-ftp-protokol)​

|Команда|Назначение|
|---|---|
|**USER**|Отправка имени пользователя|
|**PASS**|Отправка пароля|
|**PWD**|Вывод текущего каталога на сервере|
|**CWD**|Изменение текущего каталога|
|**LIST**|Вывод списка файлов в текущем каталоге|
|**RETR**|Загрузка файла с сервера|
|**STOR**|Загрузка файла на сервер|
|**DELE**|Удаление файла на сервере|
|**MKD**|Создание нового каталога|
|**RMD**|Удаление каталога|
|**PASV**|Переход в пассивный режим|
|**PORT**|Указание активного режима и порта|
|**QUIT**|Завершение сессии|

## **6. ПРОЦЕСС РАБОТЫ FTP**

Типичная сессия FTP включает следующие этапы:[timeweb+1](https://timeweb.cloud/blog/chto-takoe-ftp-protokol)​

1. **Подключение** — клиент подключается к серверу на порт 21
    
2. **Аутентификация** — отправка логина (USER) и пароля (PASS)
    
    - Возможен анонимный доступ (login: anonymous, password: email)
        
3. **Навигация** — просмотр и изменение каталогов (PWD, CWD, LIST)
    
4. **Установка параметров** — выбор режима передачи (бинарный/текстовый)
    
5. **Открытие соединения данных** — активный или пассивный режим
    
6. **Передача файла** — загрузка (RETR) или скачивание (STOR)
    
7. **Закрытие соединения данных** — разрыв соединения
    
8. **Повторение шагов 4-7** — для других файлов при необходимости
    
9. **Завершение сессии** — команда QUIT

## **9. ИСПОЛЬЗОВАНИЕ FTP**

**Где используется FTP:**

- Загрузка веб-сайтов на хостинг
    
- Резервное копирование файлов
    
- Обмен большими файлами между компьютерами
    
- Управление файловой структурой сервера
    
- Программное обеспечение для синхронизации файлов

- Системы с высокими требованиями безопасности

## **Заключение**

FTP (File Transfer Protocol) — это классический протокол прикладного уровня для передачи файлов в сети. Он работает по схеме клиент-сервер с использованием двух каналов: управляющего (порт 21) и канала данных (порт 20). FTP поддерживает два режима работы: активный и пассивный, причем пассивный режим предпочтителен в современных сетях из-за его совместимости с NAT и брандмауэрами.

Однако **FTP небезопасен** из-за отсутствия шифрования, поэтому в настоящее время рекомендуется использовать его безопасные альтернативы: **SFTP** (более безопасная, современная) или **FTPS** (реже используется). Для критичных операций с передачей данных следует выбирать SFTP, который обеспечивает полное шифрование и надежную аутентификацию.
