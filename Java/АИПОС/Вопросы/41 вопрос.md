## **ЭКЗАМЕНАЦИОННЫЙ ОТВЕТ: Протоколы SMTP, POP3, IMAP**

## **ВВЕДЕНИЕ**

Электронная почта — один из основных сервисов интернета, работающий благодаря взаимодействию нескольких протоколов прикладного уровня. Для отправки и получения писем используются три основных протокола:Lektsiia-13-14.-Prikladnoi-uroven.pdf​

- **SMTP** — для отправки сообщений
    
- **POP3** — для загрузки сообщений на устройство пользователя
    
- **IMAP** — для управления сообщениями на сервере[sky](https://sky.pro/wiki/sql/rabota-s-elektronnoj-pochtoj-protokoly-i-standarty/)​

## **2. ПРОТОКОЛ SMTP (Simple Mail Transfer Protocol)**

## **Определение и назначение**

**SMTP** — это протокол прикладного уровня для отправки электронной почты и маршрутизации сообщений между почтовыми серверами. SMTP использует простой подход "клиент-сервер" для передачи писем.[admin24+2](https://admin24.ru/blog/smtp-podkluchenue-osobennosti)​Lektsiia-13-14.-Prikladnoi-uroven.pdf​

## **Основные характеристики SMTP**

|Характеристика|Описание|
|---|---|
|**Уровень сети**|Протокол прикладного уровняLektsiia-13-14.-Prikladnoi-uroven.pdf​|
|**Основной порт**|Порт 25Lektsiia-13-14.-Prikladnoi-uroven.pdf​|
|**Порт для клиентов**|Порт 587 (для отправки с клиента)[miran](https://miran.ru/blog/protokol-smtp-udobnyy-obmen-elektronnoy-pochtoy)​|
|**Защищённый порт**|Порт 465 (SSL-соединение)[miran](https://miran.ru/blog/protokol-smtp-udobnyy-obmen-elektronnoy-pochtoy)​|
|**Транспортный протокол**|TCP (надежная доставка)Lektsiia-13-14.-Prikladnoi-uroven.pdf​|
|**Архитектура**|Клиент-серверная[admin24](https://admin24.ru/blog/smtp-podkluchenue-osobennosti)​|
|**Назначение**|Только для отправки, не для полученияLektsiia-13-14.-Prikladnoi-uroven.pdf​[selectel](https://selectel.ru/blog/smtp-protocol/)​|

## **Функции SMTP**

SMTP выполняет две основные функции:[miran+1](https://miran.ru/blog/protokol-smtp-udobnyy-obmen-elektronnoy-pochtoy)​

1. **Проверка и разрешение**
    
    - Проверка корректности настроек системы
        
    - Разрешение на отправку сообщения на указанный адрес
        
2. **Доставка и уведомление**
    
    - Маршрутизация письма адресату
        
    - Возврат сообщения об ошибке или успешной доставке
        

## **Как работает SMTP**

**Процесс отправки письма**:[selectel+2](https://selectel.ru/blog/smtp-protocol/)​

1. **Начало сеанса** — клиент отправляет команду **HELO** или **EHLO** для приветствия сервера и начала соединения[admin24](https://admin24.ru/blog/smtp-podkluchenue-osobennosti)​
    
2. **Указание адреса отправителя** — клиент отправляет команду **MAIL FROM**, указывая адрес отправителя[miran+1](https://miran.ru/blog/protokol-smtp-udobnyy-obmen-elektronnoy-pochtoy)​
    
3. **Указание адреса получателя** — клиент отправляет команду **RCPT TO**, указывая адрес получателя. Возможно несколько получателей (команда повторяется)[selectel+2](https://selectel.ru/blog/smtp-protocol/)​
    
4. **Передача данных** — клиент отправляет команду **DATA**, после которой идет содержимое письма. Конец обозначается точкой на отдельной строке[admin24+1](https://admin24.ru/blog/smtp-podkluchenue-osobennosti)​
    
5. **Доставка** — почтовый сервер отправителя передает письмо на почтовый сервер получателя, устанавливая соединение по порту 25[admin24](https://admin24.ru/blog/smtp-podkluchenue-osobennosti)​
    
6. **Подтверждение** — сервер получателя отправляет ответное сообщение, указывая успешное получение (например, код "250 OK")[admin24](https://admin24.ru/blog/smtp-podkluchenue-osobennosti)​
    
7. **Завершение** — клиент отправляет команду **QUIT** для разрыва соединения[selectel](https://selectel.ru/blog/smtp-protocol/)​
    

## **Основные команды SMTP**

|Команда|Назначение|
|---|---|
|**HELO/EHLO**|Установка соединения и приветствие сервера[admin24](https://admin24.ru/blog/smtp-podkluchenue-osobennosti)​|
|**MAIL FROM**|Указание адреса отправителя[admin24+1](https://admin24.ru/blog/smtp-podkluchenue-osobennosti)​|
|**RCPT TO**|Указание адреса получателя (может повторяться)[admin24+1](https://admin24.ru/blog/smtp-podkluchenue-osobennosti)​|
|**DATA**|Уведомление о начале передачи тела письма[miran+1](https://miran.ru/blog/protokol-smtp-udobnyy-obmen-elektronnoy-pochtoy)​|
|**QUIT**|Завершение сеанса и разрыв соединения[selectel](https://selectel.ru/blog/smtp-protocol/)​|

## **Типы SMTP-серверов**

**Исходящие SMTP-серверы**

- Используются для отправки электронной почты от клиентского приложения
    
- Проверяют письма на наличие ошибок
    
- Передают письмо на следующий узел[admin24](https://admin24.ru/blog/smtp-podkluchenue-osobennosti)​
    

**Входящие SMTP-серверы**

- Принимают почту, адресованную определенному домену
    
- Являются точкой входа для входящих писем
    
- Проверяют адреса, аутентифицируют отправителя[admin24](https://admin24.ru/blog/smtp-podkluchenue-osobennosti)​
    

## **Безопасность SMTP**

**SMTPS (Simple Mail Transfer Protocol Secure)**

- Использует SSL/TLS для шифрования[aws.amazon](https://aws.amazon.com/ru/what-is/smtp/)​
    
- Обеспечивает аутентификацию партнеров по связи
    
- Гарантирует целостность и конфиденциальность данных[aws.amazon](https://aws.amazon.com/ru/what-is/smtp/)​
    
- Порт 465 для SSL или порт 587 с STARTTLS[aws.amazon](https://aws.amazon.com/ru/what-is/smtp/)​

## **3. ПРОТОКОЛ POP3 (Post Office Protocol version 3)**

## **Определение и назначение**

**POP3** — это протокол прикладного уровня, используемый для получения сообщений с сервера электронной почты. POP3 позволяет загружать письма с сервера на локальное устройство пользователя.[selectel+1](https://selectel.ru/blog/pop3-vs-imap/)​Lektsiia-13-14.-Prikladnoi-uroven.pdf​

## **Основные характеристики POP3**

|Характеристика|Описание|
|---|---|
|**Уровень сети**|Протокол прикладного уровня|
|**Порт (незашифрованный)**|Порт 110[selectel](https://selectel.ru/blog/pop3-vs-imap/)​|
|**Порт (зашифрованный)**|Порт 995[selectel](https://selectel.ru/blog/pop3-vs-imap/)​|
|**Транспортный протокол**|TCP через стек TCP/IP[selectel](https://selectel.ru/blog/pop3-vs-imap/)​|
|**Архитектура**|Клиент-серверная|
|**Направление**|Односторонний (только получение)[r7-office](https://support.r7-office.ru/organizer/elektronnaia-pochta-i-soobshcheniia/raznica-mezhdu-imap-i-pop3/)​|
|**Удаление писем**|По умолчанию удаляются с сервера после загрузки[selectel](https://selectel.ru/blog/pop3-vs-imap/)​|

## **Как работает POP3**

**Процесс работы**:[reg+1](https://help.reg.ru/support/pochta-i-servisy/obshchiye-voprosy/imap-ili-pop-3-chto-luchshe)​

1. **Подключение** — почтовый клиент подключается к серверу POP3
    
2. **Загрузка сообщений** — клиент загружает все сообщения с сервера в папку входящих сообщений пользователя
    
3. **Сохранение локально** — сообщения сохраняются на компьютере или устройстве пользователя
    
4. **Удаление с сервера** — сообщения обычно удаляются с почтового сервера (по умолчанию через 7 дней)[r7-office+1](https://support.r7-office.ru/organizer/elektronnaia-pochta-i-soobshcheniia/raznica-mezhdu-imap-i-pop3/)​
    
5. **Автономный доступ** — пользователь может получать доступ к письмам без интернета
    

## **Особенность POP3**

**Важно отметить:** POP3 работает только с получением электронной почты и **не поддерживает отправку**. Для отправки требуется использовать SMTP.[selectel](https://selectel.ru/blog/pop3-vs-imap/)​

## **Преимущества POP3**

|Преимущество|Описание|
|---|---|
|**Автономный доступ**|Пользователи могут загружать письма и получать к ним доступ в автономном режиме без интернета[selectel](https://selectel.ru/blog/pop3-vs-imap/)​|
|**Экономия памяти на сервере**|Пользователи контролируют объем памяти на сервере, загружая и удаляя сообщения[selectel](https://selectel.ru/blog/pop3-vs-imap/)​|
|**Одно устройство**|Удобен для пользователей, использующих основном одно устройство[selectel](https://selectel.ru/blog/pop3-vs-imap/)​|
|**Экономия на сервере**|Большинство писем хранятся локально, а не на сервере[reg](https://help.reg.ru/support/pochta-i-servisy/obshchiye-voprosy/imap-ili-pop-3-chto-luchshe)​|

## **Недостатки POP3**

|Недостаток|Описание|
|---|---|
|**Только одно устройство**|Если письма удаляются с сервера, получить к ним доступ с другого устройства невозможно[reg](https://help.reg.ru/support/pochta-i-servisy/obshchiye-voprosy/imap-ili-pop-3-chto-luchshe)​|
|**Отсутствие синхронизации**|Нет синхронизации между устройствами[reg](https://help.reg.ru/support/pochta-i-servisy/obshchiye-voprosy/imap-ili-pop-3-chto-luchshe)​|
|**Ручное управление папками**|Нужно вручную создавать и управлять папками на каждом устройстве[selectel](https://selectel.ru/blog/pop3-vs-imap/)​|
|**Односторонний протокол**|Данные с локального устройства нельзя снова отправить на сервер[reg](https://help.reg.ru/support/pochta-i-servisy/obshchiye-voprosy/imap-ili-pop-3-chto-luchshe)​|

## **4. ПРОТОКОЛ IMAP (Internet Message Access Protocol)**

## **Определение и назначение**

**IMAP** — это протокол прикладного уровня, который позволяет управлять почтовым ящиком непосредственно на сервере электронной почты. IMAP предоставляет полный доступ к письмам с синхронизацией между несколькими устройствами.[sky+2](https://sky.pro/wiki/sql/protokoly-obmena-soobsheniyami-smtp-imap-pop3-i-drugie/)​

## **Основные характеристики IMAP**

|Характеристика|Описание|
|---|---|
|**Уровень сети**|Протокол прикладного уровня|
|**Стандартный порт**|Порт 143|
|**Порт (зашифрованный)**|Порт 993|
|**Транспортный протокол**|TCP|
|**Архитектура**|Клиент-серверная|
|**Направление**|Двусторонний (чтение и управление на сервере)[r7-office](https://support.r7-office.ru/organizer/elektronnaia-pochta-i-soobshcheniia/raznica-mezhdu-imap-i-pop3/)​|
|**Хранение**|Письма хранятся на сервере[selectel](https://selectel.ru/blog/pop3-vs-imap/)​|
|**Синхронизация**|Синхронизирует состояние между несколькими устройствами[sky](https://sky.pro/wiki/sql/rabota-s-elektronnoj-pochtoj-protokoly-i-standarty/)​|

## **Как работает IMAP**

**Основные операции IMAP**:[sky](https://sky.pro/wiki/sql/protokoly-obmena-soobsheniyami-smtp-imap-pop3-i-drugie/)​

1. **Установление соединения и аутентификация** — клиент подключается к серверу и вводит учетные данные
    
2. **Выбор почтового ящика** — команда SELECT выбирает папку для работы
    
3. **Получение структуры сообщений** — команда FETCH получает информацию о письмах
    
4. **Загрузка выбранных частей** — можно загружать только заголовки или выбранные части сообщения[selectel+1](https://selectel.ru/blog/pop3-vs-imap/)​
    
5. **Манипуляция сообщениями**[sky](https://sky.pro/wiki/sql/protokoly-obmena-soobsheniyami-smtp-imap-pop3-i-drugie/)​
    
    - Пометка как прочитанное/непрочитанное
        
    - Перемещение между папками
        
    - Удаление сообщений
        
6. **Поддержка поиска на сервере** — команда SEARCH позволяет искать письма на сервере[sky](https://sky.pro/wiki/sql/protokoly-obmena-soobsheniyami-smtp-imap-pop3-i-drugie/)​

- Важна **скорость** и минимальная нагрузка[selectel](https://selectel.ru/blog/pop3-vs-imap/)​
    

## **Выбирайте IMAP, если:**

- Используете **несколько устройств** (смартфон, планшет, ПК)[selectel](https://selectel.ru/blog/pop3-vs-imap/)​

- Требуется **расширенный поиск** по письмам[selectel](https://selectel.ru/blog/pop3-vs-imap/)​

## **7. АРХИТЕКТУРА РАБОТЫ ПОЧТОВОЙ СИСТЕМЫ**

## **Полный цикл доставки письма**

```bash
┌─────────────────┐
│ Отправитель     │
└────────┬────────┘
         │
         ↓ (SMTP - отправка)
   ┌─────────────────────┐
   │ SMTP сервер         │
   │ отправителя         │
   └────────┬────────────┘
            │
            ↓ (SMTP - маршрутизация)
   ┌─────────────────────┐
   │ SMTP сервер         │
   │ получателя          │
   │ (хранение письма)   │
   └────────┬────────────┘
            │
            ↓ (POP3/IMAP)
   ┌─────────────────────┐
   │ Клиент получателя   │
   └────────┬────────────┘
            │
            ↓
   ┌─────────────────────┐
   │ Получатель          │
   └─────────────────────┘

```
## **Компоненты системы**

1. **MUA (Mail User Agent)** — почтовый клиент
    
2. **MTA (Mail Transfer Agent)** — SMTP-сервер (передача между серверами)
    
3. **MDA (Mail Delivery Agent)** — почтовый сервер получателя (хранение)

## **8. БЕЗОПАСНОСТЬ ПОЧТОВЫХ ПРОТОКОЛОВ**

## **Защита данных**

Для защиты электронной почты используются:[arxiv](https://arxiv.org/pdf/2208.00388.pdf)​

- **TLS (Transport Layer Security)** — прямое или через STARTTLS[arxiv](https://arxiv.org/pdf/1511.00341.pdf)​
    
- **SSL (Secure Socket Layer)** — для шифрования соединения
    
- **PGP/GPG** — для шифрования и подписи сообщений[arxiv](https://arxiv.org/pdf/2208.00388.pdf)​
    

## **Безопасные версии протоколов**

- **SMTPS** — SMTP + SSL/TLS (порт 465, 587)
    
- **POP3S** — POP3 + SSL/TLS (порт 995)
    
- **IMAPS** — IMAP + SSL/TLS (порт 993)

## **Заключение**

**SMTP, POP3 и IMAP** — это три основных протокола электронной почты, каждый из которых выполняет специфическую функцию:

- **SMTP** отвечает за **отправку** и маршрутизацию писем между серверами
    
- **POP3** предоставляет простой способ **загружать письма на локальное устройство**
    
- **IMAP** обеспечивает современный **управление почтой со множества устройств** с синхронизацией
    

Выбор между POP3 и IMAP зависит от потребностей пользователя, но в современном мире **IMAP становится все более популярным** благодаря поддержке множества устройств и синхронизации. Для критичных операций рекомендуется использовать защищенные версии протоколов (SMTPS, POP3S, IMAPS) с шифрованием SSL/TLS.
