BGP (Border Gateway Protocol) — это внешний протокол маршрутизации (EGP), предназначенный для обмена маршрутной информацией между автономными системами в Интернете на основе политик и атрибутов пути, а не технических характеристик связи, и используется для построения глобальной маршрутизирующей инфраструктуры Интернета. Протокол работает по TCP на порту 179 и является одним из наиболее критичных механизмов, обеспечивающих функционирование Интернета наряду с DNS.[wikipedia+1](https://ru.wikipedia.org/wiki/Border_Gateway_Protocol)​

## Основные понятия BGP

## Автономная система (Autonomous System, AS)

Автономная система — это группа маршрутизаторов под единым техническим и административным управлением, использующих один или несколько протоколов внутридоменной маршрутизации (IGP, таких как OSPF, EIGRP, RIP) для определения маршрутов внутри себя, и BGP для определения маршрутов доставки пакетов в другие AS.[xgu+1](http://xgu.ru/wiki/BGP)​

Каждая AS имеет уникальный номер ASN (Autonomous System Number), распределяемый IANA (Internet Assigned Numbers Authority):

- 2-байтовые ASN: от 1 до 65535 (исторические, частично устарели).
    
- 4-байтовые ASN: от 65536 и выше (современные).
    

**Пример:** AS 8452 (Telenor, провайдер), AS 3320 (Deutsche Telekom), AS 13188 (Yandex).

## eBGP и iBGP

**eBGP (External BGP):**[sedicomm+1](https://blog.sedicomm.com/2023/10/30/vnutrennij-protokol-bgp-ibgp/)​

- Протокол маршрутизации между маршрутизаторами в **разных** AS.
    
- Соседи обычно соединены напрямую через выделенные каналы.
    
- Используется для обмена информацией о маршрутах между провайдерами и крупными операторами.
    
- Преобладающий AS Path используется для предотвращения циклических маршрутов.
    

**iBGP (Internal BGP):**[xgu+1](http://xgu.ru/wiki/BGP)​

- Протокол маршрутизации между маршрутизаторами в **пределах одной** AS.
    
- Требует полносвязной топологии (full mesh) между всеми iBGP-соседями или использования route reflector.
    
- Передача информации, полученной от eBGP-соседей, внутри AS.
    
- Сохраняет атрибуты маршрутов (Next Hop, Origin, Local Preference и т.п.).
    

## Характеристики BGP

**Протокол работает над TCP (порт 179):**[sedicomm+1](https://blog.sedicomm.com/2025/05/07/soobshheniya-bgp-pri-ustanovlenii-soedineniya/)​

- Использует надёжную доставку данных.
    
- Экономит пропускную способность: после первоначального обмена полными таблицами передаются только **инкрементальные обновления** (изменения).
    

**Поддержка масштабирования:**[wikipedia](https://ru.wikipedia.org/wiki/Border_Gateway_Protocol)​

- Работает с бесклассовой адресацией (CIDR).
    
- Использует суммирование маршрутов (route aggregation) для снижения размера таблиц маршрутизации.
    
- На текущий момент глобальная таблица маршрутизации BGP содержит более 450 тысяч маршрутов для IPv4.
    

**Выбор маршрутов на основе политик:**[xgu](http://xgu.ru/wiki/BGP)​

- BGP выбирает маршруты не на основе технических метрик (пропускная способность, задержка), как OSPF или EIGRP.
    
- Выбор основан на атрибутах пути (Path Attributes) и политиках маршрутизации, определённых администраторами.
    
- Например, маршрут может быть выбран не потому что он быстрее, а потому что он дешевле или надёжнее.
    

## Сообщения BGP

BGP использует четыре основных типа сообщений:Lektsiia-8-11.-Marshrutizatsiia.-Protokoly-dinamicheskoi-marshrutizatsii.pdf​[sedicomm](https://blog.sedicomm.com/2025/05/07/soobshheniya-bgp-pri-ustanovlenii-soedineniya/)​

## 1. OPEN (Тип 1)

Сообщение для инициализации соединения и согласования параметров между двумя маршрутизаторами BGP.Lektsiia-8-11.-Marshrutizatsiia.-Protokoly-dinamicheskoi-marshrutizatsii.pdf​[sedicomm](https://blog.sedicomm.com/2025/05/07/soobshheniya-bgp-pri-ustanovlenii-soedineniya/)​

**Содержимое:**[icebale.readthedocs+1](https://icebale.readthedocs.io/en/latest/networks/protocols-tech/BGP/BGP/)​

- **Version** — версия протокола BGP (текущая версия 4).
    
- **My Autonomous System** — номер AS отправителя.
    
- **Hold Time** — максимальное время (в секундах) между получением Keepalive или Update сообщений; по умолчанию 90 секунд.
    
- **BGP Identifier** — идентификатор маршрутизатора (обычно наибольший IP-адрес Loopback или интерфейса).
    
- **Optional Parameters** — поддерживаемые возможности (capabilities).
    

**Процесс:** когда соединение TCP установлено, первым делом передаётся OPEN. Если параметры согласованы, маршрутизатор переходит в состояние OpenConfirm и начинает отправлять Keepalive.

## 2. UPDATE (Тип 2)

Сообщение для обмена информацией о маршрутизации (объявление новых маршрутов или отзыв старых).[sedicomm](https://blog.sedicomm.com/2025/05/07/soobshheniya-bgp-pri-ustanovlenii-soedineniya/)​Lektsiia-8-11.-Marshrutizatsiia.-Protokoly-dinamicheskoi-marshrutizatsii.pdf​

**Структура:**[icebale.readthedocs+1](https://icebale.readthedocs.io/en/latest/networks/protocols-tech/BGP/BGP/)​

text

`Unfeasible Routes Length — длина списка отозванных маршрутов Withdrawn Routes — маршруты, которые больше недоступны Total Path Attribute Length — длина списка атрибутов пути Path Attributes — атрибуты маршрутов (AS_PATH, NEXT_HOP, ORIGIN, MED, Local Pref и т.п.) Network Layer Reachability Information (NLRI) — объявляемые маршруты в формате префикс/длина`

**Пример:** маршрутизатор объявляет, что сети 192.168.1.0/24 и 10.0.0.0/8 доступны через него с определёнными атрибутами (AS Path, Next Hop и т.п.).

## 3. KEEPALIVE (Тип 4)

Простое сообщение для проверки доступности соседа и поддержания соединения.[admindoc+1](http://admindoc.ru/1142/kratko-o-bgp/)​

**Характеристики:**[admindoc+1](http://admindoc.ru/1142/kratko-o-bgp/)​

- Отправляется каждые **60 секунд** по умолчанию.
    
- Содержит только заголовок (19 байт); данных нет.
    
- Если маршрутизатор не получает никаких сообщений (Keepalive или Update) в течение **Hold Time** (по умолчанию 180 секунд = 3 × 60), соседство разрывается.
    

## 4. NOTIFICATION (Тип 3)

Сообщение об ошибке, которое приводит к разрыву соединения.[sedicomm+1](https://blog.sedicomm.com/2025/05/07/soobshheniya-bgp-pri-ustanovlenii-soedineniya/)​

**Примеры ошибок:**Lektsiia-8-11.-Marshrutizatsiia.-Protokoly-dinamicheskoi-marshrutizatsii.pdf​

- Несовпадение номеров AS (ожидалось одно число, получено другое).
    
- Некорректный BGP Identifier.
    
- Hold Time истёк (сосед слишком долго молчит).
    
- Ошибка в содержимом OPEN сообщения.
    
- Проблема с обработкой UPDATE.
    

При отправке NOTIFICATION сеанс BGP завершается, все маршруты от этого соседа удаляются из таблицы, и маршрутизатор отправляет Withdrawn Routes своим другим соседям.

## Состояния соединения BGP (FSM)

BGP использует конечный автомат с шестью состояниями:[icebale.readthedocs](https://icebale.readthedocs.io/en/latest/networks/protocols-tech/BGP/BGP/)​Lektsiia-8-11.-Marshrutizatsiia.-Protokoly-dinamicheskoi-marshrutizatsii.pdf​

1. **Idle** — начальное состояние; BGP инициирует TCP-соединение к соседу.
    
2. **Connect** — ожидание завершения трёхстороннего рукопожатия TCP; если успешно → OpenSent.
    
3. **Active** — попытка установить TCP-соединение заново (если Connect завершился с ошибкой).
    
4. **OpenSent** — BGP отправил OPEN и ожидает OPEN от соседа; если получено корректное OPEN → OpenConfirm.
    
5. **OpenConfirm** — параметры согласованы; ожидание Keepalive от соседа; если получено → Established.
    
6. **Established** — соседство полностью установлено; обмен UPDATE и Keepalive сообщениями.
    

## Атрибуты пути (Path Attributes)

Path Attributes — это атрибуты маршрутов, которые определяют, какой маршрут будет выбран при наличии нескольких вариантов.[1275+1](https://1275.ru/glossary/bgp)​

## Основные атрибуты:

|Атрибут|Описание|Использование|
|---|---|---|
|**AS_PATH**|Список автономных систем, через которые прошёл маршрут; используется для предотвращения циклов|При выборе маршрута — предпочитается путь с наименьшим AS_PATH|
|**NEXT_HOP**|IP-адрес следующего маршрутизатора для достижения целевой сети|Определяет, куда отправить пакет|
|**ORIGIN**|Источник маршрута: IGP (0) — маршрут создан в пределах AS; EGP (1) — получен из EGP; Incomplete (2) — неизвестный источник|Предпочитаются маршруты с Origin=IGP|
|**LOCAL_PREF**|Приоритет маршрута внутри AS (по умолчанию 100); выше значение — выше приоритет|Только между iBGP-соседями; EBGP-соседям не передаётся|
|**MED (Multi-Exit Discriminator)**|Подсказка соседней AS о предпочтительном пути; по умолчанию 0; ниже значение — выше приоритет|Используется при выборе пути, когда есть несколько точек входа в AS|
|**AGGREGATOR**|Информация о маршрутизаторе, который выполнил агрегацию маршрутов|Для диагностики|

## Порядок выбора маршрута в BGP:[xgu](http://xgu.ru/wiki/BGP)​

1. Проверка доступности Next-Hop (Route Resolvability Condition).
    
2. Выбор маршрута с **наибольшим значением LOCAL_PREF** (если несколько маршрутов).
    
3. Выбор маршрута с **наименьшей длиной AS_PATH**.
    
4. Выбор маршрута с **наименьшим значением ORIGIN** (IGP < EGP < Incomplete).
    
5. Выбор маршрута с **наименьшим значением MED** (обычно только внутри одной AS).
    
6. Предпочтение **eBGP-маршрутов** перед iBGP-маршрутами.
    
7. Выбор маршрута с **наименьшей IGP-метрикой** до Next-Hop.
    
8. Выбор маршрута от соседа с **наименьшим BGP Identifier** (как последний критерий).
    

## Таблицы BGP

BGP хранит несколько таблиц:[xgu](http://xgu.ru/wiki/BGP)​

- **Таблица BGP (BGP RIB)** — содержит **ВСЕ** известные маршруты, включая альтернативные пути.
    
- **Таблица маршрутизации (RIB, Routing Information Base)** — содержит только **лучшие маршруты**, выбранные BGP на основе политик и атрибутов.
    

Пример записи в таблице маршрутизации:

text

`*> 192.168.1.0/24 [20/0] via 192.168.1.1, 00:05:30    ^              ^^ ^   |              || +-метрика   |              |+--доступность (20 - eBGP)   |              +---административная дистанция   +--лучший маршрут`

## Преимущества и недостатки BGP

|Характеристика|Статус|
|---|---|
|**Оперирует политиками, а не метриками**|✅ Гибкость в управлении маршрутизацией|
|**Масштабируемость для Интернета**|✅ Поддерживает сотни тысяч маршрутов|
|**Надёжная доставка (TCP)**|✅ Гарантирует доставку сообщений|
|**Инкрементальные обновления**|✅ Экономит пропускную способность|
|**Открытый стандарт**|✅ RFC 4271 (работает везде)|
|**Сложность конфигурации**|❌ Требует глубокого понимания политик и атрибутов|
|**Медленная сходимость в больших сетях**|❌ При отказе может потребоваться время на пересчёт|
|**Требует полносвязной топологии iBGP**|❌ В крупных AS используются Route Reflector или Confederation|
|**Высокие требования к памяти**|❌ Хранение полной таблицы маршрутов (450k+ записей для IPv4)|

BGP остаётся неотъемлемой частью инфраструктуры Интернета, обеспечивая глобальную маршрутизацию между миллионами узлов и автономными системами в режиме реального времени.
