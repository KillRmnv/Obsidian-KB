Маршрутизация — это основной процесс, который выполняют маршрутизаторы на сетевом уровне (L3) для передачи IP‑пакетов между различными сетями, основываясь на таблице маршрутизации и выборе оптимального пути к целевой сети. Ниже подробно разобраны принципы работы, типы маршрутизации, алгоритмы и протоколы для экзаменационного ответа.[pc+1](https://pc.ru/docs/network/routing)​

## Определение и роль маршрутизации

Маршрутизация решает задачу, которую не могла решить коммутация: при наличии в сети миллиардов устройств (как в Интернете) невозможно хранить в таблице коммутации MAC‑адреса всех узлов. Вместо этого используется IP‑адресация и логическое деление на подсети, при котором маршрутизаторы оперируют сетевыми адресами, а не отдельными узлами. Одна запись в таблице маршрутизации (например, 192.168.0.0/24) может охватывать сотни или тысячи адресов, что обеспечивает масштабируемость и иерархию сетевого пространства.[wikipedia+2](https://ru.wikipedia.org/wiki/%D0%9C%D0%B0%D1%80%D1%88%D1%80%D1%83%D1%82%D0%B8%D0%B7%D0%B0%D1%86%D0%B8%D1%8F)​

## Процесс IP‑маршрутизации (пошагово)

Когда пакет попадает на маршрутизатор, выполняются следующие шаги:[masich+1](https://masich.ru/images/lectures-pdf/ip/L3-2_IP-routing.pdf)​

1. **Приём кадра:** физический уровень получает сигнал, канальный уровень синтезирует кадр Ethernet и проверяет CRC.
    
2. **Проверка назначения:** MAC‑адрес в кадре сверяется; если он не совпадает с адресом маршрутизатора, кадр отбрасывается.
    
3. **Извлечение пакета:** демультиплексирование по полю типа; если IP, то заголовок передаётся на IP‑уровень.
    
4. **Поиск в таблице маршрутизации:** IP‑адрес назначения сравнивается с маршрутами; выбирается запись с **наиболее длинным совпадающим префиксом** (longest prefix match, LPM).
    
5. **Проверка TTL:** значение Time‑to‑Live уменьшается на 1; если оно становится 0, пакет отбрасывается, и отправляется ICMP‑сообщение об ошибке.
    
6. **Переинкапсуляция:** IP‑пакет помещается в новый Ethernet‑кадр с MAC‑адресами выходного интерфейса маршрутизатора и адреса следующего хопа (уточняется через ARP).
    
7. **Передача:** кадр отправляется на выходной интерфейс и далее к следующему маршрутизатору или сети назначения.[router+1](https://router.ru/blog/1927-technologies/article-19265/)​
    

Важно: IP‑адреса источника и назначения **не изменяются** при прохождении пакета через маршрутизаторы; меняются только MAC‑адреса на каждом шаге.[pc](https://pc.ru/docs/network/routing)​

## Таблица маршрутизации

Таблица маршрутизации (Routing Table) — это основной справочник, в котором хранятся записи вида:[router+1](https://router.ru/blog/1927-technologies/article-19265/)​

text

`[Административная дистанция]/[Метрика] Сетевой адрес via Следующий-хоп через Интерфейс`

Типичные коды (легенды) в таблице:[pc](https://pc.ru/docs/network/routing)​

- **C** — directly connected (подсети, напрямую подключённые к маршрутизатору).
    
- **S** — static route (статический маршрут, заданный администратором вручную).
    
- **R** — RIP.
    
- **D** — EIGRP.
    
- **O** — OSPF (intra‑area).
    
- **IA** — OSPF inter‑area (между зонами).
    

Пример записи таблицы:[pc](https://pc.ru/docs/network/routing)​

text

`C 192.168.1.0/24 is directly connected, FastEthernet0/0 S 192.168.2.0/24 [1/0] via 172.16.1.1 D 10.2.2.0/24 [90/156160] via 192.168.3.2, FastEthernet0/1 O 10.1.1.0/24 [110/2] via 192.168.1.2, FastEthernet0/0`

Маршрут по умолчанию (default route, 0.0.0.0/0) используется для пакетов, чьи адреса не совпадают ни с одной записью в таблице.[citforum+1](https://citforum.ru/nets/ito/2.shtml)​

## Принципы маршрутизации

В сетевых алгоритмах выделяют основные требования и подходы:[donstu+1](https://de.donstu.ru/CDOCourses/AII/POVT/opensystem/All.files/lec13.htm)​

- **Корректность и достоверность:** пакеты должны доставляться без циклических маршрутов; таблицы маршрутизации должны согласованно описывать сетевую топологию.
    
- **Оптимальность:** маршрут должен быть оптимален по выбранной метрике (количество хопов, задержка, пропускная способность, стоимость).
    
- **Сходимость:** после изменения топологии (обрыв линка, добавление маршрутизатора) алгоритм должен за приемлемое время прийти в устойчивое состояние.
    
- **Адаптивность:** алгоритм должен учитывать динамические изменения состояния каналов.
    
- **Масштабируемость и простота:** решение должно работать в крупных сетях с минимальными служебными расходами.
    

## Статическая маршрутизация

При статической маршрутизации маршруты задаются вручную администратором и остаются неизменны, пока их явно не отредактируют.[ibm+1](https://www.ibm.com/docs/ru/aix/7.1.0?topic=routing-static-dynamic)​

Синтаксис (Cisco):[habr](https://habr.com/ru/articles/335090/)​

text

`ip route <сетевой_адрес> <маска> <адрес_next_hop>`

Пример:[habr](https://habr.com/ru/articles/335090/)​

text

`ip route 192.168.2.0 255.255.255.0 10.0.1.1`

**Преимущества:**[ibm](https://www.ibm.com/docs/ru/aix/7.1.0?topic=routing-static-dynamic)​

- Простота реализации и полный контроль.
    
- Предсказуемость путей (нет непредвиденных переключений).
    
- Отсутствие служебного трафика (нет обмена маршрутной информацией).
    

**Недостатки:**[geeksforgeeks+1](https://www.geeksforgeeks.org/computer-networks/difference-between-static-and-dynamic-routing/)​

- Не масштабируется на крупных сетях.
    
- Требует ручного пересчёта при любом изменении топологии.
    
- Не реагирует на отказ линка; если линк падает, маршрут остаётся в таблице и трафик теряется.
    
- Неподходит для сложных сетей с резервированием.
    

Используется в небольших сетях, для резервных маршрутов или для выхода в Интернет через один провайдер.[ibm](https://www.ibm.com/docs/ru/aix/7.1.0?topic=routing-static-dynamic)​

## Динамическая маршрутизация

Динамическая маршрутизация использует протоколы, при помощи которых маршрутизаторы автоматически обмениваются информацией о сетевой топологии и строят таблицы маршрутизации.[ioriver+1](https://www.ioriver.io/blog/static-dynamic-routing)​

**Основные характеристики:**[interlir+1](https://interlir.com/ru/2024/06/28/osnovy-marshrutizacii-seti-i-ee-vazhnost/)​

- Автоматическое обнаружение сетей и путей доступа.
    
- Адаптивная реакция на изменения топологии (пересчёт маршрутов при сбоях).
    
- Использование метрик и алгоритмов для выбора оптимальных путей.
    
- Обмен маршрутной информацией между соседними маршрутизаторами.
    

**Классификация протоколов динамической маршрутизации:**[wikipedia](https://ru.wikipedia.org/wiki/%D0%9C%D0%B0%D1%80%D1%88%D1%80%D1%83%D1%82%D0%B8%D0%B7%D0%B0%D1%86%D0%B8%D1%8F)​

## IGP (Interior Gateway Protocols) — внутридоменные

- **Distance‑Vector:** RIP (протокол вектор расстояний).
    
- **Link‑State:** OSPF (протокол состояния канала), IS‑IS.
    
- **Hybrid:** EIGRP (гибридный; в своё время разработан Cisco).
    

## EGP (Exterior Gateway Protocols) — внешние

- **BGP** (Border Gateway Protocol) — используется между автономными системами в Интернете.
    

## Алгоритм Distance-Vector (RIP)

RIP (Routing Information Protocol) — классический протокол «вектор расстояний»; маршрутизаторы регулярно рассылают друг другу полные таблицы маршрутизации с метриками.[geeksforgeeks+1](https://www.geeksforgeeks.org/computer-networks/difference-between-static-and-dynamic-routing/)​

**Метрика:** количество хопов (number of hops); максимум 15 для RIPv2.[habr](https://habr.com/ru/articles/335090/)​

**Принцип работы:**[habr](https://habr.com/ru/articles/335090/)​

1. Маршрутизатор отправляет свой список известных сетей соседям каждые 30 секунд.
    
2. Каждый сосед добавляет 1 к метрике и пересылает информацию дальше.
    
3. Если маршрут недоступен, он анонсируется с метрикой 16 (poison reverse, отравление маршрута).
    

**Проблемы:**[habr](https://habr.com/ru/articles/335090/)​

- Медленная сходимость при изменении топологии (до 3–5 минут).
    
- Count‑to‑infinity: при разрыве линка маршруты долго считаются недостижимыми.
    
- Только один лучший маршрут; нет резервных путей.
    

**Конфигурация (Cisco):**[habr](https://habr.com/ru/articles/335090/)​

text

`router rip  version 2 no auto-summary network 192.168.1.0`

**Используется:** в старых, небольших сетях; образовательных целях.

## Алгоритм Hybrid (EIGRP)

EIGRP (Enhanced Interior Gateway Routing Protocol) — гибридный протокол, разработанный Cisco; сочетает элементы distance‑vector и link‑state подходов.[habr](https://habr.com/ru/articles/335090/)​

**Три таблицы EIGRP:**[habr](https://habr.com/ru/articles/335090/)​

1. **Neighbor Table** — список обнаруженных соседей EIGRP.
    
2. **Topology Table** — все маршруты от соседей с метриками.
    
3. **Routing Table** — отфильтрованные лучшие маршруты, готовые к использованию.
    

**Ключевые понятия:**[habr](https://habr.com/ru/articles/335090/)​

- **Successor** — маршрут с наименьшей Feasible Distance (FD); главный маршрут до целевой сети.
    
- **Feasible Successor (FS)** — резервный маршрут, который удовлетворяет условию _Advertised Distance (AD) соседа < FD текущего Successor_; гарантирует отсутствие циклов.
    
- **FD** (Feasible Distance) = AD соседа + метрика до соседа; общая стоимость пути.
    
- **AD** (Advertised Distance) = стоимость маршрута, анонсируемая соседом.
    

**Метрика:** EIGRP рассчитывает метрику как:

text

`Метрика = K1×Bandwidth + K3×Delay`

По умолчанию используются K1 и K3; Bandwidth — это минимальная пропускная способность на пути, Delay — задержка (в микросекундах).[habr](https://habr.com/ru/articles/335090/)​

**Преимущества EIGRP:**[habr](https://habr.com/ru/articles/335090/)​

- Быстрая сходимость (DUAL алгоритм).
    
- Feasible Successors обеспечивают быстрое переключение на резервный путь без пересчёта (быстрое восстановление).
    
- Неэквивалентная балансировка нагрузки (variance).
    

**Конфигурация:**[habr](https://habr.com/ru/articles/335090/)​

text

`router eigrp 1  network 192.168.1.0 0.0.0.255 no auto-summary`

**Используется:** в сетях построенных исключительно на оборудовании Cisco (так как EIGRP долгое время был проприетарным).

## Алгоритм Link-State (OSPF)

OSPF (Open Shortest Path First) — открытый стандарт link‑state протокола; каждый маршрутизатор строит полную топологическую карту сети и рассчитывает кратчайший путь алгоритмом Dijkstra.[habr](https://habr.com/ru/articles/335090/)​

**Ключевые компоненты:**[habr](https://habr.com/ru/articles/335090/)​

1. **LSA (Link-State Advertisements)** — объявления о состоянии каналов; передают информацию о соседях и подсетях каждого маршрутизатора.
    
2. **LSDB (Link-State Database)** — единая база данных состояния всех линков, построенная по информации от всех маршрутизаторов в зоне.
    
3. **SPF (Shortest Path First, алгоритм Dijkstra)** — вычисление дерева кратчайших путей от каждого маршрутизатора.
    

**Иерархия с выбранными ролями:**[habr](https://habr.com/ru/articles/335090/)​

- **DR (Designated Router)** — основной маршрутизатор в сегменте; собирает LSA от всех остальных.
    
- **BDR (Backup DR)** — резервный маршрутизатор; берёт на себя роль DR при его отказе.
    
- **DROTHER** — остальные маршрутизаторы на этом сегменте; отправляют LSA только DR и BDR.
    
- **ABR (Area Border Router)** — маршрутизатор на границе между зонами OSPF.
    

**Метрика:**[habr](https://habr.com/ru/articles/335090/)​

text

`Метрика (стоимость) = Reference_Bandwidth / Interface_Bandwidth`

По умолчанию Reference_Bandwidth = 100 Мбит/с, поэтому интерфейс Ethernet (100 Мбит/с) имеет стоимость 1, серийный линк 64 кбит/с — стоимость ~1562.[habr](https://habr.com/ru/articles/335090/)​

**Зоны OSPF:**[habr](https://habr.com/ru/articles/335090/)​

- **Area 0 (backbone)** — обязательная; связывает остальные зоны.
    
- **Остальные зоны** — подключаются только через ABR в area 0.
    
- Маршруты между зонами обозначаются кодом **IA** (inter-area).
    

**Конфигурация:**[habr](https://habr.com/ru/articles/335090/)​

text

`router ospf 1  network 192.168.1.0 0.0.0.255 area 0`

**Преимущества OSPF:**[habr](https://habr.com/ru/articles/335090/)​

- Быстрая сходимость (обновление сразу после изменения).
    
- Открытый стандарт (работает на оборудовании всех вендоров).
    
- Гибкое разделение на зоны для снижения нагрузки.
    
- Эквивалентная балансировка нагрузки (до 4 маршрутов).
    

**Используется:** универсально; де‑факто стандарт для мультивендорных сетей и крупных корпоративных сетей.

## Сравнение протоколов

|Параметр|RIP|EIGRP|OSPF|
|---|---|---|---|
|**Тип алгоритма**|Distance-Vector|Hybrid|Link-State|
|**Метрика**|Хопы (макс. 15)|Bandwidth + Delay|Bandwidth (стоимость)|
|**Сходимость**|Медленная (минуты)|Быстрая (сек)|Быстрая (сек)|
|**Наличие резервных путей**|Нет|Feasible Successors|SPF дерево|
|**Балансировка**|Только эквивалентная|Неэквивалентная (variance)|Эквивалентная (до 4)|
|**Масштабируемость**|Плохая (до 15 хопов)|Хорошая|Очень хорошая (иерархия)|
|**Открытый стандарт**|Да (RFC)|Нет (Cisco)|Да (RFC)|
|**Использование служебного трафика**|Высокое (каждые 30 сек)|Среднее|Среднее (зависит от стабильности)|

## Выбор протокола маршрутизации

- **Сети только на оборудовании Cisco:** EIGRP (быстрее, проще, рекомендуется для высоконагруженных сетей).
    
- **Мультивендорные сети:** OSPF (открытый стандарт, работает везде).
    
- **Интернет / крупные AS:** BGP.
    
- **Маленькие сети (до 15 хопов):** статическая маршрутизация + RIP (образовательные цели).
    
- **Современные большие корпоративные сети:** OSPF (универсальность, хорошая масштабируемость).
    

Часто администраторы выбирают OSPF даже в сетях Cisco, так как это обеспечивает гибкость на случай, если в будущем потребуется интегрировать оборудование других вендоров.[habr](https://habr.com/ru/articles/335090/)​
