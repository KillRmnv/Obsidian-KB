## **ЭКЗАМЕНАЦИОННЫЙ ОТВЕТ: Система доменных имен (DNS)**

## **1. ОПРЕДЕЛЕНИЕ И НАЗНАЧЕНИЕ DNS**

**DNS (Domain Name System)** — это распределенная система хранения и обработки информации о доменных зонах. DNS используется для соотнесения IP-адресов и доменными именами, более понятных и запоминающихся человеку. Система DNS служит своеобразной "телефонной книгой" интернета, которая преобразует имена хостов в IP-адреса.[cloud](https://cloud.ru/blog/domain-name-system)​Lektsiia-13-14.-Prikladnoi-uroven.pdf​

## **Основные характеристики DNS**

|Характеристика|Описание|
|---|---|
|**Назначение**|Преобразование доменных имен в IP-адреса и наоборот (обратное разрешение)[cloud](https://cloud.ru/blog/domain-name-system)​|
|**Транспортный протокол**|UDP (порт 53) и TCP (порт 53)[cloud](https://cloud.ru/blog/domain-name-system)​|
|**Архитектура**|Клиент-сервер|
|**Структура**|Распределенная иерархическая система[habr](https://habr.com/ru/companies/1cloud/articles/309018/)​|
|**Основные функции**|Кэширование, распределение нагрузки, делегирование доменов|

## **Зачем нужна DNS**

Вместо запоминания IP-адресов (например, 192.168.1.1), люди используют запоминаемые доменные имена (например, [www.google.com](http://www.google.com/)). DNS преобразует эти доменные имена в IP-адреса, которые компьютеры используют для взаимодействия.[1cloud+1](https://1cloud.ru/help/dns/dns_basics)​

## **2. ИЕРАРХИЧЕСКАЯ СТРУКТУРА DNS**

## **Пространство имен доменов (Domain Name Space)**

DNS имеет иерархическую структуру, похожую на дерево, которое "растет" из **корневого домена**:[habr](https://habr.com/ru/companies/1cloud/articles/309018/)​

```
                        . (корневой домен)
                        |
            ____________|____________
           |            |           |
          com           ru          org
          /|\          / \           |
        google yandex 1cloud      wikipedia

```

## **Уровни иерархии**

|Уровень|Описание|Примеры|
|---|---|---|
|**Корневой домен (Root)**|Самый верхний уровень иерархии, обозначается точкой (.)|.|
|**Домены верхнего уровня (TLD)**|Расширения доменов[kaspersky](https://www.kaspersky.ru/resource-center/definitions/dns)​|.com, .ru, .org, .net|
|**Домены второго уровня**|Основная часть доменного имени[habr](https://habr.com/ru/companies/1cloud/articles/309018/)​|google, yandex, 1cloud|
|**Поддомены (Subdomains)**|Домены, делегированные от основного домена[habr](https://habr.com/ru/companies/1cloud/articles/309018/)​|www, mail, ftp|

## **Пример полного доменного имени (FQDN)**

text

```bash
 
www.yandex.ru
│   │      │
│   │      └─ Домен первого уровня (TLD - .ru)
│   └──────── Домен второго уровня (yandex)
└──────────── Поддомен (www)

```

## **3. ТИПЫ DNS-СЕРВЕРОВ**

DNS использует четыре основных типа серверов:[kaspersky](https://www.kaspersky.ru/resource-center/definitions/dns)​

## **1. Корневой сервер (Root Nameserver)**

- Находится на самом верхнем уровне иерархии[kaspersky](https://www.kaspersky.ru/resource-center/definitions/dns)​
    
- Содержит информацию обо всех серверах TLD[cloud](https://cloud.ru/blog/domain-name-system)​
    
- Отвечает за перенаправление запросов на серверы доменов верхнего уровня
    
- В мире существует 13 корневых серверов[kaspersky](https://www.kaspersky.ru/resource-center/definitions/dns)​
    
- **Режим работы:** только итеративный (не рекурсивный)youtube​
    

## **2. Сервер доменов верхнего уровня (TLD - Top Level Domain)**

- Вступает в поиск на следующем этапе[kaspersky](https://www.kaspersky.ru/resource-center/definitions/dns)​
    
- Содержит информацию обо всех доменных расширениях (.com, .ru, .net и т.д.)[kaspersky](https://www.kaspersky.ru/resource-center/definitions/dns)​
    
- Отвечает за перенаправление на авторитетные серверы для конкретных доменов[kaspersky](https://www.kaspersky.ru/resource-center/definitions/dns)​
    
- Примеры: сервер для зоны .ru, сервер для зоны .com
    
- **Режим работы:** только итеративныйyoutube​
    

## **3. Авторитетный сервер имен (Authoritative Nameserver)**

- Осуществляет **последний этап поиска**[kaspersky](https://www.kaspersky.ru/resource-center/definitions/dns)​
    
- Содержит информацию, относящуюся к **конкретному доменному имени**[kaspersky](https://www.kaspersky.ru/resource-center/definitions/dns)​
    
- Может предоставить преобразующему серверу правильный IP-адрес[kaspersky](https://www.kaspersky.ru/resource-center/definitions/dns)​
    
- Отвечает за хранение ресурсных записей (A, CNAME, MX и др.) конкретного домена[1cloud](https://1cloud.ru/help/dns/dns_basics)​
    
- Пример: авторитетный сервер для домена google.com
    
- **Режим работы:** только итеративныйyoutube​
    

## **4. Преобразующий сервер (DNS Resolver / Recursive Resolver)**

- Работает в **рекурсивном режиме**[skydns+1](https://skydns.ru/blog/riekursivnyi-i-avtoritietnyi-dns-osnovnyie-printsipy-raboty-i-razlichiia)​
    
- Обслуживает запросы от клиентов (браузеров, операционных систем)[cloud](https://cloud.ru/blog/domain-name-system)​
    
- Самостоятельно проходит весь путь от корневого сервера до авторитетного[skydns](https://skydns.ru/blog/riekursivnyi-i-avtoritietnyi-dns-osnovnyie-printsipy-raboty-i-razlichiia)​
    
- **Кэширует** результаты для ускорения будущих запросов[1cloud](https://1cloud.ru/help/dns/dns_basics)​
    
- Обычно расположен в локальной сети организации или провайдераyoutube​
    
- Примеры: 8.8.8.8 (Google DNS), 1.1.1.1 (Cloudflare DNS)

## **4. ПРОЦЕСС РАЗРЕШЕНИЯ ДОМЕННОГО ИМЕНИ**

## **Типичная сессия DNS запроса на примере [www.yandex.ru](http://www.yandex.ru/)**

**Шаги процесса**:[habr+2](https://habr.com/ru/companies/1cloud/articles/309018/)​

1. **Клиент отправляет запрос**
    
    - Пользователь вводит в браузер: [www.yandex.ru](http://www.yandex.ru/)
        
    - Браузер отправляет запрос DNS-резолверу (преобразующему серверу)
        
2. **Запрос на корневой сервер**
    
    - DNS-резолвер отправляет запрос на корневой сервер
        
    - Корневой сервер отвечает: "Я не знаю этот домен, но вот тебе адрес сервера TLD для .ru"
        
3. **Запрос на сервер TLD (.ru)**
    
    - Резолвер отправляет запрос серверу TLD для зоны .ru
        
    - Сервер TLD отвечает: "Я не знаю этот домен, но вот тебе адрес авторитетного сервера для yandex.ru"
        
4. **Запрос на авторитетный сервер**
    
    - Резолвер отправляет запрос авторитетному серверу для yandex.ru
        
    - Авторитетный сервер отвечает: "Адрес [www.yandex.ru](http://www.yandex.ru/) — это 77.88.55.242"
        
5. **Кэширование и возврат результата**
    
    - DNS-резолвер кэширует результат (на определенное время TTL)
        
    - Резолвер отправляет результат обратно клиенту
        
    - Браузер получает IP-адрес и может подключиться к серверу
        

**Диаграмма процесса**:[habr](https://habr.com/ru/companies/1cloud/articles/309018/)​

```bash
Клиент (www.yandex.ru)
    ↓
Резолвер (рекурсивный)
    ↓ (итеративный запрос)
Корневой сервер → "Спросите TLD сервер .ru"
    ↓
TLD сервер .ru → "Спросите авторитетный сервер yandex.ru"
    ↓
Авторитетный сервер yandex.ru → "77.88.55.242"
    ↓
Резолвер кэширует результат
    ↓
Возвращает результат клиенту

```

## **5. РЕКУРСИВНЫЙ И ИТЕРАТИВНЫЙ РЕЖИМЫ**

## **Рекурсивный режим (Recursive Query)**

**Как работает**:[1cloud](https://1cloud.ru/help/dns/dns_basics)​youtube​

- **Клиент** отправляет серверу DNS запрос с требованием дать **окончательный ответ**[ya](https://ya.ru/neurum/c/tehnologii/q/kak_rabotaet_rekursivnaya_i_iterativnaya_obrabotka_cf03f043)​
    
- **DNS-сервер**, получивший запрос, **самостоятельно** опрашивает все узлы в порядке убывания уровней зон[1cloud](https://1cloud.ru/help/dns/dns_basics)​
    
- Сервер обращается к другим DNS-серверам за нужной информацией[ya](https://ya.ru/neurum/c/tehnologii/q/kak_rabotaet_rekursivnaya_i_iterativnaya_obrabotka_cf03f043)​
    
- Сервер возвращает готовый ответ клиенту[1cloud](https://1cloud.ru/help/dns/dns_basics)​
    

**Характеристики**:youtube​

- Используется между клиентом и резолвером
    
- Флаг **RD (Recursion Desired)** устанавливается в запросе
    
- Флаг **RA (Recursion Available)** указывает, может ли сервер работать рекурсивно
    
- Требует много ресурсов от сервера
    

**Кто использует:**

- DNS резолверы (преобразующие серверы)
    
- Обычно расположены в локальной сети организации или провайдераyoutube​
    

## **Итеративный режим (Iterative Query)**

**Как работает**:[1cloud](https://1cloud.ru/help/dns/dns_basics)​youtube​

- **Клиент** отправляет серверу DNS запрос с требованием дать наилучший ответ **без обращений к другим серверам**[ya](https://ya.ru/neurum/c/tehnologii/q/kak_rabotaet_rekursivnaya_i_iterativnaya_obrabotka_cf03f043)​
    
- **DNS-сервер** предоставляет ответ, **только если сам его знает**[ya](https://ya.ru/neurum/c/tehnologii/q/kak_rabotaet_rekursivnaya_i_iterativnaya_obrabotka_cf03f043)​
    
- Если сервер не знает ответа, он возвращает **ссылку на другой DNS-сервер**, у которого есть более точная информация[1cloud](https://1cloud.ru/help/dns/dns_basics)​
    
- **Клиент сам** должен обратиться к рекомендованному серверу[ya](https://ya.ru/neurum/c/tehnologii/q/kak_rabotaet_rekursivnaya_i_iterativnaya_obrabotka_cf03f043)​
    

**Характеристики**:youtube​

- Используется между резолвером и авторитетными серверами
    
- Резолвер активно участвует в процессе
    
- Отправляет последовательные запросы на основе полученных ссылок
    
- Экономит ресурсы сервера
    

**Кто использует:**

- Корневые серверы
    
- Серверы TLD
    
- Авторитетные серверы именyoutube​
    
- Обслуживают крупные зоны с большим количеством запросовyoutube​
    

## **Сравнение режимов**

|Аспект|Рекурсивный|Итеративный|
|---|---|---|
|**Инициатор**|Клиент требует полного ответа|Клиент требует наилучшего ответа|
|**Обращения к серверам**|Сервер сам обращается к другим серверам|Клиент сам обращается к других серверам|
|**Ответ**|Готовый окончательный результат|Ссылка на другой сервер или результат|
|**Требуемые ресурсы**|Много (сервер работает за клиента)|Мало (клиент делает работу)|
|**Использование**|Резолверы в локальных сетях|Корневые и авторитетные серверы|
|**Флаги DNS**|RD и RA установлены|RD не установлен|

## **Структура DNS-записи**

text

`Имя домена    TTL    Класс  Тип   Данные www.example.com 3600  IN     A     192.168.1.1`

- **Имя домена** — доменное имя, к которому относится запись
    
- **TTL (Time To Live)** — время в секундах, на которое сервер кэширует запись[1cloud](https://1cloud.ru/help/dns/dns_basics)​
    
- **Класс** — обычно IN (Internet)
    
- **Тип** — тип записи (A, MX, CNAME и т.д.)
    
- **Данные** — конкретные данные для данного типа

## **7. ОСНОВНЫЕ ХАРАКТЕРИСТИКИ DNS**

## **Распределенность хранения и управления**

- Каждый DNS-сервер хранит информацию только по делегированным ему доменам[1cloud](https://1cloud.ru/help/dns/dns_basics)​
    
- Ответственность за различные узлы дерева доменных имен несут разные люди/организации[1cloud](https://1cloud.ru/help/dns/dns_basics)​
    
- Это позволяет масштабировать систему без единой точки отказа
    

## **Кэширование данных**

- DNS-сервер кэширует результаты запросов на время **TTL (Time To Live)**[1cloud](https://1cloud.ru/help/dns/dns_basics)​
    
- Кэширование снижает нагрузку на сервер и ускоряет разрешение имен
    
- При истечении TTL информация удаляется из кэша
    

## **Делегирование доменов**

- Родительский домен может делегировать поддомен другому DNS-серверу[habr](https://habr.com/ru/companies/1cloud/articles/309018/)​
    
- Это позволяет распределить управление большими доменными пространствами[habr](https://habr.com/ru/companies/1cloud/articles/309018/)​
    
- Пример: .ru делегирует yandex.ru авторитетному серверу Яндекса

## **8. ПРОЦЕСС ПОИСКА DNS (Resolution Process)**

## **Полный процесс на примере поиска [www.example.com](http://www.example.com/)**

1. **Клиент проверяет локальный кэш**
    
    - ОС и браузер проверяют, есть ли результат в локальном кэше
        
    - Если есть, используется кэшированный результат
        
2. **Запрос к резолверу**
    
    - Если результата нет в локальном кэше, клиент отправляет рекурсивный запрос к резолверу
        
3. **Проверка кэша резолвера**
    
    - Резолвер проверяет свой кэш
        
    - Если результат есть, возвращает его клиенту
        
4. **Итеративный поиск по иерархии**
    
    - Резолвер отправляет итеративный запрос на корневой сервер
        
    - Корневой сервер возвращает адрес TLD сервера
        
    - Резолвер отправляет запрос TLD серверу
        
    - TLD сервер возвращает адрес авторитетного сервера
        
    - Резолвер отправляет запрос авторитетному серверу
        
    - Авторитетный сервер возвращает IP-адрес
        
5. **Кэширование результата**
    
    - Резолвер кэширует результат на время TTL
        
6. **Возврат результата клиенту**
    
    - Резолвер отправляет IP-адрес обратно клиенту
        
    - Клиент кэширует результат и использует его

## **9. БЕЗОПАСНОСТЬ DNS**

## **Уязвимости DNS**

1. **Cache Poisoning (Отравление кэша)**
    
    - Злоумышленник внедряет поддельные записи в кэш DNS-сервера[arxiv](https://arxiv.org/abs/2312.04114)​
        
    - Приводит к перенаправлению пользователей на вредоносные сайты
        
2. **DNS Spoofing (Подделка DNS)**
    
    - Перехват DNS-запроса и отправка поддельного ответа
        
3. **DNS Amplification Attack**
    
    - Использование DNS-серверов для усиления DDoS-атак
        
4. **DNS Hijacking**
    
    - Перенаправление трафика путем изменения DNS-настроек
        

## **Меры безопасности**

|Мера|Описание|
|---|---|
|**DNSSEC (DNS Security Extensions)**|Цифровая подпись для верификации подлинности DNS-ответов[arxiv](http://arxiv.org/pdf/2408.00968.pdf)​|
|**DNSSEC+**|Улучшенная версия DNSSEC с лучшей безопасностью[arxiv](http://arxiv.org/pdf/2408.00968.pdf)​|
|**Firewall правила**|Блокирование подозрительных DNS-запросов|
|**Rate limiting**|Ограничение количества запросов от одного источника|
|**DoH (DNS over HTTPS)**|Передача DNS-запросов по защищенному HTTPS-каналу|
|**DoT (DNS over TLS)**|Передача DNS-запросов по защищенному TLS-каналу|

## **10. ПРАКТИЧЕСКОЕ ПРИМЕНЕНИЕ DNS**

## **Команды для проверки DNS**

**nslookup** — поиск IP-адреса по доменному имени:

bash

`nslookup www.yandex.ru`

или с указанием конкретного DNS-сервера:

bash

`nslookup www.yandex.ru 8.8.8.8`

**dig** — детальная информация о DNS-запросе:

bash

`dig www.yandex.ru`

**host** — простой способ получить IP-адрес:

bash

`host www.yandex.ru`

## **Заключение**

**DNS (Domain Name System)** — это фундаментальная система интернета, которая преобразует запоминаемые доменные имена в числовые IP-адреса. Система работает на основе распределенной иерархической архитектуры с четырьмя типами серверов: корневыми, серверами TLD, авторитетными серверами и резолверами.

**Ключевые особенности DNS:**

- **Рекурсивный режим** — резолверы полностью берут на себя работу по разрешению имен
    
- **Итеративный режим** — авторитетные серверы предоставляют ссылки, а клиент сам ищет дальше
    
- **Кэширование** — ускоряет разрешение часто запрашиваемых доменов
    
- **Делегирование** — позволяет масштабировать систему без единой точки отказа
    

DNS остается критически важной инфраструктурой интернета, и её безопасность постоянно улучшается благодаря протоколам типа DNSSEC, DoH и DoT.
