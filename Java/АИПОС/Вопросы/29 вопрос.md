NAT (Network Address Translation) — технология трансляции сетевых адресов, которая позволяет устройству (маршрутизатору, межсетевому экрану) преобразовывать IP‑адреса в заголовках пакетов при их прохождении, обеспечивая связь между сетями с приватными адресами и публичным Интернетом, а также решая проблему дефицита IPv4‑адресов и повышая безопасность сети путём скрытия внутренней топологии. Протокол работает на третьем и четвёртом уровнях модели OSI и является критически важным компонентом современной сетевой инфраструктуры.Lektsiia-6-7.-Setevoi-uroven.pdf​[ciscotips](https://ciscotips.ru/nat)​

## Проблема и назначение NAT

С момента истощения пула публичных IPv4‑адресов (что произошло в начале 2010‑х годов), организациям требуется способ подключить большое количество устройств к Интернету, используя ограниченное количество публичных адресов. NAT решает эту проблему, позволяя множеству устройств внутри локальной сети (с приватными адресами) выходить в Интернет через один или несколько публичных IP‑адресов на маршрутизаторе.[ciscotips](https://ciscotips.ru/nat)​Lektsiia-6-7.-Setevoi-uroven.pdf​

**Основные задачи NAT:**[linkmeup.gitbook+1](https://linkmeup.gitbook.io/sdsm/5.-acl-i-nat/01-nat)​

- Трансляция приватных адресов в публичные для выхода в Интернет.
    
- Скрытие внутренней структуры сети от внешних узлов.
    
- Повышение безопасности через фильтрацию и контроль трафика.
    
- Экономия публичного адресного пространства IPv4.
    

## Архитектура и термины NAT

NAT‑устройство имеет два интерфейса и оперирует четырьмя типами адресов:[ciscotips](https://ciscotips.ru/nat)​

|Термин|Описание|Пример|
|---|---|---|
|**Inside Local (IL)**|Приватный адрес устройства внутри локальной сети; адрес, который будет транслирован|192.168.1.5|
|**Inside Global (IG)**|Публичный адрес, на который транслируется Inside Local; видим из Интернета|203.0.113.5|
|**Outside Local (OL)**|Адрес удалённого сервера, известный локальной сети (часто приватный)|10.0.0.1 (если сервер тоже за NAT)|
|**Outside Global (OG)**|Публичный адрес удалённого сервера в Интернете|8.8.8.8|

**Процесс трансляции исходящего трафика:**[ciscotips](https://ciscotips.ru/nat)​

1. Клиент внутри сети отправляет пакет с адресом отправителя IL (например, 192.168.1.5) и адресом получателя OG (8.8.8.8).
    
2. NAT‑устройство заменяет адрес отправителя IL на IG (203.0.113.5) и запоминает это преобразование в таблице трансляции.
    
3. Пакет отправляется на IG и далее в Интернет.
    
4. Ответный пакет приходит на IG (203.0.113.5).
    
5. NAT‑устройство смотрит в таблицу и видит, что этот адрес соответствует IL (192.168.1.5), заменяет адрес назначения и отправляет пакет клиенту.
    

**Процесс трансляции входящего трафика (Destination NAT):**[eltexcm](https://eltexcm.ru/baza-znanij/esr/primer-nastrojki/servisy-nat-dhcp-dns-i-t.d/esr-destanation-nat-nat-hairpinning.html)​

1. Входящий пакет приходит на публичный адрес (IG) маршрутизатора, например на порт 80.
    
2. NAT‑устройство смотрит в таблицу и видит, что порт 80 на адресе IG должен быть переведён на адрес внутреннего веб‑сервера (IL).
    
3. Адрес назначения заменяется на IL, пакет отправляется во внутреннюю сеть.
    
4. Ответный пакет от сервера обратно преобразуется (источник меняется с IL на IG) и отправляется в Интернет.
    

## Типы NAT

## 1. Static NAT (Статический NAT)

При статическом NAT администратор явно определяет соответствие один‑к‑одному между Inside Local и Inside Global адресами.[ciscozine+1](https://www.ciscozine.com/nat-and-pat-a-complete-explanation/)​

**Характеристики:**[ciscozine+1](https://www.ciscozine.com/nat-and-pat-a-complete-explanation/)​

- Каждый внутренний адрес постоянно связан с одним публичным адресом.
    
- Одна запись на маршрутизаторе существует постоянно (до её удаления).
    
- Используется для публичных серверов, которые должны быть доступны из Интернета.
    
- Требует достаточного количества публичных адресов (1 публичный на 1 приватный).
    

**Пример конфигурации (Cisco):**[linkmeup.gitbook](https://linkmeup.gitbook.io/sdsm/5.-acl-i-nat/01-nat)​

text

`ip nat inside source static 192.168.1.10 203.0.113.10`

**Применение:** веб‑сервер внутри сети, работающий на фиксированном приватном адресе 192.168.1.10, становится доступным из Интернета по публичному адресу 203.0.113.10.

## 2. Dynamic NAT (Динамический NAT)

При динамическом NAT администратор определяет пул публичных адресов и правила отбора приватных адресов, которые будут транслироваться, но конкретное соответствие между ними выбирает маршрутизатор.[ciscozine+1](https://www.ciscozine.com/nat-and-pat-a-complete-explanation/)​

**Характеристики:**[ciscotips+1](https://ciscotips.ru/nat)​

- Диапазон (пул) публичных адресов, например 203.0.113.1–203.0.113.20.
    
- Правило доступа (ACL) определяет, какие приватные адреса подлежат трансляции.
    
- Трансляции создаются динамически при первом пакете с соответствующим адресом.
    
- Трансляции удаляются из таблицы после истечения таймера бездействия (обычно через несколько минут).
    
- Требует наличия свободного адреса в пуле; если все адреса заняты, новые соединения блокируются.
    

**Пример конфигурации (Cisco):**[linkmeup.gitbook](https://linkmeup.gitbook.io/sdsm/5.-acl-i-nat/01-nat)​

text

`access-list 1 permit 192.168.1.0 0.0.0.255      # какие адреса транслировать ip nat pool POOL 203.0.113.1 203.0.113.20 netmask 255.255.255.0 ip nat inside source list 1 pool POOL`

**Применение:** несколько рабочих станций в локальной сети могут выходить в Интернет, поочередно используя адреса из пула. Например, если в пуле 20 адресов, но клиентов 100, то только 20 из них могут быть активны одновременно.

## 3. PAT (Port Address Translation) / Перегрузка NAT

PAT — частный случай NAT, который модифицирует как адреса (L3), так и порты (L4), позволяя множеству внутренних узлов транслироваться на один публичный адрес.[wikipedia+1](https://en.wikipedia.org/wiki/Network_address_translation)​

**Характеристики:**[wikipedia+1](https://en.wikipedia.org/wiki/Network_address_translation)​

- Один публичный адрес может обслуживать тысячи внутренних клиентов.
    
- Каждому клиенту выделяется уникальный номер порта на публичном адресе.
    
- Маршрутизатор использует пары (адрес, порт) для отличия одного клиента от другого при получении ответных пакетов.
    
- Максимум одновременных соединений: 65535 (количество портов).
    
- Наиболее экономичное решение для дефицита публичных адресов.
    

**Пример:**[ciscotips](https://ciscotips.ru/nat)​

text

`Клиент 1 (192.168.1.10:1024) → транслируется → (203.0.113.1:5001) Клиент 2 (192.168.1.11:1024) → транслируется → (203.0.113.1:5002) Клиент 3 (192.168.1.12:1024) → транслируется → (203.0.113.1:5003)`

Все три клиента используют один и тот же публичный IP (203.0.113.1), но разные порты.

**Конфигурация (Cisco):**[linkmeup.gitbook](https://linkmeup.gitbook.io/sdsm/5.-acl-i-nat/01-nat)​

text

`access-list 1 permit 192.168.1.0 0.0.0.255 ip nat inside source list 1 interface Fa0/0 overload`

Ключевое слово `overload` включает режим PAT (перегрузки).

## 4. Static PAT

Комбинация Static NAT и PAT для проброса портов: определённый порт на публичном адресе всегда транслируется на конкретный порт внутреннего адреса.[ciscozine+1](https://www.ciscozine.com/nat-and-pat-a-complete-explanation/)​

**Пример конфигурации:**[linkmeup.gitbook](https://linkmeup.gitbook.io/sdsm/5.-acl-i-nat/01-nat)​

text

`ip nat inside source static tcp 192.168.1.10 80 203.0.113.10 8080`

Входящие TCP‑пакеты на публичный адрес 203.0.113.10 порт 8080 будут перенаправлены на внутренний адрес 192.168.1.10 порт 80.

## Таблица трансляции (Translation Table)

NAT‑устройство ведёт таблицу активных трансляций, в которой хранится информация о соответствиях между локальными и глобальными адресами.[wikipedia+1](https://en.wikipedia.org/wiki/Network_address_translation)​

**Структура записи:**

text

`Proto Inside_Local    Inside_Global    Outside_Local    Outside_Global     State tcp   192.168.1.10:1024  203.0.113.1:5001  8.8.8.8:53  8.8.8.8:53        ESTABLISHED`

**Просмотр таблицы трансляции (Cisco):**[ciscotips](https://ciscotips.ru/nat)​

text

`Router#show ip nat translations Router#show ip nat translation verbose`

## NAT Hairpinning

NAT Hairpinning (волосяная заколка) — технология, позволяющая клиентам внутри сети обращаться к серверу, находящемуся также во внутренней сети, используя публичный адрес маршрутизатора вместо приватного адреса сервера.[eltexcm](https://eltexcm.ru/baza-znanij/esr/primer-nastrojki/servisy-nat-dhcp-dns-i-t.d/esr-destanation-nat-nat-hairpinning.html)​

**Проблема:** клиент в сети хочет подключиться к веб‑сервису на адресе [www.example.com](http://www.example.com/), однако DNS возвращает публичный IP маршрутизатора, а не приватный адрес сервера.

**Решение:** маршрутизатор должен изменить не только адрес назначения (Destination NAT), но и адрес источника (Source NAT), чтобы ответные пакеты от сервера могли вернуться к клиенту.

**Конфигурация (Eltex):**[eltexcm](https://eltexcm.ru/baza-znanij/esr/primer-nastrojki/servisy-nat-dhcp-dns-i-t.d/esr-destanation-nat-nat-hairpinning.html)​

text

`nat destination  ruleset loopback  from zone trusted   rule 10    match protocol tcp    match destination-address Net_pub    match destination-port Srv_http    action destination-nat pool Server_ip    enable`

## Проблемы и ограничения NAT

|Проблема|Описание|
|---|---|
|**Нарушение сквозного принципа IP**|NAT нарушает принцип, что адрес остаётся неизменным при прохождении пакета через сеть|
|**Проблемы с IPsec**|Шифрование заголовков IPsec конфликтует с модификацией адресов NAT|
|**Проблемы с протоколами, встраивающими адреса**|FTP, SIP, RTSP и другие протоколы встраивают IP‑адреса в тело пакета; NAT не может их обновить|
|**Невозможность входящих соединений**|Сервисы за NAT не могут принимать входящие соединения без явного проброса портов|
|**Задержки и нагрузка**|NAT‑обработка требует ресурсов и может вызвать задержку|

## Преимущества и недостатки NAT

|Характеристика|Статус|
|---|---|
|**Экономия публичных адресов**|✅ Позволяет большому числу хостов использовать один публичный адрес (PAT)|
|**Безопасность**|✅ Скрывает внутреннюю топологию; действует как простой межсетевой экран|
|**Совместимость**|✅ Работает с существующими приватными сетями|
|**Стоимость**|✅ Экономит расходы на публичные адреса|
|**Сложность диагностики**|❌ Затрудняет трассировку соединений и отладку|
|**Проблемы с некоторыми приложениями**|❌ Несовместимо с приложениями, которые встраивают адреса в данные|
|**Технический долг**|❌ NAT — временное решение; долгосрочное решение — IPv6|

## Переход на IPv6

NAT была вынужденной мерой для борьбы с дефицитом адресов IPv4. IPv6 с его адресным пространством в 21282^{128}2128 адресов устраняет необходимость в NAT, обеспечивая уникальный адрес для каждого устройства и поддерживая сквозную архитектуру Интернета.Lektsiia-6-7.-Setevoi-uroven.pdf​[ciscotips](https://ciscotips.ru/nat)​

Однако полный переход на IPv6 остаётся неполным, и NAT по-прежнему широко используется в сетях по всему миру для совместимости между IPv4 и IPv6 (технология NAT64).
