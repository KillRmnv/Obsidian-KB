# Стек протоколов TCP/IP

**TCP/IP** (Transmission Control Protocol / Internet Protocol) — это набор (стек) сетевых протоколов передачи данных, используемых в сетях Интернет.

В отличие от модели OSI, которая является теоретической, TCP/IP — это **промышленный стандарт**. Исторически он был разработан Министерством обороны США (поэтому его иногда называют моделью **DoD** — Department of Defense).

## 1. Структура стека (4 уровня)

Классическая модель TCP/IP состоит из **4 уровней**.  
_Важно:_ На экзамене обязательно нужно сопоставить их с 7 уровнями модели OSI.

|Уровень TCP/IP|Соответствие OSI|Основные функции|
|---|---|---|
|**4. Прикладной** (Application)|5, 6, 7 (Сеансовый, Представления, Прикладной)|Взаимодействие с пользователем, кодирование данных, управление сессиями.|
|**3. Транспортный** (Transport)|4 (Транспортный)|Связь «хост-хост», надежность, порты.|
|**2. Межсетевой** (Internet)|3 (Сетевой)|Логическая адресация (IP), маршрутизация.|
|**1. Уровень сетевого доступа** (Network Access / Link)|1, 2 (Физический, Канальный)|Физическая передача данных, MAC-адресация.|

---

## 2. Подробное описание уровней и протоколов

## 1. Уровень сетевого доступа (Network Access Layer)

Самый нижний уровень. Отвечает за физическое подключение к носителю и передачу данных между двумя устройствами в одной сети.

- **Функции:** Управление драйверами устройств, физическая адресация (MAC), инкапсуляция IP-пакетов в кадры.
    
- **Протоколы и технологии:** Ethernet (802.3), Wi-Fi (802.11), PPP, DSL.
    
- **Особенность:** Стек TCP/IP практически не регламентирует физический уровень, он просто умеет работать с популярными стандартами (Ethernet и др.).
    

## 2. Межсетевой уровень (Internet Layer)

Ключевой уровень стека («сердце» Интернета). Отвечает за передачу данных между **разными** сетями.

- **Функции:** Логическая адресация (IP-адреса), маршрутизация (выбор лучшего пути для пакета), фрагментация пакетов.
    
- **Основные протоколы:**
    
    - **IP (Internet Protocol):** Основной протокол доставки. Бывает IPv4 и IPv6. Работает по принципу «best effort» (лучшая попытка) — не гарантирует доставку, но старается доставить.
        
    - **ICMP:** Управляющий протокол. Используется для диагностики и сообщения об ошибках (утилиты `ping`, `traceroute`).
        
    - **ARP (Address Resolution Protocol):** Очень важный протокол «клея» между 2 и 3 уровнями. Он находит MAC-адрес устройства по его известному IP-адресу.
        

## 3. Транспортный уровень (Transport Layer)

Обеспечивает логическую связь между процессами (программами) на разных хостах.

- **Функции:** Разделение потоков данных с помощью **портов**, контроль целостности.
    
- **Два главных протокола:**
    
    1. **TCP (Transmission Control Protocol):**
        
        - _Надежный:_ Гарантирует доставку данных, правильный порядок пакетов и отсутствие дублей.
            
        - _С установлением соединения:_ Перед передачей происходит «трехкратное рукопожатие» (3-way handshake).
            
        - _Тяжелый:_ Медленнее из-за проверок.
            
        - _Примеры:_ Загрузка веб-страниц, почта, передача файлов.
            
    2. **UDP (User Datagram Protocol):**
        
        - _Ненадежный:_ Отправляет пакеты («датаграммы») без проверки доставки. Потерялся пакет — ну и ладно.
            
        - _Без соединения:_ Просто «выстрелил и забыл».
            
        - _Быстрый:_ Нет накладных расходов на подтверждения.
            
        - _Примеры:_ Потоковое видео, онлайн-игры, голосовая связь (VoIP), DNS-запросы.
            

## 4. Прикладной уровень (Application Layer)

Объединяет функции трех верхних уровней OSI. Это протоколы, которые используют наши программы (браузеры, почтовые клиенты) для работы с сетью.

- **Основные протоколы:**
    
    - **HTTP / HTTPS:** Просмотр веб-страниц.
        
    - **DNS:** Система доменных имен (преобразует `google.com` в IP-адрес).
        
    - **DHCP:** Автоматическая раздача IP-адресов клиентам.
        
    - **FTP:** Передача файлов.
        
    - **SMTP / POP3 / IMAP:** Электронная почта.
        
    - **SSH:** Безопасное удаленное управление сервером.

## 3. Адресация в стеке TCP/IP

Чтобы доставить данные конкретному приложению на конкретном компьютере, используются три типа адресов:

1. **MAC-адрес (Физический):** Работает на уровне сетевого доступа. Нужен для доставки кадра внутри локальной сети (от свитча к компьютеру).
    
2. **IP-адрес (Логический):** Работает на межсетевом уровне. Нужен для доставки пакета через весь мир от отправителя к получателю. Уникален в глобальной сети (для публичных IP).
    
3. **Порт (Port):** Работает на транспортном уровне. Нужен, чтобы понять, какой именно программе отдать данные (например, веб-сервер слушает порт 80, почта — 25).
    

> **Резюме для ответа:**  
> Стек TCP/IP — это 4-уровневая модель, являющаяся основой Интернета. Главное отличие от OSI — она более практична, объединяет три верхних уровня в один (Прикладной) и два нижних в один (Сетевого доступа). Основные «рабочие лошадки» стека — это протоколы IP (маршрутизация), TCP (надежность) и UDP (скорость).
