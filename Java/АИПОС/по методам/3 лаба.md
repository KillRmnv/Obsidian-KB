Команда `show ip route` на маршрутизаторах Cisco отображает **таблицу маршрутизации** (Routing Table). Первая секция вывода, которую вы привели, содержит **коды маршрутизации** — однобуквенные обозначения, указывающие, каким образом данный маршрут был добавлен в таблицу.

Вот подробное объяснение обозначений, которые видны в вашем выводе:

## 📜 Основные коды маршрутов (Routing Codes)

Эти коды появляются в начале каждой строки, указывая на источник маршрута:

|**Код**|**Название (English)**|**Значение**|**Описание**|
|---|---|---|---|
|**L**|Local|**Локальный**|Маршрут, созданный для **конкретного IP-адреса** на интерфейсе маршрутизатора. Используется для более быстрого принятия решения о локально адресованных пакетах. В вашем примере: `192.168.10.1/32` и `192.168.11.1/32` — это локальные адреса интерфейсов.|
|**C**|Connected|**Подключенный**|Маршрут к сети, которая **напрямую подключена** к интерфейсу маршрутизатора. Создается автоматически, когда интерфейс активируется и получает IP-адрес. В вашем примере: `192.168.10.0/24`, `192.168.11.0/24`, `209.165.200.224/30`.|
|**D**|EIGRP|**Маршрут EIGRP**|Маршрут, полученный с помощью протокола динамической маршрутизации **EIGRP** (Enhanced Interior Gateway Routing Protocol). В вашем примере: `10.1.1.0/24` и `10.1.2.0/24`.|
|**S**|Static|**Статический**|Маршрут, вручную настроенный администратором.|
|**R**|RIP|**Маршрут RIP**|Маршрут, полученный с помощью протокола динамической маршрутизации **RIP** (Routing Information Protocol).|
|**O**|OSPF|**Маршрут OSPF**|Маршрут, полученный с помощью протокола динамической маршрутизации **OSPF** (Open Shortest Path First).|
|**B**|BGP|**Маршрут BGP**|Маршрут, полученный с помощью протокола динамической маршрутизации **BGP** (Border Gateway Protocol).|
|*****|Candidate Default|**Кандидат в маршрут по умолчанию**|Маршрут, который может быть использован в качестве маршрута по умолчанию (`0.0.0.0/0`).|

## 🧭 Расшифровка маршрута EIGRP (D)

Строка маршрута EIGRP (`D 10.1.1.0/24 [90/2170112] via 209.165.200.226, 00:00:05, Serial0/0/0`) содержит следующую важную информацию:

### 1. Адрес сети и маска

- `10.1.1.0/24`: **Сеть назначения** и **длина префикса** (маска).
    

### 2. Административная дистанция и Метрика (в скобках)

- `[90/2170112]`: Это два ключевых показателя для выбора маршрута:
    
    - **90** (слева) — это **Административная дистанция (Administrative Distance, AD)**. Это мера **доверия** к источнику маршрута. Чем меньше число, тем выше доверие. AD для EIGRP по умолчанию равна 90.
        
    - **2170112** (справа) — это **Метрика** (Cost/Metric). Это значение, рассчитанное протоколом EIGRP (на основе пропускной способности, задержки и т.д.), которое используется для определения **лучшего маршрута** к этой сети.
        

### 3. Следующий переход (Next Hop)

- `via 209.165.200.226`: Это **IP-адрес следующего маршрутизатора (Next Hop)**, которому нужно передать пакет, чтобы он попал в сеть `10.1.1.0/24`.
    

### 4. Время и Интерфейс

- `00:00:05`: **Время**, прошедшее с момента последнего обновления этого маршрута.
    
- `Serial0/0/0`: **Локальный исходящий интерфейс** маршрутизатора, через который будет отправлен пакет, чтобы достичь Next Hop'а `209.165.200.226`.

## 🔢 Другие технические обозначения

В верхней части вывода также присутствуют другие специфические обозначения, связанные с протоколами и функцией VLSM:

- `10.0.0.0/8 is **variably subnetted**, 2 subnets, 2 masks`: Это означает, что для сети `10.0.0.0/8` используются **маски переменной длины (VLSM)**. В таблице есть 2 подсети этой главной сети, использующие 2 разные маски.
    
- `EIGRP, EX - EIGRP external`: Обозначение для **внешних маршрутов EIGRP** (маршруты, перераспределенные из другого протокола или статики в EIGRP).
    
- `O - OSPF, IA - OSPF inter area`: Обозначения для маршрутов, полученных через OSPF:
    
    - **O** — внутри одной зоны OSPF (Intra-Area).
        
    - **IA** — между разными зонами OSPF (Inter-Area).
        
- `N1, N2, E1, E2`: Обозначения для **внешних маршрутов OSPF** (те, что перераспределяются в OSPF извне), где N и E обозначают разные типы (NSSA - Not-So-Stubby Area, External).
